# 本番環境での動作確認計画

作成日：2026-05-04

---

## 目的

各フェーズの実装・テスト環境での確認が完了した後、
本番スプレッドシート・本番GASデプロイでの最終動作確認を行う。

---

## 本番確認の基本ルール

- 本番確認は実際の会員データを使わずに行う（テスト用ダミーデータを使う）
- テスト用の会員データは確認後に必ず削除する
- 本番確認は営業時間外に行う
- 問題が発生した場合はすぐにオーナーに報告する

---

## テスト用ダミーデータ

本番確認用に使用するテストユーザー。実在する個人情報は使わない。

| 項目 | テストデータ |
|---|---|
| 氏名 | テスト 太郎（ひらがな） |
| フリガナ | テスト タロウ |
| 生年月日 | 1990-01-01（仮） |
| 住所 | 兵庫県朝来市（テスト用・実在しない番地） |
| 電話番号 | 0901234xxxx（架空番号） |
| コース | 一般コース（PLAN-001） |

## Phase 1 本番確認

### 確認項目

- [ ] 本番スプレッドシートの全シートにアクセスできる
- [ ] GASスクリプトが本番スプレッドシートに接続できる
- [ ] getSpreadsheetId() が正しいIDを返す
- [ ] getSheet(SHEET_NAMES.SETTINGS) でSettingsシートを取得できる
- [ ] getSetting('member_id_prefix') が 'W' を返す

---

## Phase 2 本番確認

### 確認手順

1. タブレットでGAS WebアプリのURLを開く
2. ダミーデータを使って全ステップを入力する
3. 確認画面で内容を確認する
4. 「この内容で申し込む」をタップする
5. 受付番号が表示されることを確認する
6. 本番スプレッドシートのIntakeApplicationsシートを開いて確認する
7. データが正しく保存されていることを確認する
8. テストデータを削除する

### 確認項目

- [ ] タブレット（iPad）でフォームが正常に表示される
- [ ] 全ステップの入力ができる
- [ ] 確認画面に入力内容が正しく表示される
- [ ] 申込後にIntakeApplicationsシートにデータが保存される
- [ ] 受付番号が表示される

---

## Phase 3 本番確認

### 確認手順

1. Phase 2 で入力したテスト申込を使用する
2. スタッフPCでGAS WebアプリのURLを開く（スタッフアカウントでログイン）
3. 申込一覧にテスト申込が表示されることを確認する
4. テスト申込をクリックして詳細画面を開く
5. 会員番号・鍵番号を割り当てて「正式登録する」をクリックする
6. 各シートの更新を確認する
7. テストデータを削除する

### 確認項目

- [ ] スタッフ画面が表示される（認証が必要な場合はログインを確認）
- [ ] 未確認申込が一覧に表示される
- [ ] 初回費用が正しく計算されて表示される
- [ ] 正式登録後にMembersシートに行が追加される
- [ ] 正式登録後にKeyCardsシートが更新される
- [ ] 正式登録後にPaymentsシートに初回費用が記録される
- [ ] 正式登録後にAuditLogsシートに記録される

---

## Phase 5 本番確認（ステータス変更）

### 確認手順

テスト会員を作成して（Phase 3 の手順で作成）、以下を確認する。

1. テスト会員に休会処理を行う → Membersのstatus = pause を確認
2. StatusHistoryに記録されることを確認する
3. テスト会員に再開処理を行う → Membersのstatus = active を確認
4. StatusHistoryに記録されることを確認する
5. テスト会員に退会処理を行う → Membersのstatus = withdrawn を確認
6. StatusHistoryに記録されることを確認する
7. テストデータを削除する

---

## Phase 7 本番確認（リコーリースエクスポート）

### 確認手順

実際の本番会員データを使って確認する（CSVをダウンロードするが、リコーリースには提出しない）。

1. 請求ダッシュボードを開く
2. 対象月を選択してプレビューを表示する
3. 対象会員数・合計金額が正しいことを確認する
4. 「CSVエクスポート」をクリックしてCSVをダウンロードする
5. ダウンロードしたCSVをExcelで開いて内容を確認する
6. BillingExportsシートに記録されていることを確認する

### 確認項目

- [ ] 対象会員数が正しい（アクティブ会員のみ）
- [ ] 休会中会員が除外されている
- [ ] 退会済み会員が除外されている
- [ ] CSVのフォーマットがリコーリースの要求通りである
- [ ] BillingExportsシートに記録される

---

## テストデータの削除手順

本番確認で作成したテストデータは確認後に必ず削除する。

1. Membersシートのテスト会員行を削除する
2. IntakeApplicationsシートのテスト申込行を削除する
3. PaymentsシートのテストPayment行を削除する
4. StatusHistoryシートのテスト履歴行を削除する
5. KeyCardsシートの該当番号を available に戻す
6. AuditLogsシートのテストログ行を削除する（操作ログも含めて削除）
