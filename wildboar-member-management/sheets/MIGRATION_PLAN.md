# 既存スプレッドシートから新システムへのデータ移行計画

作成日：2026-05-04

---

## 概要

現在スタッフが管理している既存のGoogleスプレッドシート（手入力の会員一覧）から、
新システムのスプレッドシート（SHEET_SCHEMA.md で定義した構造）へデータを移行する手順。

---

## 移行の前提条件

- 新システムのスプレッドシートが INITIAL_SETTINGS.md の手順で作成済みであること
- 既存スプレッドシートの列構成を事前に確認していること（要オーナーによる確認）
- 移行作業は営業時間外（会員の入退会が発生しない時間帯）に行うこと
- 移行前に既存スプレッドシートのバックアップを取ること

---

## 移行前の確認事項

### 既存スプレッドシートの構造確認

移行前に以下を確認して、このドキュメントに記録する。

| 確認項目 | 確認結果 |
|---|---|
| 既存スプレッドシートの列構成 | 未確認（Phase 1 開始前に確認する） |
| 現在の会員数（アクティブ） | 未確認 |
| 現在の会員数（休会中） | 未確認 |
| 現在の会員数（退会済み） | 未確認 |
| 最大会員番号 | 未確認 |
| 使用中の鍵番号 | 未確認 |
| 空き鍵番号 | 未確認 |

### データクレンジングのチェックリスト

移行前に既存データで確認・修正する項目。

- [ ] 会員番号の重複がないか確認する
- [ ] 鍵番号の重複がないか確認する
- [ ] 氏名のフリガナが全角カタカナになっているか確認する
- [ ] 電話番号のフォーマットを統一する（ハイフンなし数字のみ）
- [ ] 郵便番号のフォーマットを統一する（7桁、ハイフンなし）
- [ ] ステータス表記を統一する（在籍・休会・退会 → active/pause/withdrawn）
- [ ] 生年月日のフォーマットを統一する（YYYY-MM-DD形式）
- [ ] 入会日のフォーマットを統一する（YYYY-MM-DD形式）

---

## 移行手順

### Step 1：既存データのエクスポート

1. 既存スプレッドシートをGoogleドライブから開く
2. メニュー「ファイル」→「ダウンロード」→「カンマ区切り形式（.csv）」でエクスポートする
3. エクスポートしたCSVを安全な場所に保存する（移行後も削除しない）

### Step 2：データクレンジング

エクスポートしたCSVをExcelまたはGoogleスプレッドシートで開き、以下の処理を行う。

1. 不要な列を削除する（新システムで使用しない列）
2. 新システムの列名に合わせてヘッダーを変更する
3. ステータスの表記を変換する（在籍→active, 休会→pause, 退会→withdrawn）
4. フリガナを全角カタカナに統一する
5. 電話番号からハイフンを除去する
6. 郵便番号を7桁統一する
7. 日付をYYYY-MM-DD形式に統一する
8. 新システムのフィールドで現在の既存データにない項目（intake_application_id等）は空欄にする

### Step 3：KeyCardsシートの準備

1. 既存データから各会員の鍵番号を確認する
2. KeyCardsシートに全鍵番号を入力する（status = available）
3. 会員データから使用中の鍵番号を確認し、status = in_use に変更する

### Step 4：MembersシートへのデータインポートGAS版

1. クレンジング済みのCSVを確認する
2. データをMembersシートに貼り付ける（ヘッダー行の下から）
3. 全行の列が正しい位置に入っていることを確認する

### Step 5：整合性確認

インポート後に以下を確認する。

- [ ] インポート件数が既存データの件数と一致する
- [ ] status別の件数が一致する
- [ ] 会員番号の最大値が既存データと一致する
- [ ] 使用中の鍵番号がKeyCardsシートと一致する
- [ ] 無作為に5件サンプリングして内容を目視確認する

### Step 6：GASのgetNextMemberIdを確認

データインポート後、次に採番される会員番号が正しい番号になっているか確認する。
（既存の最大番号 + 1 が次の番号として生成されること）

---

## 注意事項

- 移行作業中は新規の入会申込を受け付けない
- 移行中に既存スプレッドシートを変更しない
- 移行後は一定期間（1ヶ月程度）既存スプレッドシートを参照可能な状態で残す
- 移行完了後に既存スプレッドシートを読み取り専用に変更する

---

## フォールバック計画

移行に問題が発生した場合の対処。

1. 移行前のバックアップCSVが手元にある
2. 既存スプレッドシートは削除していないので、既存の運用に戻すことができる
3. 新システムのMembersシートのデータを全削除して最初からやり直す
