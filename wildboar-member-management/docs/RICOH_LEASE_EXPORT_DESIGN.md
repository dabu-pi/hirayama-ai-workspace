# リコーリース集金代行エクスポート設計

作成日：2026-05-04

注意：リコーリースの実際の提出フォーマット・必須項目については、
リコーリースの担当者に確認してから実装すること。
このファイルは設計のひな型であり、確認後に更新する。

---

## リコーリース集金代行の仕組み

リコーリース（Ricoh Leasing）が提供する集金代行サービスを利用して、
会員から月会費を口座振替で回収する。

### 月次フロー

```
毎月（日）までに提出   振替日（日）
      |                    |
      v                    v
リコーリースへ          会員口座から
集金データ提出          自動振替実行
```

1. ジム側：毎月指定の締め日までに対象会員リストをリコーリースへ提出する
2. リコーリース：指定の振替日に会員の口座から月会費を引き落とす
3. リコーリース：回収した金額をジムの口座に入金する
4. リコーリース：振替結果（成功・失敗）をジムに通知する

### 現在の課題

- 集金データの作成が毎月手作業で数時間かかっている
- 休会・退会者の除外漏れによる誤請求リスクがある
- 提出後の確認記録が残らない

## エクスポートに必要なデータ項目

以下は想定される項目。実際の必須項目はリコーリースに確認すること。

| 項目 | 説明 | Membersシートの対応フィールド |
|---|---|---|
| 契約者番号 | リコーリースが付番した会員識別番号（要確認：会員番号と同一か別途管理か） | member_id または別フィールド |
| 契約者氏名 | 口座名義人氏名（フリガナ） | family_name_kana + given_name_kana |
| 引落金額 | 月会費（円） | MembershipPlans.monthly_fee |
| 備考 | 月度・コース等（必要かどうか要確認） | 生成 |

実際に必要な項目（銀行口座情報・金融機関コード等）はリコーリースに確認する。
現在のシステムに銀行口座情報フィールドがない場合は Members シートに追加が必要。

## 出力形式

### 想定フォーマット（要確認）

- ファイル形式：CSV（Shift-JISまたはUTF-8）
- 文字コード：リコーリースの要求に従う（多くの場合 Shift-JIS）
- ヘッダー行：あり / なし（要確認）
- 区切り文字：カンマ（,）

### ファイル命名規則

```
wildboar_billing_YYYYMM.csv
```

例：2026年6月分 → `wildboar_billing_202606.csv`

## エクスポートの処理フロー

### システム上の操作

1. スタッフが請求ダッシュボード（billing-dashboard.html）を開く
2. 「集金データ作成」ボタンをクリックする
3. 対象月を確認する（デフォルトは翌月）
4. プレビュー画面で対象会員リストを確認する
5. 問題なければ「エクスポート」ボタンをクリックする
6. CSVファイルがダウンロードされる

### 対象会員の判定（自動処理）

| 会員ステータス | 対象月の振替 |
|---|---|
| active | 対象に含める |
| pause（休会中） | 対象から除外する（休会費がある場合は別途処理） |
| withdrawn（退会済み） | 対象から除外する |

判定のロジック：対象月の1日時点でのステータスを StatusHistory から確認する。

---

## 休会・退会者の除外処理

### 月次の判定方法

エクスポート実行時に以下を確認する。

1. Members シートの現在ステータスが active の会員を取得する
2. StatusHistory を確認し、対象月に休会・退会が発生した会員を除外する
3. 対象月に再開した会員は含める

### 月中ステータス変更の扱い

| ケース | 対応 |
|---|---|
| 対象月の途中で退会 | 退会月の振替は設定による（全額 / 日割り / なし）（要確認） |
| 対象月の途中で休会開始 | 翌月から除外（当月は通常請求）（要確認） |
| 対象月の途中で再開 | 再開月の振替は設定による（日割り / 全額 / なし）（要確認） |

## BillingExports シートとの連携

エクスポートを実行するたびに BillingExports シートに記録する。

| 記録内容 | 説明 |
|---|---|
| export_id | エクスポートID（自動採番） |
| billing_month | 集金対象月（YYYY-MM） |
| export_date | エクスポート実行日時 |
| exported_by | 実行したスタッフ |
| member_count | 対象会員数 |
| total_amount | 合計集金額（円） |
| file_name | 出力ファイル名 |
| status | exported（初期）→ submitted（提出後）→ completed（振替完了後） |

## プレビュー画面の表示項目

エクスポート前に以下を表示して確認させる。

- 対象月
- 対象会員数
- 合計金額
- 会員一覧（会員番号・氏名・コース・金額）
- 前月との差分（増減した会員）
- 除外された会員（休会・退会者）

---

## エラーハンドリング

| エラーケース | 対応 |
|---|---|
| 月会費が未設定の会員がいる | 警告を表示して対象から除外し、スタッフに確認を促す |
| 同じ月のエクスポートが既に存在する | 警告を表示する。上書きは確認後のみ許可する |
| 対象会員が0名 | 警告を表示する（正常か確認を促す） |

---

## セキュリティ上の注意

- エクスポートしたCSVには個人情報（氏名・金融機関情報等）が含まれる場合がある
- ダウンロードしたファイルの管理はスタッフの責任で行う
- ファイルをメール送信する際は暗号化または専用のファイル転送ツールを使う
- ファイルを使用後は適切に削除する
