# 個人情報・セキュリティ方針

作成日：2026-05-04

---

## 収集する個人情報の種類

本システムは以下の個人情報を収集・管理する。

### 必須収集項目

| 情報の種類 | 項目 | 収集目的 |
|---|---|---|
| 氏名 | 漢字氏名・フリガナ | 会員識別・集金代行データ |
| 生年月日 | 生年月日・年齢 | 年齢確認・コース適用判定 |
| 性別 | 性別 | サービス提供・統計 |
| 住所 | 郵便番号・都道府県・市区町村・番地 | 顧客管理・書類送付 |
| 電話番号 | 携帯電話番号 | 緊急連絡・業務連絡 |
| 緊急連絡先 | 氏名・続柄・電話番号 | 緊急時の連絡 |

### 任意収集項目

| 情報の種類 | 項目 | 収集目的 |
|---|---|---|
| 電話番号 | 自宅電話番号 | 業務連絡 |
| メールアドレス | メールアドレス | 業務連絡・案内 |
| 住所補足 | 建物名・部屋番号 | 書類送付 |
| 職業 | 職業 | 統計・サービス改善 |

### システム管理項目（入力者には非公開）

| 情報の種類 | 項目 | 管理目的 |
|---|---|---|
| 会員番号 | member_id | 会員識別 |
| 鍵番号 | key_card_number | 入退館管理 |
| 入会日 | join_date | 請求計算 |
| ステータス履歴 | StatusHistory | 休会・退会の追跡 |
| 支払い記録 | Payments | 請求管理 |

## AIへの送信禁止方針

本システムの開発・運用において、以下を厳守する。

### 禁止事項

- 会員の氏名・住所・電話番号・メールアドレス・生年月日等の個人情報を、Claude / ChatGPT 等のAIサービスに入力・送信しない
- スプレッドシートのスクリーンショットや出力内容（個人情報が含まれる可能性がある場合）をAIに送らない
- AIとの会話ログに個人情報が記録されないよう注意する

### AIを使ってよい用途

- システムの設計・仕様の相談
- コードのデバッグ・改善の相談
- テスト用のダミーデータ生成（実在する個人情報は使わない）
- エラーメッセージのトラブルシューティング（個人情報を含まない形で）

## アクセス権限設計

### GAS Webアプリの設定

| 画面 | アクセス権限 | 理由 |
|---|---|---|
| 入会フォーム | 全員（ログイン不要） | 来店者がタブレットから入力するため |
| スタッフ管理画面 | 組織のユーザーのみ（またはGoogleアカウントログイン必須） | スタッフのみが操作するため |

### スプレッドシートのアクセス権限

| 対象 | 権限 |
|---|---|
| ジムオーナー | 編集権限 |
| スタッフ | 編集権限（シートによっては閲覧のみ） |
| その他 | アクセス不可 |

### 将来的なロール設計（Webアプリ版）

| ロール | 権限 |
|---|---|
| admin | 全機能へのアクセス。スタッフ管理・設定変更が可能 |
| staff | 会員管理・申込確認・ステータス変更が可能。設定変更は不可 |
| viewer | 会員一覧・詳細の閲覧のみ。変更不可 |

## バックアップ方針

### GAS版

- Google Sheetsは自動的にバージョン履歴が保存される（変更履歴から復元可能）
- 毎月末に手動でスプレッドシートをGoogle Driveにコピーして保管する（手順は INITIAL_SETTINGS.md を参照）
- GASのコードは GitHub で管理する

### Webアプリ版（Supabase移行後）

- Supabaseの自動バックアップを有効にする（Point-in-time Recovery）
- 毎週日曜日に手動エクスポートを実施してローカルに保管する

## 退会後のデータ保持期間

| 情報の種類 | 保持期間 | 理由 |
|---|---|---|
| 会員情報（氏名・住所等） | 退会後（設定値）年（要確認） | 法的要件・問い合わせ対応 |
| 支払い記録 | 退会後（設定値）年（要確認） | 経理・税務上の要件 |
| 操作ログ | 退会後（設定値）年（要確認） | トラブル対応 |

保持期間の具体的な年数はオーナーが確認して設定すること（税務上の要件：7年が目安）。

保持期間終了後の処理：
- Members シートから当該会員の行を削除する、またはマスキングする
- 関連する支払い記録・StatusHistory の削除または匿名化
- 削除操作は必ず AuditLogs に記録する

トレーニングジム ワイルドボア（以下「当ジム」）は、入会申込にあたり、

個人情報の開示・訂正・削除については当ジムスタッフにお申し出ください。

## セキュリティインシデント対応

| インシデント | 対応手順 |
|---|---|
| スプレッドシートへの不正アクセス | Googleアカウントのパスワード変更・共有設定の確認・アクセス履歴確認 |
| データの紛失 | Google Sheetsのバージョン履歴から復元 |
| 情報漏洩の疑い | 対象会員への通知・原因調査・再発防止策の実施 |
| スタッフアカウントの不正使用 | 該当アカウントのアクセス権を即時削除 |

## 開発時の注意事項（AI_RULES.md との整合）

- テストデータは架空の氏名・住所を使用する（実在する人物の情報を使わない）
- テスト用のスプレッドシートは本番スプレッドシートとは別に作成する
- GitHub に個人情報・認証情報を push しない
- スプレッドシートIDは GitHub には記載しない（Config.gs のコメントにのみ書く）
