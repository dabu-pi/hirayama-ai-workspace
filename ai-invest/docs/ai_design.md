# AIスクリーニング補助 設計メモ

最終更新: 2026-03-10
ステータス: 設計中（Phase0）

---

## AIをどう使うか

AIは「判断する」のではなく「整理・補助する」ために使う。

### 用途リスト

| 用途 | 具体的なアクション |
|---|---|
| スクリーニング条件チェック | 銘柄データをAIに渡し、SCREENING_RULES.md の条件を満たすか確認させる |
| 決算・ニュース要約 | 決算短信・IR資料・ニュースをAIに要約させる |
| 銘柄比較 | 複数候補をAIに比較させ、強み・弱みを整理させる |
| 週次・月次振り返り | 取引記録をAIに渡し、パターン分析・改善点提案をさせる |
| 相場状況サマリー | 相場環境の確認補助（マクロ・セクター動向） |

---

## 使わないこと（AIに任せてはいけないこと）

| 禁止事項 | 理由 |
|---|---|
| 「この銘柄を買え」という指示に従う | 最終判断は人間 |
| AIが「良い」と言った銘柄を無確認でエントリー | ルール違反 |
| 損切りラインをAIに変更させる | ルールの骨格は人間が守る |

---

## 将来の自動化構想（Phase3以降）

- Python スクリプトで財務データ・株価データを取得
- Claude API でスクリーニング条件のチェックを自動化
- 結果を Google Sheets に書き込む
- 週次レポートを自動生成

---

## 実装優先順位

1. まず手動でスクリーニング・取引記録を運用する（Phase0〜1）
2. 繰り返し作業が発生したら自動化を検討する（Phase2以降）
3. AI自動化はルールが固まってから（Phase3）
