## 全体シート構成

Google Sheets 1ファイルに以下のシートを作成する。

| シート名 | 役割 | 優先度 |
|---|---|---|
| **スクリーニング記録** | 16銘柄の条件チェック結果を記録 | **最初に作る** |
| **取引記録** | エントリー・エグジットの全取引履歴 | Phase 1 開始時 |
| 保有一覧 | 現在の保有銘柄・評価損益の確認 | Phase 2 以降 |
| 損益サマリー | 月次・累計の損益集計 | Phase 2 以降 |

> まず「スクリーニング記録」と「取引記録」の2シートを作る。他はPhase 2以降。

## シート①: スクリーニング記録

### 目的

16銘柄を SCREENING_RULES.md のチェックリスト（5ステップ）に沿って手動確認し、
「通過 / 保留 / 除外」に分類するための記録表。
月に1回程度を目安に実施する（定期スクリーニング）。

### 列設計（A列〜O列）

| 列 | 列名 | 入力内容 | 入力方法 |
|---|---|---|---|
| A | 確認日 | スクリーニングを実施した日付（YYYY-MM-DD） | 手入力 |
| B | コード | 証券コード（例: 6861） | 手入力 |
| C | 銘柄名 | 銘柄名（例: キーエンス） | 手入力 |
| D | 分類 | A / B / C / D / E | 手入力 |
| E | 株価>25日MA | ○ / ✕ | 手入力（チャートで目視確認） |
| F | 株価>75日MA | ○ / ✕ | 手入力（チャートで目視確認） |
| G | 1か月騰落率>+5% | ○ / ✕ | 手入力（始値・終値で計算） |
| H | 出来高条件OK | ○ / ✕ | 手入力（5日平均 ≥ 20日平均×80%） |
| I | 売上成長OK | ○ / ✕ | 手入力（直近決算で前年比プラスか） |
| J | 営業利益成長OK | ○ / ✕ | 手入力（直近決算で前年比プラスか） |
| K | PER≦50 | ○ / ✕ | 手入力（Yahoo!ファイナンス等で確認） |
| L | 自己資本比率≧30% | ○ / ✕ | 手入力（決算短信・四季報で確認） |
| M | 総合判定 | **通過 / 保留 / 除外** | 手入力（下記ルールで判定） |
| N | 保留・除外理由 | 満たさなかった条件を簡潔に記入 | 手入力（例:「75日MA下」「PER62倍」） |
| O | 監視メモ | 次回確認時のポイント・気づきなど | 手入力 |

### 総合判定ルール

| 判定 | 条件 |
|---|---|
| **通過** | E〜L の8項目すべてが ○ |
| **保留** | 7項目が ○（1項目のみ ✕）かつ、✕の項目が「出来高」または「騰落率」のみ |
| **除外** | 2項目以上が ✕、またはテクニカル（E/F）・ファンダ主要（I/J）が ✕ |

> 「惜しい」は保留ではなく除外に回すのが基本。保留銘柄は翌月に再チェックする。

### 入力の手順（1銘柄あたり5〜10分）

1. Yahoo!ファイナンスで対象銘柄のページを開く
2. チャートで 25日・75日移動平均線を表示し、E・F列を入力
3. 1ヶ月前の株価と現在値を比較し G列を入力（（現在値÷1か月前値−1）×100 が +5% 超か）
4. 出来高グラフで 5日平均と 20日平均を比べ H列を入力
5. IRページ・決算短信で最新決算を開き I・J列を入力（決算発表日も確認。30日以内か）
6. Yahoo!ファイナンスの「財務」タブで PER・自己資本比率を確認し K・L列を入力
7. M・N・O列に判定とメモを入力

### シートのヘッダー行イメージ

```
A: 確認日 | B: コード | C: 銘柄名 | D: 分類 | E: 株価>25日MA | F: 株価>75日MA |
G: 騰落率>+5% | H: 出来高OK | I: 売上成長 | J: 利益成長 | K: PER≦50 |
L: 自己資本≧30% | M: 総合判定 | N: 保留・除外理由 | O: 監視メモ
```

### 運用ルール

- 実施頻度: **月1回**（相場急変時は追加実施してよい）
- 記録はその月のスクリーニング結果を1行1銘柄で追記していく（上書きしない）
- 「通過」の銘柄が実際のエントリー候補になる。エントリーは取引記録シートに移行して管理

## シート②: 取引記録

### 目的

エントリー・エグジットの全取引履歴を記録し、損益管理と振り返りに使う。
ペーパートレード（Phase 1）も同じシートに記録し、Phase 2 以降の実取引と区別する列を設ける。

### 列設計（A列〜O列）

| 列 | 列名 | 入力内容 |
|---|---|---|
| A | 日付 | 取引日（YYYY-MM-DD） |
| B | 種別 | **ペーパー / 実取引** |
| C | コード | 証券コード |
| D | 銘柄名 | |
| E | 売買区分 | 買 / 売（部分） / 売（全） |
| F | 取引単価 | |
| G | 数量 | |
| H | 取引金額 | F×G で自動計算可 |
| I | 損切りライン | エントリー時に設定した価格（買い時のみ） |
| J | 部分利確ライン | エントリー価格×1.10（買い時のみ） |
| K | 全利確ライン | エントリー価格×1.20（買い時のみ） |
| L | 損益（円） | エグジット時に記入（売り時のみ） |
| M | 損益（%） | エグジット単価÷エントリー単価−1（売り時のみ） |
| N | エグジット理由 | 損切り / 部分利確 / 全利確 / 期間利確 / 根拠消滅 |
| O | エントリー根拠・メモ | スクリーニング通過の主な根拠を一言で |

### 取引記録の運用ルール

- エントリー時は B〜K・O列を必ず記入してからエントリー（ペーパーも実取引も同じ）
- エグジット時は L・M・N列を記入する
- 1銘柄で部分利確（+10%で50%売却）するときは、売却分と残り保有分を別行で管理する
- 損益（%）の列は月次レビュー時に合計を確認する
