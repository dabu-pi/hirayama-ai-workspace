# SCREENING_RULES.md — スクリーニング・エントリー条件定義

最終更新: 2026-03-10
バージョン: 0.1（設計中）

---

## 概要

このファイルは「エントリーしてよい条件」を定義する。
すべての条件を満たしたときのみエントリーを検討する。
1つでも満たさない場合は見送る。

---

## エントリー前チェックリスト（必須）

### ステップ1: ユニバース確認

- [ ] `UNIVERSE.md` に記載されている銘柄・資産か

### ステップ2: 資金管理確認

- [ ] 1銘柄の投資額が総資産の30%以下か
- [ ] 同一セクターへの投資が総資産の40%以下か
- [ ] 当月の損失が -10% を超えていないか（超えていればエントリー禁止）

### ステップ3: スクリーニング条件（日本株）

> ※ 具体的な数値は設計中。以下はたたき台。

| 条件カテゴリ | チェック項目 | 基準（設計中） |
|---|---|---|
| トレンド | 株価が200日移動平均線より上か | 上 |
| モメンタム | 直近1ヶ月の株価変化率 | +5%以上（目安） |
| 出来高 | 直近5日平均出来高 vs 20日平均 | 増加傾向 |
| 業績 | 直近決算：増収または増益か | どちらか一方以上 |
| バリュエーション | PER（株価収益率） | セクター平均以下 |
| 財務 | 有利子負債 / 自己資本比率 | 要確認 |

### ステップ4: 定性確認

- [ ] 直近1週間以内に重大ネガティブニュースがないか
- [ ] 決算発表が直近2週間以内にある場合は見送るか検討する
- [ ] エントリー根拠を一言で言えるか（言えない場合は見送り）

### ステップ5: エグジット条件の事前設定（エントリー前に必ず決める）

- [ ] 損切りライン: エントリー価格の **-5%** を事前に設定した
- [ ] 利確ライン: 目標価格を事前に設定した（変更しない）

---

## エントリー禁止ルール

| 禁止パターン | 理由 |
|---|---|
| 損切りラインを設定せずエントリー | 損失が青天井になる |
| 「戻るかも」でラインを変更 | 損失拡大の典型パターン |
| ニュース・SNSの熱狂に乗る | 感覚売買。条件未確認 |
| 当月損失 -10% 超の後のエントリー | 月次上限ルール違反 |
| ユニバース外の銘柄 | ルール違反 |

## 暗号資産スクリーニング（設計中）

> ※ BTC・ETH のみ対象。条件は日本株より保守的に設計する。

| 条件 | 基準（設計中） |
|---|---|
| トレンド | 週足で上昇トレンド |
| 投資上限 | 総資産の20%以下 |
| エントリータイミング | 相場が過熱していない局面（RSI 70以下が目安） |
| 損切り | エントリー価格の -7%（変動が大きいため日本株より広め） |

---

## AIスクリーニング補助の使い方

1. スクリーニング候補をAIに渡し、上記条件のチェック補助をさせる
2. 決算・ニュース要約を依頼し、定性確認の補助をさせる
3. AIが「良い」と言っても、自分でチェックリストを通す
4. 最終判断は必ず自分で行う
