### データソース

| 表示項目 | 取得元 | 備考 |
|---|---|---|
| 患者ID / 氏名 / 電話番号 | Patients | 主情報 |
| 最終来院日 | SelfPayVisits（MAX 来院日） | per patientId |
| 来院数 | SelfPayVisits（COUNT） | per patientId |
| 未収残高 | Payments（SUM 未収） | 入金状態=未収 or 一部入金 |

### 動作仕様

- 検索: 氏名・フリガナ・患者ID・電話番号の部分一致
- 未収あり患者は行を橙背景で強調
- 行クリック → S02 患者詳細統合画面へ遷移
- ページネーション: 20件/ページ

| サマリー項目 | データソース | 計算 |
|---|---|---|
| 最終来院日 | SelfPayVisits | MAX(来院日) |

| 現在の主訴 | SelfPayVisits | 最新来院の主訴 |
| 状態メモ | SelfPayChart | 最新カルテの次回予定 |
| 次回予定 | SelfPayVisits | 最新来院の次回方針 |

### メニュー選択 UI

- MenuMaster の `有効フラグ=TRUE` のみ表示
- カテゴリ別にグループ化（主力 → 個別パーツ → 評価入口）
- 選択後、税別価格・税込価格を自動セット
- 税込小計 = `Math.floor(税別単価 × 数量 × 1.10)`

### 保存処理（バックエンド）

```
1. itemId 生成: SPI_selfPayVisitKey_連番（明細行ごと）
2. SelfPayItems に INSERT（各明細行）
3. paymentId 生成: SPP_selfPayVisitKey
4. Payments に INSERT
5. SelfPayVisits.会計状態 を「会計済」に UPDATE
6. DailySales を当日集計で UPDATE（または新規 INSERT）
7. Run_Log に記録
8. → 領収書確認画面(S05)へ遷移
```

## 9. 技術方針（GAS HTML Service）

### ファイル構成（実装時）

```
jrec-sf01-selfpay/
  JREC_SF01_Setup.gs     ← 初期セットアップ（実装済み）
  JREC_SF01_Main.gs      ← doGet() Web アプリエントリ
  JREC_SF01_Patient.gs   ← 患者 CRUD
  JREC_SF01_Visit.gs     ← 来院・カルテ CRUD
  JREC_SF01_Billing.gs   ← 会計・支払・領収書
  JREC_SF01_Query.gs     ← 集計・検索クエリ
  index.html             ← SPA ベース HTML
  patient-list.html      ← S01
  patient-detail.html    ← S02（タブ含む）
  visit-form.html        ← S03
  billing-form.html      ← S04
  receipt.html           ← S05
  patient-form.html      ← S06
  style.css              ← 共通スタイル
```

### SPA vs MPA 方針

- **MPA（Multi-Page）推奨**：GAS HTML Service は SPA 構築が複雑になるため、画面遷移は URL パラメータ（`?page=detail&id=P0001`）で制御する
- `doGet(e)` の `e.parameter.page` で振り分け

### 認証

- GAS Web アプリのアクセス制限: **自分のみ（院内利用）**
- `Execute as: Me` / `Who has access: Only myself` で設定
