## 1. 全体画面構成

| ID | 画面名 | 主な役割 | Phase |

| S01 | 患者一覧画面 | 患者検索・一覧表示・患者選択 | Phase 2 MVP |

### 設計方針

| 項目 | 方針 |
|---|---|
| 検索対象 | 氏名・フリガナ・患者ID・電話番号（部分一致）|
| 未収強調 | 未収額 > 0 の行を橙背景（`#FFF3CD`）で表示 |
| 行クリック | S02 患者詳細統合画面へ遷移（`?page=detail&id=P0001`）|
| ページネーション | 20件/ページ |
| ソート | デフォルト: 最終来院日の降順 |
| 登録ボタン | 右上に固定。`?page=register` へ遷移 |

### データソース

| 表示列 | シート | 計算 |
|---|---|---|
| 患者ID / 氏名 / 電話番号 | Patients | 直接参照 |
| 最終来院日 | SelfPayVisits | MAX(visitDate) per patientId |
| 来院回数 | SelfPayVisits | COUNT(*) per patientId |
| 未収額 | Payments | SUM(totalTaxInc) WHERE paymentStatus=未収 or 一部入金 |

## 11. デザイン方針

| 項目 | 方針 |
|---|---|
| 配色 | 白ベース / アクセント色: 青（`#1a73e8`）/ 未収強調: 橙（`#FFF3CD`）|
| フォント | システムデフォルト。日本語が読みやすいサイズ（14〜16px）|

| 入力画面 | 縦スクロールで完結。横スクロール不要 |
| 受付用導線 | 患者一覧 → 患者詳細の2ステップで操作完結 |
| 誤操作防止 | 保存前に確認ダイアログは設けず、代わりにキャンセルボタンを常時表示 |
| 不可逆操作 | 削除機能は Phase 2 では実装しない |
