# JREC-SF01 自費カルテ・会計システム

## プロジェクト名

**JREC-SF01** — 自費カルテ・会計システム

JREC（柔整レコードシステム）の自費特化サブプロジェクト。

---

## 目的

接骨院・整骨院における**自費施術**専用のカルテ・会計・領収書・売上集計を管理する。

- 自費患者のカルテを記録する
- 自費施術の会計を行う
- 領収書を発行・管理する
- 日次・月次の自費売上を集計する
- 保険請求とは完全分離した管理体系を提供する

---

## JREC本体との違い

| 項目 | JREC本体 (`jyu-gas-ver3.1`) | JREC-SF01 (本プロジェクト) |
|---|---|---|
| 対象 | 保険施術 | 自費施術 |
| 主な機能 | 療養費支給申請、施術録、制度準拠ロジック | 自費カルテ、自費会計、領収書、売上集計 |
| スプレッドシート | 保険JREC専用 | 自費専用（別シート） |
| GASプロジェクト | jyu-gas-ver3.1 | jrec-sf01-selfpay |
| 制度制約 | 柔整療養費制度に準拠必須 | 自費のため制度制約なし |
| 将来展開 | 保険請求の精緻化・電子化 | Webアプリ化・外販モデル化 |

---

## JREC本体に混ぜない理由

1. **保険請求ロジックの保護** — 保険側に自費ロジックが混入すると制度準拠性・監査耐性が低下する
2. **clasp push 対象の分離** — GASプロジェクトを分けることでデプロイミスを防ぐ
3. **スプレッドシート構造の独立** — 自費特有のシート・列を自由に追加できる
4. **将来Webアプリ化** — 自費単独でNext.js / Supabase化しやすい設計にする
5. **外販時の切り離し** — 保険JRECを含まず自費パッケージとして外販できる
