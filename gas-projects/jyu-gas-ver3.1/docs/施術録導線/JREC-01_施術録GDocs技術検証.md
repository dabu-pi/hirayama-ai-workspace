## 4. 差し込みアプローチ比較

### アプローチ A: replaceText() プレースホルダー方式

```
body.replaceText('{{patient_name}}', '田中 花子')
```

| 項目 | 評価 |
|---|---|
| 実装容易度 | ○ シンプルで読みやすい |
| 前提条件 | テンプレートに `{{KEY}}` マーカーを挿入する必要あり |
| 固定欄への適用 | ◎ 最適（患者名・保険番号・負傷名等） |
| 繰り返し行への適用 | △ 行数が可変なので難しい |
| レイアウト影響 | ○ テキスト置換のみなのでレイアウト崩れリスク低 |

### アプローチ B: テーブルセル直接書き込み方式

```
tables[i].getRow(r).getCell(c).setText('値')
```

| 項目 | 評価 |
|---|---|
| 実装容易度 | △ テーブル Index / 行 / セルを事前に特定する必要あり |
| 前提条件 | テンプレートの構造が変わると壊れる |
| 固定欄への適用 | ○ 可能だが Index 管理が必要 |
| 繰り返し行への適用 | ◎ 最適（日別明細行への書き込みに必須） |
| レイアウト影響 | ○ setText() はフォント・サイズを保持 |

### 採用方針（暫定）

| 対象 | 採用アプローチ |
|---|---|
| 表面 固定欄（患者名・保険番号・負傷名等） | **A: replaceText** （テンプレートにプレースホルダーを追加後） |
| 裏面 日別明細行 | **B: テーブルセル直接書き込み** |
| 裏面 ①②③ 月次集計行 | **B: テーブルセル直接書き込み** |

## 5. sandbox コード

ファイル: `C:\hirayama-ai-workspace\claude-sandbox\WS_SR_sandbox.js`

### 実行手順

1. GAS スクリプトエディタ（任意のプロジェクト）に `WS_SR_sandbox.js` の内容を貼り付ける
2. `wsrSandboxMain()` を実行する
3. ログで以下を確認する：
   - 総テーブル数
   - 各テーブルの行数・セル数・先頭セルのテキスト
   - 書き込みの成否
   - PDF ファイル ID

4. ログ結果を元に各定数を修正する：

```javascript
var WS_SR_TABLE_HYOMEN   = 2;  // ← 表面：患者名テーブルのIndex
var WS_SR_ROW_PATIENT    = 1;  // ← 患者名の行Index
var WS_SR_CELL_PATIENT   = 1;  // ← 患者名のセルIndex
var WS_SR_TABLE_URAME    = 7;  // ← 裏面：日別テーブルのIndex
var WS_SR_FIRST_DATA_ROW = 1;  // ← データ開始行Index（ヘッダ行の次）
```

5. 出力された PDF をドライブで開いてレイアウト確認する

### テンプレート構造のみ確認（書き込みなし）

```javascript
wsrInspectTemplateOnly()  // テンプレート原本を直接確認（変更なし）
```
