## 0. このドキュメントの目的

「JRECの日次入力データを月次で施術録へ転記・確定保存する導線」を、
どの粒度で管理するかを判断し、その根拠・命名・フォルダ構造・再開手順を記録する。

## 1. 判断結果: JREC-01 内ワークストリーム化（独立プロジェクト化しない）

### 推奨結論

**JREC-01 内の独立ワークストリーム「WS-SR（施術録導線）」として管理する。**
新しい project_id は作らない。

### 判断理由

| 観点 | 判断内容 |
|---|---|
| **データ依存性** | 施術録の元データはすべて JREC-01 のシート（来院ヘッダ・施術明細・患者マスタ）に存在する。別プロジェクトにすると「JREC-01のシートを読む外部プロジェクト」という歪な依存関係になる |
| **技術基盤** | GAS コードは同一 clasp プロジェクト（jyu-gas-ver3.1）に同居する。Cloud Run も同一。シート基盤も同一スプレッドシート内 |
| **先例との整合** | 申請書生成（B案）という「月次出力層」がすでに JREC-01 内に実装済み。施術録もその並列機能として自然に収まる |
| **管理コスト** | 新 project_id を作ると Dashboard の Projects 行・Task_Queue の project_id・de コマンドの -ProjectId 指定がすべて増え、管理負荷が上がる |
| **JASSESS-01 との違い** | JASSESS-01 は「評価システム」として独自のシート・GAS・処理フローを持ち、JREC-01 への依存がないため独立が自然だった。施術録導線は JREC-01 なしには成立しない |

### 独立プロジェクト化を否定した理由の補足

施術録は「監査・経過閲覧の正本レイヤー」として独自のライフサイクルを持つが、
それはあくまで JREC-01 の出力物の一形態であり、申請書生成と同格の「月次出力機能」である。
規模が大きくなる場合でも、JREC-01 内のワークストリームとして専用ドキュメント領域を確保することで対応する。

## 4. 概念設計（現時点方針・実装前確定分）

### 4-1. 基本方針

| 方針 | 内容 | ステータス |
|---|---|---|
| 管理単位 | 「1患者 × 1負傷系列」を基本 | 現時点方針 |
| 転記サイクル | 月次で追記・転記・確定保存 | 現時点方針 |
| 出力形式 | テンプレート形式（Word/Excel/Google Sheets系） → 最終 PDF 保存 | 現時点方針 |
| 受傷部位図解 | 初版は手入力 | 現時点方針 |
| 転記後元データ | **削除しない。Archive化 / 退避前提で再検討** | 確定（削除案は不採用）|
| 法定保管 | 施術録は5年間保管義務（再診経過記録テンプレにも記載済み） | 確定 |

### 4-2. 「1患者 × 1負傷系列」の意味

同一患者でも負傷・症例の流れごとに施術録を持つ考え方。
月次追記形式と組み合わせる前提。

- 腰部捻挫の経過 → 腰部捻挫の施術録
- 頚部痛が別発生 → 頚部痛の施術録（別ファイル）

### 4-3. 自動転記候補（現時点整理）

**A. 自動転記候補（そのまま転記可能な見込み）:**

- 負傷の日時
- 負傷の場所
- 患者基本情報
- 保険情報
- 負傷名、負傷年月日、初検年月日、転帰、施術回数
- 来院日、算定区分、部位情報、窓口負担額、一部負担金、月次集計

**B. 自動転記候補だが整形ルールが必要なもの:**

- 負傷時の状況（文章整形）
- 初検時の所見（文章整形）
- 初検時相談支援の内容（文章整形）
- 施術経過所見（月次まとめ方未確定 → 未確定論点①）
- 負傷名・部位情報（表記統一ルール未確定）
- 金額・回数集計（集計軸の確認要）

**C. 手入力:**

- 受傷部位図解

**D. 初版後回し:**

- 同意医師氏名
- 同意年月日
- 労務不能に関する意見
- その他特殊ケース欄

### 4-4. 既存システムとの位置づけ

```
【日次】JREC-01 入力
  ├─ 来院ヘッダ（保険算定・会計）
  ├─ 自費明細シート
  └─ 施術明細シート（詳細upsert）
        ↓（月末）
【月次出力A】申請書生成（B案）← 既存・稼働中
【月次出力B】施術録生成（WS-SR）← 今回検討中・未実装
```

## 6. JREC-01 との境界・参照元の整理

| 項目 | JREC-01（既存） | WS-SR（新規導線） |
|---|---|---|
| 正本 | 来院ヘッダ・施術明細・患者マスタ（日次運用データ） + 申請書テンプレ xlsx | Google Docs の施術録 MASTER 1本 + 施術録生成コード |

| 更新頻度 | 毎来院時 | 月次（or 随時追記） |
| 法定要件 | 療養費支給申請書（保険者提出） | 施術録（院内5年保管） |
| GAS コード | Ver3_core.js / Ver3_amounts.js 等 | 新関数を同一ファイル or 新 .js に追加 |

> テンプレート正本と成果物の分離ルールは `docs/JREC-01_出力物テンプレート管理.md` と `docs/JREC-01_TEMPLATE_INDEX.md` を参照する。
