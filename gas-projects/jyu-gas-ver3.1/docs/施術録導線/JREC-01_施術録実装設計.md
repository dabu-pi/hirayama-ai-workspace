# JREC-01 WS-SR 施術録出力 — 本実装設計

作成日: 2026-04-02
更新日: 2026-04-02（プレースホルダー差分確認 → 4件修正・確定）
ステータス: **プレースホルダー最終確定 / Phase 1 実装着手可**
対象フェーズ: WS-SR Phase 1（保険施術録 月次 1患者出力）

---

## 1. 概要

JREC の日次入力データを集約し、柔道整復師療養費支給申請書に添付する
「施術録」を Google Docs テンプレートから月次で自動生成・PDF 化する機能。

| 項目 | 内容 |
|---|---|
| 入力 | 患者ID（`patientId`） + 対象年月（`YYYY-MM`） |
| 出力 | Google Docs コピー + PDF（Drive 保存） |
| トリガー | スプレッドシート カスタムメニュー「施術録を出力」→ ダイアログで月・患者を指定 |
| 実装場所 | `gas-projects/jyu-gas-ver3.1/Ver3_shuRecorder.js`（新規ファイル）|

### 3-4. 裏面 日別明細（テーブルセル直接書き込み）

来院ヘッダ + 施術明細シートを `visitKey` で結合し、対象月の来院日順に並べる。

| 列Index | 施術録列ヘッダ | 書き込む値 | 取得元定数キー |
|---|---|---|---|
| 0 | 月 | `M`（月数字） | `treatDate` |
| 1 | 日 | `D`（日数字） | `treatDate` |
| 2 | チェック | `✓` | 固定 |
| 3 | 初検料/時間外等 | 初版: 空白（手入力残し） | — |
| 4 | 整復料等 | 初版: 空白（手入力残し） | — |
| 5 | 後療料 | `baseOut` | `AM_DETAIL_COLS.baseOut` |
| 6 | 冷罨法料/温電法料 | `coldOut + warmOut`（合算） | `coldOut` / `warmOut` |
| 7 | 電療料 | `electroOut` | `AM_DETAIL_COLS.electroOut` |
| 8 | 明細書発行体制加算 | 初版: 空白（手入力残し） | — |
| 9 | 一部負担金 | `copay` | `HEADER_COLS.copay` |
| 10 | 整復・施術等の施術経過所見 | `shoken`（または `keikaNow`） | `CASE_COLS.shoken` |

> **裏面 所見列の主転記列: `CASE_COLS.shoken`（= "所見"）を優先。空の場合は `keikaNow`（= "経過_今回"）を補完。**

## 4. 初版スコープ確定

### 自動転記（Phase 1 v1）

| 対象 | 内容 |
|---|---|
| 表面 患者・保険情報 | 患者マスタから `replaceText` で全欄 |
| 表面 負傷名一覧 | 最大2件（部位1・部位2） |
| 表面 負傷記録 | 初検情報履歴から日時・場所・状況・初検所見 |
| 裏面 日別明細 | 後療料・冷温・電療・一部負担金・所見（対象月全来院日） |
| 裏面 ①月次集計 | 回数・合計金額・一部負担額・期間を集計して記入 |

### 手入力で残す（Phase 1）

| 対象 | 理由 |
|---|---|
| 一部負担割合チェックマーク | チェックボックス操作が GAS では困難 |
| 裏面 初検料・整復料列 | 初版は実装コスト大・頻度を見て判断 |
| 裏面 明細書発行体制加算 | 月次固定値 → 手入力後で確認 |
| 同意医師氏名・同意年月日 | 署名欄のため手書きが主 |
| 労務不能欄 | まれなケース |
| 受傷部位図解 | 手書きスタンプ等で対応 |
| 初検時相談支援 ①②③ チェック | チェックボックス操作が困難 |
| ②③月次集計行 | 複数月対応は Phase 2 以降 |

### Phase 1 スコープ外

| 対象 | 理由 |
|---|---|
| 市町村番号・受給者番号・公費番号 | 公費対応は別フェーズ |
| 自費混在ケース | データ構造が別ルート |
| 3部位以上の負傷名 | 実運用上まれなケース |
| 保険者所在地 | 患者マスタに列がない場合 |

## 5. 出力ファイル管理

### 命名規則

```
Google Docs: 施術録_{氏名}_{YYYY}年{MM}月
PDF:         施術録_{氏名}_{YYYY}年{MM}月.pdf
```

例: `施術録_田中花子_2026年04月` / `施術録_田中花子_2026年04月.pdf`

### 出力先フォルダ構造

```
平山接骨院再開プロジェクト2026（ID: 1XMx2e1ufCRqp7bhpfRRjvPDyXCESL83V）
└── 施術録/
    ├── 2026-04/
    │   ├── 施術録_田中花子_2026年04月       ← Google Docs（修正可能版）
    │   └── 施術録_田中花子_2026年04月.pdf  ← 確定提出版
    └── 2026-05/
        └── ...
```

> `施術録/YYYY-MM/` フォルダがなければ GAS が自動作成する。

### 再実行時の挙動

| 状況 | 挙動 |
|---|---|
| 同名ファイルが存在しない | 通常通り生成 |
| 同名 Google Docs が存在する | ダイアログ: 「上書き生成しますか？ はい / いいえ」|
| はい | 既存ファイルをトラッシュ → 再生成 |
| いいえ | キャンセル（処理中断） |

## 7. GASモジュール設計（`Ver3_shuRecorder.js`）

### 関数一覧

```
Ver3_shuRecorder.js
│
├── srShowDialog()                    ← カスタムメニューから呼び出し
│                                        患者ID・対象月の入力ダイアログを表示
│
├── srGenerateDocument(patientId, yearMonth)   ← メイン処理
│   ├── srGetPatientData_(patientId)           ← 患者マスタ取得
│   ├── srGetCaseData_(patientId, yearMonth)   ← 来院ケース取得（月内アクティブ）
│   ├── srGetVisitRows_(patientId, yearMonth)  ← 来院ヘッダ + 施術明細 結合
│   ├── srGetInitExamData_(patientId)          ← 初検情報履歴取得
│   ├── srGetOrCreateOutputFolder_(yearMonth)  ← 出力フォルダ取得/作成
│   ├── srCheckExistingFile_(folder, filename) ← 既存ファイル確認
│   ├── srDuplicateTemplate_(folder, filename) ← テンプレート複製
│   ├── srInsertHyomenData_(docId, allData)    ← 表面 replaceText
│   ├── srInsertUrameData_(docId, visitRows)   ← 裏面 日別明細 + 集計
│   ├── srExportPdf_(docId, folder, filename)  ← PDF 出力
│   └── srLogOutput_(patientId, yearMonth, docId, pdfId)  ← ログ記録
│
├── srFindUrameTable_(body)           ← 裏面テーブル content-based 検索
├── srFindSummaryRows_(table)         ← ①②③ 行 Index 検索
├── srSetCell_(row, cellIdx, text)    ← セル書き込みヘルパー（範囲外ガード付き）
└── srFormatDate_(dateVal, format)    ← 日付整形ユーティリティ
```

## 10. プレースホルダー差分確認記録（2026-04-02）

院長がテンプレートへの手入力を進行中に「施術回数1 はあるが 施術回数2 がない」差分を発見。
設計書を全精査した結果、以下の4件を追加・修正した。

| # | 種別 | プレースホルダー | 原因 | 処置 |
|---|---|---|---|---|
| G-01 | **設計漏れ（§3-2・§6）** | `{{施術回数2}}` | 施術回数は部位1だけ定義し部位2を忘れていた | §3-2・§6 に追加 |
| G-02 | **§6 漏れ** | `{{被保険者生年月日}}` | §3-1 には定義済みだが §6 リストから脱落 | §6 に追加 |
| G-03 | **設計未定義（§3-2・§6）** | `{{日数1}}` / `{{日数2}}` | 帳票上の「日数」欄が設計書に完全未定義だった | §3-2・§6 に追加 |
| G-04 | **扱い修正** | `{{保険種別}}` / `{{一部負担割合}}` | チェックボックス形式のため replaceText 不可 | §3-1 の扱いをスコープ外に修正。§6 への追加なし |

**テンプレートへ追加すべき残作業（院長が対応）:**

```
{{被保険者生年月日}}  ← 被保険者の生年月日欄
{{日数1}}            ← 負傷名1行目の「日数」欄
{{日数2}}            ← 負傷名2行目の「日数」欄
{{施術回数2}}        ← 負傷名2行目の「施術回数」欄
```

これら4点を追加すれば T-SR-01 完了。
