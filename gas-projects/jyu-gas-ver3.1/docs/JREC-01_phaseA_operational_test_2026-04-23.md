## 4. 観察項目

### 4-1 検索速度

- 患者マスタ件数が増えた状態での体感速度を確認する
- 目安: キーワード入力後 3 秒以内に候補が表示されること
- 3 秒を超える場合は記録欄に件数・端末・回線環境を記録する

### 4-2 保存成功 / 失敗

- 自費明細の保存が正常完了するか確認する（「N 件の自費明細を保存しました」表示）
- 失敗した場合はエラーメッセージ・操作手順・visitKey を記録する
- GAS ログ（Apps Script エディタ > 実行数）でエラー率も確認する（1 週間後メド）
- 確認対象関数: `getSelfPayDataByVisitKey_V3` / `saveSelfPayDetailsFromDialog_V3`

### 4-3 操作の迷い

- 「次に何をすればいいかわからない」場面があれば記録する
- 特に「患者選択後の遷移」「自費明細保存後の戻り先」を重点確認する

### 4-4 スマートフォン操作性

- iPhone または Android の実機で以下の一連を試す
  1. Web App URL を開く
  2. 患者をキーワード検索
  3. 患者カードをタップ
  4. 「自費明細入力 →」をタップ
  5. メニュー選択・数量入力・保存
- タップ対象が小さすぎる / スクロール不要な部分がスクロールされる / キーボードが出てUIが隠れる などを確認する

### 4-5 受付業務への適合性

- B7（会計区分）が selfPayWeb に正しく引き継がれているか確認する
- 既存スプレッドシートメニュー（来院登録・算定実行）と Web UI を同日混在使用した場合の挙動を確認する
- 「シートを開かずに自費入力を完結できたか」を記録する

## 8-C. Phase A-1 SC1 設計課題 — 来院ヘッダ未作成で自費明細保存できる設計ギャップ

### 記録日

2026-04-26

### 発生事象

SC1「前回自費明細がある患者で前回明細が反映される」を実施したところ、
「前回来院（04/22）に自費明細がありません。通常の明細入力を出します」が表示された。

実データ上は 04/23 に自費明細が存在していた。日付ずれ（UTC/JST）ではなく、設計ギャップと判断。

### 根本原因

```
来院ヘッダを作る唯一の経路: saveVisit_V3 のみ

Web UI や saveSelfPayDetailsFromDialog_V3 は来院ヘッダを作らない。
→ 来院ヘッダなしでも自費明細だけが保存できてしまう。

loadPrevSelfPayToDialog_V3 は前回来院日を「来院ヘッダ」から探す。
→ ヘッダに 04/23 行がなければ、04/23 の自費明細があっても前回明細として取得できない。
```

**調査経路の詳細:**

| 確認事項 | 結果 |
|---|---|
| 来院ヘッダ作成 | `saveVisit_V3` のみ |
| `setPatientAndDate_V3` のヘッダ作成 | なし（B2/B4 書き込みのみ） |
| `saveSelfPayDetailsFromDialog_V3` のヘッダ確認 | なし（ヘッダなしでも保存可） |
| `loadPrevSelfPayToDialog_V3` の前回来院日検索先 | 来院ヘッダのみ（自費明細シートは検索しない） |
| visitKey の日付フォーマット（`buildVisitKey_` / `fmt_`） | `"Asia/Tokyo"` 固定。UTC ずれなし ✓ |
| `patientSearch.html` の `todayYMD()` | ブラウザ JST。正しい ✓ |

### Phase A での対応方針

**コード変更なし。記録のみ。**

- `saveVisit_V3` は保険算定・月次集計・申請書・施術録に近い重要処理
- upsert 化は影響範囲が大きく、Phase A の実受付評価がぶれる
- 来院ヘッダの自動作成は二重記録禁止ポリシーと密接に関係する

### Phase B での対応方針

「本日の来院ラインを確定する」明示操作を設ける。

```
患者選択後 → [本日の来院ラインを確定する] ボタン
  → ensureVisitHeaderForPatientDate_V3 (設計候補)
    ├─ visitKey が来院ヘッダに存在: 既存行を再利用
    └─ 存在しない: 最小ヘッダ行を作成（保険算定列は空）
  → 確定後に自費明細・施術記録・会計へ進む
  → saveVisit_V3 は upsert 化（既存行に金額を上書き）
```

設計詳細: `docs/JREC-01_single_page_ui_design_2026-04-25.md` §8 参照  
PROJECT_STATUS.md 2026-04-26 セクション参照

SC1 設計課題に対して、Phase A 内の低リスク暫定対応として以下を実施した。

| 誤取得防止: 他患者・当日・未来日・不正 visitKey スキップ | ✅ PASS（コードレビュー） |
