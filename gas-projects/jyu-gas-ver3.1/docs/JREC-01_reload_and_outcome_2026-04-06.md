# JREC-01 過去日再読み込み・転帰後付け更新 実装記録

- 実装日: 2026-04-06
- 対象: `【毎日記録】来店管理施術録ver3.1`
- 追加ファイル: `Ver3_core.js`

---

## 背景・目的

- 過去の来院日を B4 に入力し、その日の記録を UI 上で振り返りたい
- 後から「治癒」「中止」等の転帰を来院ケースに記録したいケースがある
- 保険算定・金額・初検/再検ロジックは安全優先で一切再計算しない

---

| メニュー | 関数名 |

| 柔整管理 > 当日内容を再読み込み | `reloadVisitToUI_V3` |
| 柔整管理 > 転帰を更新 | `updateOutcomeFromUI_V3` |

## 追加した関数

### `reloadVisitToUI_V3`（公開）

患者画面の患者ID（C2）と来院日（B4）に一致する来院ケースを検索し、その日の入力内容をそのまま UI に復元する。

**処理フロー:**

1. C2（患者ID）・B4（来院日）のバリデーション
2. 未保存入力がある場合は確認ダイアログ
3. `findCaseRowByPatientDateCaseNo_` で Case1 / Case2 を検索
4. 両方なければ「見つかりません」アラートで終了
5. `writeExactCaseSrcToUI_` で各ケースを UI に書き込む
6. 区分（`readCaseKubun_` で取得）を C10 / C34 に反映
7. 所見（shoken）があれば書き込む
8. toast で完了通知

**自動引継ぎとの違い:**

| 項目 | 自動引継ぎ | 再読み込み |
|---|---|---|
| 参照する日付 | 対象日より前の直近日 | 対象日ちょうど |
| 温罨法・電療 | 強制 true に補完 | 記録値をそのまま復元 |
| 用途 | 当日入力の補助 | 過去日の閲覧・転帰更新前確認 |

---

### `writeExactCaseSrcToUI_`（プライベートヘルパー）

`findCaseRowByPatientDateCaseNo_` の戻り値を UI の 2 行に直接書き込む。

- `applyCaseRowToUI_Safe_` は前回値の補完用ロジックを持つため、別関数として実装
- src が null のときは行をクリアしてチェックボックスを FALSE に戻す
- 終了済み部位（`isEnded_`）には `applyEndedProtection_` で視覚保護を適用

**書き込む値:**

| UI 列 | 内容 |
|---|---|
| A | 部位 |
| B | 傷病 |
| C | 受傷日 |
| D | 冷罨法 |
| E | 温罨法 |
| F | 電療 |
| G | 終了日 |
| H | 転帰 |

---

### `readCaseKubun_`（プライベートヘルパー）

`findCaseRowByPatientDateCaseNo_` は kubun を返さないため、visitKey + caseNo で行を特定し kubun 列を直接読む補完ヘルパー。

---

### `updateOutcomeFromUI_V3`（公開）

UI 上の転帰（H列）と終了日（G列）を来院ケースシートに後付けで書き込む。

**処理フロー:**

1. C2（患者ID）・B4（来院日）のバリデーション
2. UI から 4 行分（Case1 部位1/2 + Case2 部位1/2）の転帰・終了日を読む
3. 終了日⇔転帰の整合チェック（BLOCKING）
   - 終了日あり + 転帰なし → エラー
   - 転帰あり + 終了日なし → エラー
4. `findCaseRowByVisitKeyAndCaseNo_` で Case1 / Case2 の行を検索
5. 更新内容のサマリーを確認ダイアログに表示
6. OK 後に cases シートの該当行へ書き込む

**更新する列のみ:**

| CASE_COLS | 内容 |
|---|---|
| `end1` / `tenki1` | ケース内 部位1 の終了日・転帰 |
| `end2` / `tenki2` | ケース内 部位2 の終了日・転帰 |

**絶対に触らない列:** 部位・傷病・受傷日・治療法フラグ（冷/温/電）・kubun・金額系・createdAt

---

## UI上の操作想定フロー（振り返り → 転帰更新）

```
1. B2 で患者を選択
2. B4 に過去の来院日を入力
3. 柔整管理 > 当日内容を再読み込み
   → その日の来院内容が UI に表示される
4. ケース行の H 列（転帰）と G 列（終了日）を入力
5. 柔整管理 > 転帰を更新
   → 確認ダイアログ → OK で来院ケースに書き込まれる
```

---

## 転帰後付け更新の運用ルール

- 転帰 = 治癒 / 中止 / 転医 のとき、終了日は原則「その来院日」を採用する
- 転帰と終了日はセットで入力する（片方だけでは保存できない）
- 金額・請求系は一切変更しない
