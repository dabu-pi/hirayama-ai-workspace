## 設計背景・目的

現在の保存後 UI:
- `writeAmountsToUI_V3_` が E2〜E6 に保険金額を書く
- `clearAfterSaveUI_V3_` は「入力欄をクリア + E2〜E6 は意図的に残す」設計
- `clearEntryUI_V3`（手動クリア）で E2〜E6 を消す
- アラートで visitKey / 来院合計 / 窓口負担 / 保険請求 を一時表示するが、閉じると消える

**課題:** アラートを閉じると金額が確認できない / 患者名・会計区分・保険外・合計が一画面で見えない

**目的:**
1. 保存後に「直前保存サマリー」として患者名・来院日・4項目の金額・visitKey を一覧表示する
2. 将来の領収証印刷の参照元をこのサマリーに統一する

---

## サマリーの定義

### 表示項目（領収証様式準拠）

| 項目名 | 定義 | 現在の変数 / 取得元 |
|---|---|---|
| 患者名 | 保存時の患者表示名 | 患者マスタから取得 or UI B3 |
| 来院日 | 施術日 | UI B4 |
| 会計区分 | 保険のみ / 保険+自費 / 自費のみ | UI B7 |
| 保険分合計 | 保険算定合計（参考表示）| `amounts.visitTotal` |
| 一部負担金 | 患者窓口負担額 | `amounts.windowPay` |
| 保険外 | 自費売上合計 | 自費明細シートの当該 visitKey 合計 |
| **合計金額** | **= 一部負担金 + 保険外** | **実際の受領額** |
| 保存時刻 | GAS 実行時刻 | `new Date()` |
| visitKey | 冪等キー | `visitKey` |

> **合計金額 = 一部負担金 + 保険外**（保険分合計は参考表示）
> 実際に患者から受け取る額は「合計金額」のみ。

## 配置案の比較

### 案A: 患者画面シート内の専用サマリーエリア【推奨・短期】

**場所:** 患者画面シートの行 55〜62（旧会計ブロック跡地）を「直前保存サマリー」として再活用

**実装概要:**
- `writeSavedSummaryUI_V3_(uiSh, summaryObj)` を新設
- `clearAfterSaveUI_V3_` 末尾でサマリーを書く（入力欄クリアとは独立）
- `clearEntryUI_V3` でサマリーもクリア（患者画面全クリア時に消える）
- `setupSelfPayValidation_V3_` の `"A55:H62".clearContent()` を廃止 or 初回1回だけに変更

**注意:** 現在 `setupSelfPayValidation_V3_` が「旧会計ブロック残骸クリア」として A55:H62 を毎回クリアしている。この処理を廃止または初回1回だけに変更する必要がある（**実装時に要注意**）。

**利点:**
- 患者画面を離れずに確認できる
- アラートの代替として機能する
- シート追加不要

**欠点:**
- 患者画面シートが縦に長くなる（行55〜62 は通常スクロール外）
- 印刷時は印刷範囲外にする設定が必要

---

### 案B: 専用「会計サマリー」シート【推奨・中長期・印刷対応】

**場所:** 新シート「会計サマリー」を追加

**実装概要:**
- 保存時に `writeSummarySheet_V3_(summaryObj)` で最新1行を書く（または先頭行を上書き）
- シートレイアウトを領収証に合わせて整形する
- 将来の「印刷」ボタン導線をこのシートに向ける

**利点:**
- 患者画面の構造を変えない
- 印刷レイアウトと完全統合できる
- 「領収証シート = 会計サマリーシート」として一本化できる

**欠点:**
- シート追加が必要
- 患者画面から別タブへの切り替えが必要

---

### 案C: 来院ヘッダ参照式（コード変更最小）

**場所:** 患者画面シートの右端（例: J2:K8）に QUERY / INDEX 式を設定

**内容:**
```
=IFERROR(QUERY(来院ヘッダ!A:Z, "SELECT 患者ID,窓口負担額,... ORDER BY 作成日時 DESC LIMIT 1"), "---")
```

**利点:** GAS 変更なし or 最小限

**欠点:**
- 保存前から最新ヘッダを表示してしまう（誤認リスク）
- 自費明細シートの合計を同時に取得する式が複雑になる
- スプレッドシートの計算負荷が上がる

## 既存処理との切り分け方針

### clearAfterSaveUI_V3_ との関係

| 処理 | 現在の挙動 | 変更後 |
|---|---|---|
| 入力欄クリア（B2・ケース行等）| ✅ クリア | 維持 |
| 自費ブロッククリア（B7〜H8）| ✅ クリア | 維持 |
| E2〜E6（保険金額表示）| ❌ 残す（意図的）| 維持 |
| サマリーエリア（行55〜62）| 対象外（現在クリア処理あり）| **書き込む（残す）** |

→ `clearAfterSaveUI_V3_` の末尾に `writeSavedSummaryUI_V3_(uiSh, ...)` を追加するだけ。既存処理の変更は最小限。

### clearEntryUI_V3 との関係

| 処理 | 現在の挙動 | 変更後 |
|---|---|---|
| E2:E6（保険金額）| ✅ クリア | 維持 |
| サマリーエリア（行55〜62）| ✅ クリア（現在も）| 維持（次患者入力前に消える）|

→ 変更不要。clearEntryUI_V3 は行55〜62 を元々クリアしている（"B5:B7" + "E2:E6" の範囲外だが `clearEntryUI_V3` の全体動作を要確認）。

## 案A v1 実装詳細（2026-03-31）— 参考記録

### 採用スコープ（v1）

- **配置:** 患者画面シート J55:Q62（A:H には触れない）
- **前回サマリー:** 次回保存時に `writeSavedSummaryUI_V3_` が上書き（clearAfterSaveUI_V3_ は関与しない）
- **全クリア時:** `clearEntryUI_V3` が `UI.summary_area` = "J55:Q62" を `breakApart().clearContent().clearFormat()`

### 実装した変数・関数

| 変数/関数 | 内容 |
|---|---|
| `UI.summary_area = "J55:Q62"` | サマリーエリアのレンジ定数 |
| `writeSavedSummaryUI_V3_(uiSh, obj)` | 新設。保存直後に呼ばれる |
| `saveVisit_V3` 変更 | ⑥ の前に patientDisplayName / selfPayTotal 取得、clearAfterSaveUI 後に writeSavedSummary 呼び出し |
| `clearEntryUI_V3` 変更 | summary_area を breakApart + clearContent + clearFormat |
| `setupSelfPayValidation_V3_` 変更 | summary_area を初期化（「来院保存後に表示」プレースホルダ）|
