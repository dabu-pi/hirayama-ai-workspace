# JREC-01 copyInsurerToMaster_V3 — ラベル名ベース転記への変更方針

**日付:** 2026-04-22  
**対象ファイル:** `Ver3_core.js`  
**変更種別:** リファクタリング（既存運用を壊さない最小修正）

---

## 変更前の問題点

| 問題 | 詳細 |
|---|---|
| セル番地固定参照 | `insurerSh.getRange(1, 2)` で患者IDを取得（B1固定） |
| 転記ロジックが硬直 | `FIELD_MAP` に書かれた項目しか転記できない |
| 表記ゆれに非対応 | 「確認日 :」のようなラベルを正規化していない |
| 転記漏れが分からない | Logger出力が不足。スキップ理由が「配偶者」と「列なし」だけ |
| 不足フィールド | 記号・枝番・本人家族の別・被保険者氏名が転記対象外 |

---

## 変更後の設計

### 補助関数 4本

| 関数名 | 役割 |
|---|---|
| `normalizeInsurerLabel_(label)` | 末尾の ` :` `：` を除去してラベルを正規化 |
| `getMasterHeaderMap_(sheet)` | 患者マスタ1行目から `{列名: 1-based列番号}` を構築 |
| `buildInsurerRecordByLabel_(sheet, blockStartCol)` | 保険者情報シートを `{正規化ラベル: 値}` 辞書に変換。重複ラベルは先頭優先 |
| `copyInsurerFieldsToMasterRow_(masterSheet, rowIndex, headerMap, insurerRecord)` | 辞書のキーと患者マスタ列名の一致で転記。Logger出力付き |

### 転記スコープの変更

| 旧 | 新 |
|---|---|
| `FIELD_MAP` に列挙した13項目のみ | insurerRecord のキーと headerMap の列名が**完全一致するすべて**の項目 |

→ 列追加・順番変更があっても自動対応。新しいラベルが保険者情報に加わればそのまま転記される。

### 患者ID取得方法の変更

| 旧 | 新 |
|---|---|
| `insurerSh.getRange(1, 2).getValue()` （B1固定） | `insurerRecord["照会番号"]` （ラベル名で取得） |

### Logger 出力（3種）

```
[copyInsurer] 転記した項目: 氏名, フリガナ, ...
[copyInsurer] 転記元にあるが患者マスタに列なし: 記号, ...
[copyInsurer] 転記対象なのに値が空: 枝番, ...
[copyInsurer] 必須フィールドが保険者情報シートに見つからない: 本人・家族の別  ← 保険者情報側に列がない場合
```

### 必須フィールド（REQUIRED_FIELDS）17項目

氏名 / フリガナ / 生年月日 / 性別 / 区分 / 負担割合 / 保険者番号 / 保険者名 / 記号 / 番号 / 枝番 / 本人・家族の別 / 被保険者氏名 / 有効開始日 / 資格取得年月日 / 確認日 / 要配慮情報（備考）

※ REQUIRED_FIELDS は転記を制限するリストではなく「ログ確認用のチェックリスト」。  
　 実際の転記はラベル一致で行われるため、REQUIRED_FIELDS 外の列も転記される。

---

## 配偶者情報の除外ロジック

旧: `seenOnce` で 氏名・性別 の2回目以降をスキップ  
新: `buildInsurerRecordByLabel_` が重複ラベルを先頭優先で格納するため、自動的に同じ効果を得る。`seenOnce` ロジックは削除。

---

## blockStartCol パラメータの意味

`buildInsurerRecordByLabel_(sheet, blockStartCol)` の `blockStartCol` は、ラベル列の1-based列番号。

| blockStartCol | ラベル列 | 値列 |
|---|---|---|
| 1 (既定) | A | B |
| 3 | C | D |

将来、保険者情報シートに複数患者ブロックが横並びになっても対応可能。

---

## 既存運用への影響

- `copyInsurerToMaster_V3` の関数名・メニュー配置は変更なし
- 転記結果ダイアログの内容に「患者マスタに列なし」「転記対象だが値が空」が追加
- 転記13項目（旧FIELD_MAP）はすべて引き続き転記される
- 新たに 記号・枝番・本人家族の別・被保険者氏名 が転記対象に追加（患者マスタに列があれば）

---

## 次回確認手順

1. 保険者情報シートに患者IDを入れた状態で「保険者情報を患者マスタへ反映」を実行
2. 結果ダイアログで転記件数・スキップ項目を確認
3. Apps Script ログで `[copyInsurer]` 行を確認
4. 患者マスタの対象行に 氏名・フリガナ・記号・番号・枝番 等が正しく入っているか確認
5. ラベルに ` :` が付いていた場合に正規化が効いているか確認（ラベルが「確認日 :」等）
