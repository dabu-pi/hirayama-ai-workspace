# JREC-01 申請書出力修正 完了記録

- 日付: 2026-04-06
- ステータス: **完了（正式帳票で反映確認済み）**

---

## 問題の発端

申請書の出力内容に以下の不備があった。

| 項目 | 旧仕様（問題） | 新仕様（修正後） |
|---|---|---|
| 負傷名の番号 | 番号なし（「頸部 捻挫」のまま） | 1行目に（1）、2行目に（2） |
| 負傷の原因の連結 | ケース1とケース2が " / " で1行連結 | ケースごとに1行・改行区切り |
| 負傷の原因の番号 | 番号なし | 行頭に（1）（2） |
| 負傷の原因の並び順 | 場所 → 状況 → 日時（不自然） | 日時 → 場所 → 状況（自然な日本語順） |

---

## 根本原因の整理

### 原因1: 出力経路が2系統あることに気づいていなかった

| 経路 | ファイル | 出力先 |
|---|---|---|
| GAS 直反映 | `Ver3_transferData.js` → `V3TR_writeToApplication_` | スプレッドシート テンプレートシート |
| Cloud Run xlsx 生成 | `write_application.py` → `write_application()` | Excel テンプレート（Drive 保存） |

`write_application.py` のみを修正し GAS 側を修正していなかったため、
スプレッドシートの確認用テンプレに変更が反映されなかった。

### 原因2: 現物テンプレ反映関数がメニュー未登録だった

`V3TR_menuWriteApplication()`（テンプレシートへ書き込む関数）が存在していたが
メニューに登録されておらず、ユーザーが実行できる手段がなかった。

### 原因3: Cloud Run の再デプロイが必要だった

`write_application.py` を修正しても、Cloud Run は Docker イメージとして動いているため
コードを変更しただけでは自動反映されない。再ビルド → push → deploy が必要だった。

---

## 対応内容

### 1. write_application.py の修正（commit: 9af6fc4）

| 修正箇所 | 変更前 | 変更後 |
|---|---|---|
| 負傷名書き込みループ | `put(m["name"], d["name"])` | i=0→`（1）`prefix、i=1→`（2）`prefix |
| 原因の並び順 | `("負傷の場所", "負傷の状況", "負傷の日時")` | `("負傷の日時", "負傷の場所", "負傷の状況")` |
| 原因の連結 | `" / ".join(unique_texts)` | `"\n".join([f"（{i+1}）{t}" ...])` |

### 2. Ver3_transferData.js の修正（commit: 7dadd97）

write_application.py と同内容を GAS 側にも適用。

| 修正箇所 | 変更前 | 変更後 |
|---|---|---|
| 負傷名書き込み | `put(m.name, d.name)` | i===0→`（1）`prefix、i===1→`（2）`prefix |
| 原因の並び順 | place → status → dt | dt → place → status |
| 原因の連結 | `d4Parts.join(" / ")` | `d4Lines.join("\n")` with `（N）`prefix |

clasp push 済み。

### 3. メニュー整理（commit: 2f80975, bdaae16）

| 変更 | 内容 |
|---|---|
| メニュー追加 | 「申請書テンプレに反映（画面確認用）」→ `V3TR_menuWriteApplication` |
| ラベル修正 | 「申請書転記データ作成」→「申請書転記データを作成」 |

### 4. Cloud Run 再デプロイ

`write_application.py` の修正を正式帳票経路（Cloud Run xlsx 出力）に反映するため
Docker イメージを再ビルド → Artifact Registry に push → Cloud Run デプロイ。

APPGEN_ENDPOINT をスクリプトプロパティに再設定して動作確認済み。

## 申請書系メニュー 運用方針（確定）

| メニュー | 役割 | 正本か |
|---|---|---|
| 申請書を生成して Drive に保存 | **正式帳票出力**（Cloud Run → Excel テンプレ → xlsx → Drive） | ✅ 正本 |
| 申請書テンプレに反映（画面確認用） | 確認・デバッグ用（スプレッドシートへ直書き） | ❌ 正本ではない |
| 申請書転記データを作成 | 中間データ作成（「申請書_転記データ」シートのみ更新） | ─ |

**正式帳票の正本 = Cloud Run 経路の xlsx（`application_template.xlsx` / シート「新　様式第5号」）**

スプレッドシート直反映は列幅・結合・印刷レイアウト崩れのリスクがあるため、
確認・デバッグ用に限定し、正式帳票には使わない。
