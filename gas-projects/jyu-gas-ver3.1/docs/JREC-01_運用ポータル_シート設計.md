## 1. シートの目的

接骨院プレオープン運用における**総合入口（ポータル）シート**。

3つの目的を1シートで同時に満たす。

| 目的 | 実現手段 |
|---|---|
| **総合入口** | 手順書・テンプレ・入力画面へのリンクを一覧化 |
| **手順順表示** | 1行1工程・手順順に並べ、初めての人が何から始めるか分かる |
| **今どの段階か見える** | `実務進行段階`列で患者対応中の現在地を確認できる |

### 解決したい課題

| 課題 | 現状 | このシートで解決 |
|---|---|---|
| 手順書がバラバラ | docs/配下に8ファイル以上存在 | 1シートに集約してリンク一覧化 |
| 今どの段階か分からない | 来院中の患者がどの工程まで進んでいるか見えない | `実務進行段階`列で受付中/施術者待ち等を確認 |
| 整備状況が混在 | 未整備の工程と完了工程が区別されない | `整備状況`列で工程の準備度を管理 |
| 初めての人が迷う | スプレッドシートを開いても何から始めるか不明 | 手順順に1行1工程で並べる |
| 必須項目が分からない | 全工程が横並びで落としてはいけない工程が見えない | `必須区分`列で「必須/条件付き/任意」を識別 |
| 将来の拡張で崩れる | 追記のたびに構造が変わる | 結合セルなし・行追加のみで拡張可能な構造 |

### 対象ユーザー

- 受付担当者（どの工程か確認 + 手順書リンクへアクセス）
- 施術者（テンプレ・入力画面へのアクセス + 実務進行段階の確認）
- 院長（整備状況・プレオープン前提チェック・進捗管理）

## 2. 列設計

### 列一覧（A〜M列、13列固定）

| 列 | 列名 | 型 | 内容 | 例 |
|---|---|---|---|---|
| A | 管理ID | テキスト | 工程の一意識別子。プレフィックス+連番 | `OP-001` |
| B | カテゴリ | 選択肢 | 工程の大分類 | `新患` / `再診` / `会計` / `月次` / `設定` / `将来対応` |
| C | フェーズ | 選択肢 | カテゴリ内の段階 | `受付` / `問診` / `評価` / `カルテ入力` / `施術` / `会計` / `確認` |
| D | 工程名 | テキスト | この行が表す操作・確認の名称 | `新患問診票 配布・回収` |
| E | 担当 | 選択肢 | 実施者 | `受付` / `施術者` / `院長` / `システム` / `院長+施術者` |
| F | 関連ファイル名 | テキスト | 参照する docs ファイル名またはシート名 | `JREC-01_新患問診票テンプレ.md` |
| G | リンク/場所 | テキスト | URL または「シート名:セル番地」形式 | `docs/JREC-01_受付者向け手順書.md` |
| H | **整備状況** | 選択肢 | **この工程の準備・文書化の完了度**（患者対応状況ではない）| `✅完了` / `⚠️要確認` / `❌未整備` / `🚧準備中` / `⏭️将来対応` |
| I | **実務進行段階** | 選択肢 | **この工程が対応する患者対応フェーズ**（来院中の現在地）| `未着手` / `受付中` / `施術者待ち` / `入力中` / `保存済` / `会計待ち` / `完了` |
| J | プレオープン対象 | チェック | プレオープン中に使う工程か | `TRUE` / `FALSE` |
| K | **必須区分** | 選択肢 | **必須/条件付き/任意 の3択**（プレオープン安全確認用）| `必須` / `条件付き` / `任意` |
| L | タグ | テキスト | 複数タグをカンマ区切りで付与 | `保険,初検,新患` |
| M | 備考 | テキスト | 運用メモ・警告・未解決事項 | `保険証コピー必須` |

### 列名の使い分け（H列とI列の違い）

| 列 | 何を表すか | 誰が更新するか | 更新頻度 |
|---|---|---|---|
| H: 整備状況 | 手順書・テンプレ・システム機能の**準備完了度** | 院長 / Claude | 整備作業のたびに更新 |
| I: 実務進行段階 | この工程が実施されるときの**患者対応の現在地** | （固定値・運用定義）| 工程の定義変更時のみ |

> **I列は各来院の実況更新欄ではない。** 各工程が「患者来院フローのどの段階に属するか」を固定的に定義する列。
> 受付者・施術者が「患者は今どこにいる？」を把握するときに I列でフィルタして確認する。

### 列設計の方針

- **A列（管理ID）** は変更しない。ソート・フィルタ後も同じ工程を一意に特定。
- **G列（リンク/場所）** は HYPERLINK 関数で実リンクを張る。GitHub URL か「患者画面!B2」形式。
- **H列（整備状況）** は入力規則（ドロップダウン）で統制。自由記述不可。
- **I列（実務進行段階）** も入力規則で統制。来院フローの7段階から選択。
- **J列（プレオープン対象）** はチェックボックス。`TRUE` でフィルタすると今使う工程のみ表示。
- **K列（必須区分）** は入力規則で統制。プレオープン中に `必須` だけフィルタすると落としてはいけない工程一覧になる。
- 結合セルは使わない。フェーズ・カテゴリが同じ行が連続してもそれぞれ独立した行にする。

### 「表示順」列を追加しない理由

管理IDの連番体系（`OP-001〜OP-009` が設定、`OP-010〜OP-029` が新患フロー、...）が表示順を兼ねている。
別途「表示順」列を持つと、管理IDと表示順の二重管理になり「どちらが正か」が曖昧になる。
管理IDで昇順ソートすれば手順順になる設計で十分。

## 4. 運用ルール

### 基本ルール

| ルール | 内容 |
|---|---|
| 行を削除しない | 廃止した工程は整備状況を `❌未整備` にして残す（履歴のため）|
| 行追加のみで拡張 | 既存行の変更は最小限。新工程は末尾か連番の空きに追加 |
| 管理ID は変更しない | ソート・フィルタ後も同じ工程を一意に特定するための固定ID |
| 整備状況は入力規則で統制 | 5択のみ。自由記述不可 |
| 実務進行段階は入力規則で統制 | 7択のみ。自由記述不可 |
| 必須区分は入力規則で統制 | 3択のみ。自由記述不可 |
| リンクは HYPERLINK 関数 | `=HYPERLINK("URL","ファイル名")` 形式で押せるリンクにする |
| プレオープン対象チェック | `J列=TRUE` でフィルタすると「今使う工程」だけ絞り込める |

### 整備状況（H列）の定義

| 値 | 意味 |
|---|---|
| ✅完了 | 手順書・テンプレ・システム機能の準備が完了。運用可能 |
| ⚠️要確認 | 整備はしたが院長確認・実機確認が未完了 |
| ❌未整備 | 設計はあるが実装・文書化がされていない |
| 🚧準備中 | 現在作業中 |
| ⏭️将来対応 | 設計済みだがプレオープン対象外（電療解禁後など）|

### 実務進行段階（I列）の定義

| 値 | 意味 |
|---|---|
| 未着手 | 患者来院前の準備工程（設定・月次等）|
| 受付中 | 患者が受付カウンターにいる段階 |
| 施術者待ち | 受付完了・施術者が問診中または評価中の段階 |
| 入力中 | 施術者が患者画面へ入力中の段階 |
| 保存済 | 保存完了・院長確認待ち or 会計待ちの段階 |
| 会計待ち | 施術・確認完了・受付が会計処理中の段階 |
| 完了 | その来院の対応が完了した段階 |

### 必須区分（K列）の定義

| 値 | 意味 | フィルタ時の用途 |
|---|---|---|
| 必須 | 来院ごとに必ず実施。省略不可 | プレオープン安全チェックリスト |
| 条件付き | 特定条件のみ実施（保険ありの場合、新患の場合 等）| 条件一致時の確認 |
| 任意 | 実施推奨だが省略可能 | 余裕があるときの品質向上 |

### 管理IDの体系（表示順を兼ねる）

| 範囲 | カテゴリ |
|---|---|
| OP-001〜OP-009 | 設定（プレオープン前作業）|
| OP-010〜OP-029 | 新患フロー |
| OP-030〜OP-039 | 会計 |
| OP-040〜OP-049 | 再診フロー |
| OP-050〜OP-059 | 月次確認・申請 |
| OP-060〜OP-079 | 予備（将来の保険・自費・混在の詳細フロー用）|
| OP-090〜OP-099 | 将来対応（電療・システム拡張）|

> 管理IDの昇順ソート = 手順順表示。別途「表示順」列は持たない。

## 5. プレオープンでの使い方

### フィルタ組み合わせ例

| やりたいこと | フィルタ設定 |
|---|---|
| 今日使う工程だけ見たい | J列(プレオープン対象) = TRUE |
| 絶対に落としてはいけない工程だけ | K列(必須区分) = 必須 |
| 受付中の工程だけ確認 | I列(実務進行段階) = 受付中 |
| 新患フローだけ | B列(カテゴリ) = 新患 |
| まだ整備できていない工程を洗い出す | H列(整備状況) = ⚠️要確認 または ❌未整備 |
| 入力中のフェーズで施術者がやること | I列 = 入力中 かつ E列(担当) = 施術者 |

### プレオープン開始前の院長チェック手順

1. H列(整備状況) でフィルタ → `⚠️要確認` と `❌未整備` がないことを確認
2. J列(プレオープン対象) = TRUE かつ K列(必須区分) = 必須 でフィルタ → 全行が `✅完了` であることを確認
3. 特に `OP-005`（保険者情報シート整備）が `⚠️要確認` のままなら月次申請前に完了させる

## 6. Dashboard / 関連文書 反映の判断

### 反映が必要なもの（実施済み）

| 対象 | 内容 | 状態 |
|---|---|---|
| `JREC-01_プレオープン運用手順書.md` | §2 シート一覧に `運用ポータル` を追記 | 実施済み（2026-03-30）|
| `PROJECT_STATUS.md` | 設計書追加と院長TO-DO(5)を記録 | 実施済み（2026-03-30）|

### 反映が不要なもの（理由付き）

| 対象 | 不要な理由 |
|---|---|
| Dashboard Projects シート（GAS同期）| 下記詳細を参照 |
| Task_Queue | 運用ポータルの「シート作成・行追加」は院長が手動実施する作業。GASが自動化すべきタスクではない |
| Run_Log | `git commit / push` 完了時点で GitHub が正本として機能。ChatGPT は GitHub から再開判断できる |

**Dashboard Projects シートへの同期が不要な理由（運用ルールに照らした判断）:**

`de -ProjectId JREC-01` コマンドが Projects シートに書き込む内容は「次アクション / 最終更新日 / 補足」の3列。
これは JREC-01 の**保険請求システム機能（来院登録・算定・申請書生成）の実装進捗**を管理するために使う。

今回の変更は以下の2種類：
1. 運用ポータルの設計書（Markdown ドキュメント）
2. ポータルシートのセットアップスクリプト（運用補助ツール）

どちらも「保険請求システム機能の実装」ではない。JREC-01 の「次アクション」は変わらず
**「プレオープン開始（院長が手順書・ポータルを整備後）」**であり、Dashboard に同期すべき状態変化がない。

commit/push で GitHub が更新されているため、ChatGPT はブランチから最新状態を確認できる。
これが CLAUDE.md に定めた「GitHub を single source of truth として扱う」の実践であり、
Dashboard 同期は状態変化が生じたとき（例: 機能が実装完了した、次アクションが変わった）に行う。

### シート実装（GAS）の判断

**現時点: 設計書のみとし、院長が Google Sheets 上でシートを手動作成する。**

| 判断軸 | 結論 |
|---|---|
| 初回のみ作業 | シート構造は一度作れば維持される。GAS化の優先度は低 |
| 行数 | 約28行。手動入力可能な量 |
| GAS化する条件 | 院長から「GASで自動作成してほしい」と依頼があった場合 |

## 7. GAS 実装方針・実行方法・再実行注意

### 実装状況

| 項目 | 内容 |
|---|---|
| コードファイル | `gas-projects/jrec-portal/Code.gs` |
| 実装日 | 2026-03-30（v1.1: onEdit 追加）|
| 対象 SS ID | `121BkW7jEnKUjmU_NNVAPyJRs_UVmmoqkHDPMHL-RJeA` |
| 主要関数 | `setupOperationPortal_()` / `onEdit(e)` / `refreshFormatOnly_()` / `resetDataRows_()` |

### なぜ `jrec-portal/` を独立ディレクトリにするか

`jyu-gas-ver3.1/` 配下の `.clasp.json` には来店管理施術録ver3.1 の `scriptId` が設定されている。
clasp は **1プロジェクト = 1 GAS スクリプト = 1 SS** の構造を前提とする。

運用ポータルは **別の SS**（ID: `121BkW7jEnKUjmU...`）に対するスクリプトであるため、
同じ `jyu-gas-ver3.1/` 配下に置くと `clasp push` 時に来店管理施術録ver3.1 へ誤デプロイされる。

これは好みの問題ではなく **技術的な制約**。`jrec-portal/` を独立させる以外の選択肢はない。

> 将来、運用ポータルを来店管理施術録ver3.1 と **同一 SS** に統合する場合は、
> `jyu-gas-ver3.1/` 配下に移設してよい。その場合は `clasp.json` の `scriptId` が共通になる。

### 実行方法（初回）

1. [対象スプレッドシート](https://docs.google.com/spreadsheets/d/121BkW7jEnKUjmU_NNVAPyJRs_UVmmoqkHDPMHL-RJeA) を開く
2. メニュー「拡張機能」>「Apps Script」をクリック
3. `gas-projects/jrec-portal/Code.gs` の全文をエディタに貼り付けて保存（Ctrl+S）
4. `appsscript.json` の内容も同名ファイルに貼り付けて保存
5. 実行関数を「`setupOperationPortal_`」に設定して「実行」ボタンをクリック
6. 初回は権限承認ダイアログが出る → スプレッドシートへのアクセスを許可

> **container-bound 推奨:** 対象 SS のシートを開いたまま「拡張機能 > Apps Script」から追加すると、onOpen() が有効になりカスタムメニューが使えるようになる。

### 実行後に確認すること

| 確認項目 | 内容 |
|---|---|
| ヘッダー行 | 13列が正しく設定されているか |
| H/I/K列 | ドロップダウンが出るか（クリックして確認）|
| J列 | チェックボックスが出るか |
| 行背景色 | ✅完了 = 薄緑 / ⚠️要確認 = 薄黄 になっているか |
| フィルター | 1行目にフィルターアイコンが出ているか |
| G列リンク | 現状はテキストのみ。HYPERLINK 関数への変換は手動で行う |

### 再実行時の動作

| 状況 | 動作 |
|---|---|
| シートが存在しない | 新規作成 → 書式設定 → 初期データ（11行）挿入 |
| シートが存在する | 書式・ヘッダー・入力規則のみ更新。**データ行は一切変更しない** |
| データもリセットしたい | カスタムメニュー「データ行をリセット（要注意）」から実行（確認ダイアログあり）|
| フィルターが壊れた | カスタムメニュー「フィルターをリセット」から実行 |

### H列変更時の背景色自動反映について

| 状況 | 背景色の更新 |
|---|---|
| `setupOperationPortal_()` を実行した | ✅ 全行を再計算して反映 |
| `refreshFormatOnly_()` を実行した | ✅ 全行を再計算して反映 |
| H列をドロップダウンで手動変更した | ✅ **自動反映される**（`onEdit(e)` が変更行だけを更新）|
| H列をコピペで複数行変更した | ✅ 変更範囲が全て更新される（`onEdit(e)` は複数行対応）|
| スタンドアロン GAS として実行した場合 | ⚠️ `onEdit` はトリガー手動登録が必要。自動反映されない |

> `onEdit(e)` は **container-bound**（対象SSの「拡張機能 > Apps Script」に直接貼り付け）の場合のみ自動動作する。
> スタンドアロンの場合は GAS エディタの「トリガー」> `onEdit` をスプレッドシートの「編集時」に登録することで有効になる。

### 再実行時の注意

- `setupOperationPortal_()` は何度実行しても**既存データを上書きしない**（シートが存在する場合）
- `resetDataRows_()` は**既存データをすべて消す**。実行前に必ずバックアップを取ること
- G列のリンクを HYPERLINK 関数に変換した後に再実行してもフォーミュラは保持される（`clearContent` を呼ばないため）

## 8. シート反映案（GASセットアップ関数の概要）

> 全文コードは `gas-projects/jrec-portal/Code.gs` を参照。
> 以下はイメージ確認用の抜粋。

```javascript
/**
 * 運用ポータルシートを初期セットアップする（初回のみ実行）
 * 実行方法: GASエディタから手動実行 or メニュー「柔整ツール > 運用ポータル初期化」
 */
function setupOperationPortal_() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheetName = "運用ポータル";

  if (ss.getSheetByName(sheetName)) {
    SpreadsheetApp.getUi().alert(sheetName + " はすでに存在します。初期化をスキップします。");
    return;
  }

  var sh = ss.insertSheet(sheetName);

  // ヘッダ行（13列 A〜M）
  var headers = [
    "管理ID", "カテゴリ", "フェーズ", "工程名", "担当",
    "関連ファイル名", "リンク/場所",
    "整備状況", "実務進行段階", "プレオープン対象", "必須区分",
    "タグ", "備考"
  ];
  sh.getRange(1, 1, 1, headers.length).setValues([headers]);

  // ヘッダ書式
  sh.getRange(1, 1, 1, headers.length)
    .setBackground("#263238")
    .setFontColor("#FFFFFF")
    .setFontWeight("bold");

  // 列幅調整
  sh.setColumnWidth(1, 80);   // A: 管理ID
  sh.setColumnWidth(2, 90);   // B: カテゴリ
  sh.setColumnWidth(3, 90);   // C: フェーズ
  sh.setColumnWidth(4, 220);  // D: 工程名
  sh.setColumnWidth(5, 100);  // E: 担当
  sh.setColumnWidth(6, 220);  // F: 関連ファイル名
  sh.setColumnWidth(7, 200);  // G: リンク/場所
  sh.setColumnWidth(8, 110);  // H: 整備状況
  sh.setColumnWidth(9, 120);  // I: 実務進行段階
  sh.setColumnWidth(10, 130); // J: プレオープン対象
  sh.setColumnWidth(11, 100); // K: 必須区分
  sh.setColumnWidth(12, 180); // L: タグ
  sh.setColumnWidth(13, 220); // M: 備考

  // H列（整備状況）入力規則
  var statusRule = SpreadsheetApp.newDataValidation()
    .requireValueInList(["✅完了", "⚠️要確認", "❌未整備", "🚧準備中", "⏭️将来対応"], true)
    .setAllowInvalid(false).build();
  sh.getRange("H2:H200").setDataValidation(statusRule);

  // I列（実務進行段階）入力規則
  var stageRule = SpreadsheetApp.newDataValidation()
    .requireValueInList(["未着手", "受付中", "施術者待ち", "入力中", "保存済", "会計待ち", "完了"], true)
    .setAllowInvalid(false).build();
  sh.getRange("I2:I200").setDataValidation(stageRule);

  // J列（プレオープン対象）チェックボックス
  sh.getRange("J2:J200").insertCheckboxes();

  // K列（必須区分）入力規則
  var priorityRule = SpreadsheetApp.newDataValidation()
    .requireValueInList(["必須", "条件付き", "任意"], true)
    .setAllowInvalid(false).build();
  sh.getRange("K2:K200").setDataValidation(priorityRule);

  // 先頭行を固定
  sh.setFrozenRows(1);

  SpreadsheetApp.getUi().alert(sheetName + " を初期セットアップしました（13列）。\nデータを手動で入力してください。");
}
```
