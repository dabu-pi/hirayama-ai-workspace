# JREC-01 運用ポータル シート設計書

作成日: 2026-03-30
対象シート名: `運用ポータル`（来店管理施術録ver3.1 内に新規追加）
ステータス: 設計完了・未実装

---

## 1. シートの目的

接骨院プレオープン運用における**総合入口（ポータル）シート**。

### 解決したい課題

| 課題 | 現状 | このシートで解決 |
|---|---|---|
| 手順書がバラバラ | docs/配下に7ファイル以上存在 | 1シートに集約してリンク一覧化 |
| 今どの段階か分からない | 状態を記録する場所がない | 状態列で「完了/要確認/未整備」を可視化 |
| 初めての人が迷う | スプレッドシートを開いても何から始めるか不明 | 手順順に1行1工程で並べる |
| 将来の拡張で崩れる | 追記のたびに構造が変わる | 結合セルなし・行追加のみで拡張可能な構造 |

### 対象ユーザー

- 受付担当者（手順書と状態確認が主な用途）
- 施術者（入力画面と関連テンプレへのアクセス）
- 院長（状態確認・プレオープン進捗管理）

## 2. 列設計

### 列一覧（A〜K列、11列固定）

| 列 | 列名 | 型 | 内容 | 例 |
|---|---|---|---|---|
| A | 管理ID | テキスト | 工程の一意識別子。プレフィックス+連番 | `OP-001` |
| B | カテゴリ | 選択肢 | 工程の大分類 | `新患` / `再診` / `会計` / `月次` / `設定` / `将来対応` |
| C | フェーズ | 選択肢 | カテゴリ内の段階 | `受付` / `問診` / `評価` / `カルテ入力` / `施術` / `会計` / `確認` |
| D | 工程名 | テキスト | この行が表す操作・確認の名称 | `新患問診票 配布・回収` |
| E | 担当 | 選択肢 | 実施者 | `受付` / `施術者` / `院長` / `システム` / `院長+施術者` |
| F | 関連ファイル名 | テキスト | 参照する docs ファイル名またはシート名 | `JREC-01_新患問診票テンプレ.md` |
| G | リンク/場所 | テキスト | URL または「シート名:セル番地」形式 | `docs/JREC-01_受付者向け手順書.md` |
| H | 状態 | 選択肢 | 現在の整備・実施状況 | `✅完了` / `⚠️要確認` / `❌未整備` / `🚧準備中` / `⏭️将来対応` |
| I | プレオープン対象 | チェック | プレオープン中に使う工程か | `TRUE` / `FALSE` |
| J | タグ | テキスト | 複数タグをカンマ区切りで付与 | `保険,初検,新患` |
| K | 備考 | テキスト | 運用メモ・警告・未解決事項 | `保険証コピー必須` |

### 列設計の方針

- **A列（管理ID）** は変更しない。行を移動しても ID は固定。ソート・検索の基準。
- **G列（リンク/場所）** はスプレッドシートの HYPERLINK 関数で実リンクを張る。GitHub URL か「患者画面!B2」形式のシート参照。
- **H列（状態）** は入力規則（ドロップダウン）で統制し、自由記述不可。
- **I列（プレオープン対象）** はチェックボックス。フィルタで `TRUE` だけ絞れば今使う工程のみ表示できる。
- **J列（タグ）** は `保険`, `自費`, `混在`, `初検`, `後療`, `新患`, `再診`, `月次`, `電療` など。将来フィルタで活用。
- 結合セルは使わない。フェーズ・カテゴリが同じ行が連続してもそれぞれ独立した行にする。

## 4. 運用ルール

### 基本ルール

| ルール | 内容 |
|---|---|
| 行を削除しない | 廃止した工程は状態を `❌未整備` にして残す（履歴のため）|
| 行追加のみで拡張 | 既存行の変更は最小限。新工程は末尾か連番の空きに追加 |
| 管理ID は変更しない | ソート・フィルタ後も同じ工程を一意に特定するための固定ID |
| 状態は入力規則で統制 | `✅完了` / `⚠️要確認` / `❌未整備` / `🚧準備中` / `⏭️将来対応` の5択のみ |
| リンクは HYPERLINK 関数 | `=HYPERLINK("URL","ファイル名")` 形式で押せるリンクにする |
| プレオープン対象チェック | `I列=TRUE` でフィルタすると「今使う工程」だけ絞り込める |

### 状態の定義

| 状態 | 意味 |
|---|---|
| ✅完了 | 準備・整備が完了。運用可能な状態 |
| ⚠️要確認 | 整備はしたが院長確認・実機確認が未完了 |
| ❌未整備 | 設計はあるが実装・文書化がされていない |
| 🚧準備中 | 現在作業中 |
| ⏭️将来対応 | 設計済みだがプレオープン対象外（電療解禁後など）|

### 管理IDの体系

| 範囲 | カテゴリ |
|---|---|
| OP-001〜OP-009 | 設定（プレオープン前作業）|
| OP-010〜OP-029 | 新患フロー |
| OP-030〜OP-039 | 会計 |
| OP-040〜OP-049 | 再診フロー |
| OP-050〜OP-059 | 月次確認・申請 |
| OP-060〜OP-079 | 予備（将来の保険・自費・混在の詳細フロー用）|
| OP-090〜OP-099 | 将来対応（電療・システム拡張）|

## 5. Dashboard / 関連文書 反映の判断

### 反映が必要なもの

| 対象 | 内容 | 優先度 |
|---|---|---|
| `JREC-01_プレオープン運用手順書.md` | `§2 シート一覧` に `運用ポータル` を追記 | 高（本設計書作成時に実施）|
| `PROJECT_STATUS.md` | プレオープン体制セクションに「運用ポータルシート 設計完了・未実装」を追記 | 中（次のコミット時）|

### 反映が不要なもの（理由付き）

| 対象 | 不要な理由 |
|---|---|
| Dashboard Projects シート | docs 設計のみ・コード変更なし。`de -ProjectId JREC-01` は来院ヘッダ等の機能実装完了時に使う |
| Task_Queue | 「シート作成」は院長が手動で実施する作業。GAS実装タスクが発生した場合に追加する |
| Run_Log | commit 時に自動記録される。手動追加不要 |

### シート実装（GAS）の判断

設計書作成 → **院長が Google Sheets 上でシートを手動作成する**のが最速。
GAS 自動セットアップが必要な場合は以下の観点で判断する:

| 判断軸 | 内容 |
|---|---|
| 初回のみ作業 | シート構造は一度作れば維持される。GAS化の優先度は低 |
| 行数が多い | 初期データは約25行。手動入力可能な量 |
| リンクが多い | HYPERLINK 関数は手動でも設定可能 |
| GAS化する場合 | `setupOperationPortal_()` 関数として `Ver3_core.js` に追加する方式が妥当 |

## 6. シート反映案（GAS セットアップ関数）

院長から「GASで自動作成してほしい」と依頼があった場合に実装する。
現時点では**設計書のみ**とし、GAS実装は次のフェーズへ。

### 実装時のイメージ

```javascript
/**
 * 運用ポータルシートを初期セットアップする（初回のみ実行）
 * 実行方法: GASエディタから手動実行 or メニュー「柔整ツール > 運用ポータル初期化」
 */
function setupOperationPortal_() {
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var sheetName = "運用ポータル";

  // すでにシートがある場合はスキップ
  if (ss.getSheetByName(sheetName)) {
    SpreadsheetApp.getUi().alert(sheetName + " はすでに存在します。初期化をスキップします。");
    return;
  }

  var sh = ss.insertSheet(sheetName);

  // ヘッダ行
  var headers = [
    "管理ID", "カテゴリ", "フェーズ", "工程名", "担当",
    "関連ファイル名", "リンク/場所", "状態", "プレオープン対象", "タグ", "備考"
  ];
  sh.getRange(1, 1, 1, headers.length).setValues([headers]);

  // ヘッダ書式
  sh.getRange(1, 1, 1, headers.length)
    .setBackground("#263238")
    .setFontColor("#FFFFFF")
    .setFontWeight("bold");

  // 列幅調整
  sh.setColumnWidth(1, 80);   // A: 管理ID
  sh.setColumnWidth(2, 90);   // B: カテゴリ
  sh.setColumnWidth(3, 90);   // C: フェーズ
  sh.setColumnWidth(4, 220);  // D: 工程名
  sh.setColumnWidth(5, 100);  // E: 担当
  sh.setColumnWidth(6, 220);  // F: 関連ファイル名
  sh.setColumnWidth(7, 200);  // G: リンク/場所
  sh.setColumnWidth(8, 110);  // H: 状態
  sh.setColumnWidth(9, 130);  // I: プレオープン対象
  sh.setColumnWidth(10, 180); // J: タグ
  sh.setColumnWidth(11, 220); // K: 備考

  // H列（状態）に入力規則
  var statusRule = SpreadsheetApp.newDataValidation()
    .requireValueInList(["✅完了", "⚠️要確認", "❌未整備", "🚧準備中", "⏭️将来対応"], true)
    .setAllowInvalid(false)
    .build();
  sh.getRange("H2:H200").setDataValidation(statusRule);

  // I列（プレオープン対象）チェックボックス
  sh.getRange("I2:I200").insertCheckboxes();

  // 先頭行を固定
  sh.setFrozenRows(1);

  SpreadsheetApp.getUi().alert(sheetName + " を初期セットアップしました。\nデータを手動で入力してください。");
```
