# 柔整 Ver3 金額計算 仕様書（SPEC.md）

最終更新: 2026-03-17
対象: Ver3_amounts（来院ヘッダ金額計算・安全制御）

---

## 0. 業務背景と目的

### 業務背景

本システムは **接骨院（柔道整復師）の患者管理・保険請求業務** を支援する。

1. **来院受付** — 患者の来院を記録し、施術内容・部位・傷病名を登録する
2. **施術記録** — 毎回の施術内容、所見、経過を記録する
3. **金額算定** — 保険算定ルールに基づき、初検料・後療料・加算等を自動計算する
4. **窓口会計** — 患者負担額（窓口負担）を算出し、徴収する
5. **療養費支給申請書の作成** — 施術内容・金額を集計し、公的機関（健康保険組合・市区町村等）へ提出する正式書類を生成する
6. **保険請求** — 申請書を提出し、保険者から療養費の支払いを受ける

### 重要事項

- **療養費支給申請書は公的機関に提出する正式書類であり、記載内容に誤りがあってはならない**
- 金額の誤算定は返戻（差し戻し）・査定（減額）の原因となり、経営に直接影響する
- 不正請求は法的処分の対象となるため、算定ルールの厳密な遵守が必須である
- 本システムの金額計算ロジックは上記を前提に設計されている

### 設計方針

- レセプト事故ゼロ（返戻・査定を最小化）
- UI入力を尊重しつつ、算定不可は自動制御
- 抑制・算定不可・減額の理由を必ず残す（監査耐性）
- 疑義がある場合は要確認フラグで人の判断を促す（自動却下しない）

## 1. 前提（データモデル）

### キー体系

| キー | 形式 | 単位 | 例 |
|------|------|------|-----|
| visitKey | `患者ID_yyyy-MM-dd` | 1来院日 | `hirayamaka_2026-01-01` |
| caseKey | `患者ID_エピソード開始日_C{caseNo}` | 1受傷エピソード | `hirayamaka_2026-01-01_C1` |

- **visitKey** = 患者×来院日の一意キー（来院ヘッダの主キー）
- **caseKey** = 1つの受傷エピソード（初検日で固定、エピソード中は不変）
- **来院ケース行の一意キー** = `visitKey + caseNo` の複合キー
- **別ケース判定**: UIで caseKey を新規作成した場合のみ別ケース
- 同一エピソード（初検→再検→後療…）は同一 caseKey を共有する
- Ver3は原則最大2部位（将来拡張で3部位以上を想定）

### 用語定義

| 用語 | 定義 |
|------|------|
| 初検日(算定) | 初検料が実際に算定された来院日 |
| 初検日(UI区分) | ユーザーがUI上で選択した「初検」区分の日 |
| 初検日 | 文脈により上記いずれかを指す。算定ロジック内では「初検日(算定)」を意味する |

---

## 2. 判定優先順位（絶対順序）

1) 30日判定  
2) 受傷日経過日数による加算可否判定  
3) 区分確定（初検/再検/後療）  
4) 患者×月 上限制御
   ※ 区分確定(3)の結果を受けて上限制御(4)を判定する
5) 要確認フラグ付与  
6) 多部位逓減  
7) 長期減額  
8) 来院合計算出  

---

## 3. 30日ルール

- 前回来院から **30日以内（≤30日）** → 継続（後療）
- 前回来院から **31日以上（>30日）** → 初検リセット
- 初検リセット時は長期減額カウントもリセット

---

## 4. 患者×月 上限制御

同一患者・同一月において最大1回：

- 初検料
- 相談支援料

別ケースでも合算で1回。

> 再検料は月上限ではなく「初検後最初の後療日1回」で管理する（§6参照）。

---

## 4-1. 混在来院日の課金優先順位

同一 visitKey に case1・case2 が存在し、区分が混在する場合の visit ヘッダ課金。

| hasBillableInitial | hasReexam | 算定 |
|---|---|---|
| true | - | 初検料 + 相談支援料（再検料は立てない）|
| false | true | 再検料（初検料は立てない）|
| false | false | 0（後療のみ）|

- **hasBillableInitial** = 初検区分かつ月内未算定（`initFee > 0`）
- **hasReexam** = いずれかのケースが `kubun === "再検"`（caseKey単位で `priorCount === 1`）
- 優先順位: 初検 > 再検 > 後療（visit単位で排他）
- 「初検区分が存在するだけ」で初検優先にしない（抑制された初検は `hasBillableInitial = false`）

### 代表ケース

| case1 | case2 | 結果 |
|---|---|---|
| 再検 | 初検（抑制あり）| 再検料410 / 初検料0 |
| 再検 | 初検（抑制なし）| 初検料1550 / 再検料0 |
| 後療 | 初検（抑制あり）| 初検料0 / 再検料0 |
| 初検 | 初検 | 初検料1回のみ / 両ケース施療料 |

## 5. 初検料

### 算定可能
- 当月に未算定
- かつ
  - caseKey新規
  - または30日超リセット

### 抑制
- 当月に既算定 → 初検料=0
- 区分は「後療」
- 要確認=TRUE

---

## 6. 再検料

### 算定可能（初検後・最初の後療日のみ）

以下をすべて満たす場合：

- 同一 caseKey の初検料算定日より後の来院であること
- 同一 caseKey で再検料が未算定であること
- 当該来院日が「同一 caseKey の初検後、最初の後療日」であること

#### 最初の後療日の定義
同一 caseKey で初検料を算定した日より後の、最初の来院日を指す。

#### ケース横断の扱い
再検料は caseKey 単位で管理する（case1・case2 は独立して判定）。
患者×月での通算管理は行わない。

### 抑制
- 初検後2回目以降の後療日 → 再検料=0
- 要確認理由: 「初検後2回目以降の後療のため再検料不可」を記録
- 同一 caseKey で再検料算定済み → 再検料=0

---

## 7. 後療料

### 定義
初検日以外の継続施術日に算定する基本施術料。

区分が「後療」または「再検」の日に算定対象。

### 算定条件
- 30日以内の継続来院日
- 同月別ケース抑制初回日（区分=後療）
- 再検料算定日（再検＋後療）

### 算定不可
- 初検料算定日
- 施術実態がない日

### 部位単位
部位×傷病単位で算定。逓減対象。

---

## 8. 相談支援料

- 初検料を算定する初検日にのみ算定可
- 当月1回のみ

---

## 9. 加算（冷罨法・温罨法・電療）

### 共通
- 部位×傷病単位で判定
- 算定不可日は金額0円
- チェックは消さない

---

### 9.1 冷罨法

受傷日からの経過日数（dayDiff = 施術日 − 受傷日）で判定する。

| 傷病種別 | 算定可能条件 | dayDiff |
|----------|-------------|---------|
| 打撲・捻挫・挫傷 | 受傷日当日または翌日 | ≤ 1 |
| 脱臼 | 受傷日から5日以内 | ≤ 4 |
| 骨折・不全骨折 | 受傷日から7日以内 | ≤ 6 |

※ 初検日に限らず、上記日数内であれば後療日でも算定可。

---

### 9.2 温罨法・電療

受傷日からの経過日数（dayDiff = 施術日 − 受傷日）で判定する。

| 傷病種別 | 算定不可期間 | 算定可能条件 | dayDiff |
|----------|-------------|-------------|---------|
| 打撲・捻挫・挫傷・脱臼 | 受傷後5日間 | 6日目以降 | ≥ 5 |
| 骨折・不全骨折 | 受傷後7日間 | 8日目以降 | ≥ 7 |

#### 初検日特例（厚労省通知 P43）
- 骨折・不全骨折の受傷日から8日以上経過していても、**整復・固定を行った初検日には温罨法を算定不可**
- 脱臼・打撲・捻挫の受傷日から6日以上経過していても、**整復・施療を行った初検日には温罨法を算定不可**
- ただし、初検日より**後療のみ**を行う場合は算定可

> **実装済み（2026-03-17）**: `calcOnePartAmount_V3_` で `kubun === "初検"` の場合に温罨法=0・要確認理由を付与する初検日特例を実装。TC17a/b PASS 確認済み。

再検/後療と同時算定可。

## 10. 多部位逓減

### 対象料金
- 後療料
- 冷罨法
- 温罨法
- 電療

※ 施療料（初検日の基本料）、整復料、固定料は逓減対象外。

### 逓減率
- 1部位目：100%
- 2部位目：100%
- 3部位目：60%
- 4部位目以降：後療料・温罨法・冷罨法・電療は算定不可（3部位目までに含む）

### 対象傷病
- 骨折・不全骨折・脱臼・捻挫・打撲の全てが対象

### 現行UI制約
- 現行UIは最大2部位のため、逓減は実質100%×2まで。3部位目係数は将来拡張用。

### 部位数減少時
- 特定部位が先に治癒し部位数が減少した場合、減少後の部位数に応じた逓減率を適用
- 3部位以上の施術期間中に、ある施術日で2部位のみ施術した場合でも逓減率は変更されない（他部位が治癒した場合のみ変更）

### 端数処理
- 部位ごとの算定過程で1円未満の端数が生じた場合は、その都度小数点以下1桁目を四捨五入

---

## 11. 長期減額

### 対象傷病
- 脱臼・打撲・捻挫・挫傷が対象
- **骨折・不全骨折は長期減額の対象外**

### 対象料金
- 後療料・温罨法・冷罨法・電療

### 起算月
- 各部位の初検日を含む月（ただし初検日が月の16日以降の場合は翌月）から起算
- 部位ごとに個別に起算する
- **起算日の定義**: caseKeyの初検日（エピソード開始日＝受傷日_確定）で固定する

### 75%
- 起算から5か月超（6か月目以降）

### 50%
- 起算から5か月超、かつ月10回以上の施術を5か月連続
- 翌月（6か月目）以降50%

### 長期かつ頻回の特別料金
- 50%算定の場合、75%相当額との差額の範囲内で患者から特別の料金を徴収可能
- 国の公費負担医療受給者からは特別料金の徴収不可

### 30日リセット
長期カウントもリセット（エピソード管理で30日以上空くと新エピソード→初検扱いとなるため自動リセット）

### GAS実装

- `calcLongTermCoef_V3_(injuryType, injuryDate, treatDate, monthlyVisitCounts)` が起算月・経過月数・頻回判定を一括算出
- **75%**: 実装済み。起算月から5か月超で後療料・冷・温・電に0.75を適用（四捨五入）
- **50%**: **実装済み（2026-03-17）**。monthsElapsed≥5 かつ起算月1〜5が全て月10回以上 → ltCoef=0.50
  - 月別来院回数は `buildMonthlyVisitCounts_V3_(headerValues, headMap, patientId, caseKey, injuryDate)` で来院ヘッダから集計（caseKey単位、1visit=1回）
  - fixture テスト TC16a/b/c（31/31 PASS 2026-03-17確認済み）
- **骨折・不全骨折除外**: 実装済み。`injuryType` が骨折/不全骨折なら `ltCoef=1.0`
- 初検日の基本料（施療料/整復料/固定料）と待機料は長期減額の対象外
- 多部位逓減（§10）との併用: 長期75%/50%を各料金に先に適用 → 多部位逓減係数を合計に適用

## 12. 実装安全弁

算定不可でも：

1. チェックは残す
2. 金額は0
3. 要確認=TRUE
4. 要確認理由に記録（;区切り）

例:
冷罨法 算定不可（捻挫：受傷後3日）

## 13. 来院合計

### 設計統一（推奨）

明細合計 = 基本料（施療料/後療料/整復料/固定料） + 冷罨法 + 温罨法 + 電療 + 待機料
来院合計 = 初検料 + 再検料 + 相談支援料 + Σ明細合計（全部位）

※ 基本料は傷病種別と区分（初検/再検/後療）に応じて自動決定（§18参照）

### 待機料
- **対象**: C群（打撲・捻挫・挫傷）のみ
- **算定条件**: 温罨法または電療が算定された場合（金額>0）に加算
- 温/電が0に抑制された日は待機料も0
- 骨折・脱臼の待機料は現時点で未定義（将来対応）
- 単価キー: `待機_打撲捻挫挫傷`

### 端数処理
- 窓口負担のみ roundToUnit_ 適用
- 保険請求額は丸めない

## 14. 施術明細upsert（ステップ④）

### 目的

saveVisit_V3 の金額計算結果を施術明細シートへ部位単位で書き込む。
申請書_転記データ（Ver3_transferData.js）が施術明細を参照するため必須。

### 実装済み（Ver3_core.js: upsertDetailRows_V3_）

### detailID

```
detailID = visitKey + "_C" + caseNo + "_P" + partOrder
```

- 例: `P0001_2026-02-15_C1_P1`
- `caseNo` = UIのケーススロット番号（1または2）
- `partOrder` = 同一ケース内の部位順（1始まり、多部位逓減に使用）
- 既存行があれば上書き、なければ末尾追記（冪等）

### 施術明細シート列（全27列）

| 列名 | 内容 | データソース |
|------|------|------------|
| detailID | 一意キー（主キー） | visitKey_C{n}_P{n} で生成 |
| visitKey | 来院キー | ctx.visitKey |
| 患者ID | | ctx.patientId |
| 施術日 | | ctx.treatDate |
| 区分 | 初検/再検/後療 | ctx.kubun1 or kubun2 |
| caseNo | ケーススロット番号(1/2) | 1=ケース1, 2=ケース2 |
| caseKey | エピソードキー | buildCaseKey_() で生成 |
| 受傷日_確定 | Date | calcOnePartAmount_V3_ の injuryDate |
| 受傷日(入力) | ※未書込（将来用） | — |
| 部位 | 部位名 | calcCaseDetailAmount_V3_ が part.bui に追加 |
| 傷病 | 傷病名 | calcOnePartAmount_V3_ の byomei |
| 部位順位 | 1,2,3... | calcOnePartAmount_V3_ の partOrder |
| 冷 | チェックON/OFF | coldChk |
| 温 | チェックON/OFF | warmChk |
| 電 | チェックON/OFF | electroChk |
| 金属副子チェック | チェックON/OFF（§18.3） | metalChk |
| 運動後療チェック | チェックON/OFF（将来用） | exerciseChk |
| 係数 | 多部位逓減係数 | coef (1.0 or 0.6) |
| 基本料_確定 | 施療料/後療料/整復料/固定料 | base（長期減額前） |
| 初検相談_確定 | 0固定（明細行には分配しない） | 0 |
| 冷_確定 | 冷罨法金額（算定不可は0） | cold |
| 温_確定 | 温罨法金額（算定不可は0） | warm |
| 電_確定 | 電療金額（算定不可は0） | electro |
| 待機_確定 | 待機料金額 | taiki |
| 金属副子_確定 | 金属副子等加算金額（§18.3） | metalOut（逓減対象外） |
| 運動後療_確定 | 柔道整復運動後療料金額（将来用） | exerciseOut（逓減対象外） |
| 行合計_確定 | (ltBase+冷+温+電+待機)×coef + metalOut + exerciseOut | total |

> **注意**: `基本料_確定` は長期減額前の raw base。長期減額（ltCoef）は `行合計_確定` の計算に組み込まれている。
> `金属副子_確定` / `運動後療_確定` は多部位逓減（coef）・長期逓減（ltCoef）の対象外。`行合計_確定` に加算のみ。

### データフロー

```
saveVisit_V3
  └─ calcHeaderAmountsByVisitKey_V3_()
        └─ calcCaseDetailAmount_V3_(caseNo=1) → { total, parts: [part1, part2] }
              └─ calcOnePartAmount_V3_() → { base, cold, warm, electro, taiki,
                                              metalOut, exerciseOut,
                                              coef, longTermCoef, total,
                                              byomei, partOrder, injuryDate,
                                              coldChk, warmChk, electroChk,
                                              metalChk, exerciseChk }
                ※ calcCaseDetailAmount_V3_ が part.bui = 部位名 を追加
        └─ calcCaseDetailAmount_V3_(caseNo=2) → 同様
        └─ amounts.details = { case1Parts: [...], case2Parts: [...] }
  └─ upsertDetailRows_V3_(detailSh, detailMap, ctx)
        ctx = {
          visitKey, patientId, treatDate,
          kubun1, kubun2,
          amounts,   // amounts.details を使用
          ep1, ep2,  // episodeStartDate で caseKey を生成
          now
        }
```

### 冪等性

- `findRowByKey_` で `detailID` 列を検索
- 既存行があれば `setValues` で上書き
- なければ `lastRow + 1` に追記
- 同一 visitKey に対して `saveVisit_V3` を再実行しても行は増えない
  （ただし来院ヘッダの二重登録禁止チェックにより、saveVisit_V3 自体が再実行を禁止している）

### シートセットアップ

メニュー「施術明細ヘッダーセットアップ」(`ensureDetailHeaders_V3`) で全23列のヘッダを1行目に設定する。施術明細シートが存在することが前提。

## 15-2. UI入力バリデーション

### 患者画面の部位入力行レイアウト

```
A=部位名, B=傷病名, C=受傷日, D=冷, E=温, F=電, G=終了日
```

## 18. 傷病種別ごとの算定体系

### 18.1 対象傷病種別と分類

柔道整復施術療養費の対象は外傷性が明らかな以下の5種別。

| 分類 | 傷病種別 | 初検日の料金名 | 後療日の料金名 | 医師同意 |
|------|---------|-------------|-------------|---------|
| A | 骨折 | 整復料 | 後療料（骨折） | 応急手当以外は必要 |
| A | 不全骨折 | 固定料 | 後療料（不全骨折） | 応急手当以外は必要 |
| B | 脱臼 | 整復料（脱臼） | 後療料（脱臼） | 応急手当以外は必要 |
| C | 打撲 | 施療料 | 後療料（打撲） | 不要 |
| C | 捻挫 | 施療料 | 後療料（捻挫） | 不要 |
| C | 挫傷（肉ばなれ） | 施療料 | 後療料（打撲の部で算定） | 不要 |

※ 挫傷は打撲の部の所定料金で算定する（厚労省通知 P40）。
※ 挫傷の算定部位は胸部・背部・上腕部・前腕部・大腿部・下腿部の6部位に限る。

### 18.2 脱臼の算定ルール

#### 基本料

| 区分 | 料金 | 設定シートキー |
|------|------|-------------|
| 初検日 | 整復料（脱臼） | `整復料_脱臼` |
| 再検・後療日 | 後療料（脱臼） | `後療料_脱臼` |

#### 医師同意
- 応急手当を除き、施術に対する**担当医師の同意が必要**
- 同意を求める医師は原則として当該負傷の診療を担当している医師
- 申請書の「摘要」欄に同意した医師の氏名と同意年月日を記載

#### 長期施術
- 打撲・捻挫と同様に**5か月超で75%逓減**、長期かつ頻回で50%逓減の対象
- 3か月を超えて継続する場合は「長期施術継続理由書」の添付が必要（§20参照）

#### 多部位逓減
- 3部位目は60%逓減の対象（打撲・捻挫と同じ）
- 4部位目以降の後療料・罨法料・電療料は算定不可（3部位目までに含まれる）

#### 顎関節脱臼の特例
- 左右各1部位として算定可
- 同時に生じた同側の顔面部打撲は脱臼の所定料金のみで算定（別途算定不可）

### 18.3 骨折・不全骨折の算定ルール

#### 基本料

| 区分 | 骨折 | 不全骨折 | 設定シートキー |
|------|------|---------|-------------|
| 初検日 | 整復料（部位別） | 固定料（部位別） | `整復料_骨折_*` / `固定料_*` |
| 再検・後療日 | 後療料（骨折） | 後療料（不全骨折） | `後療料_骨折` / `後療料_不全骨折` |

#### 部位別単価（整復料・固定料）

初検日の基本料は部位ごとに単価が異なる。設定シートに部位別キーを登録し、
`mapBuiToSettingKey_V3_` で部位名→キーサフィックスをマッピングする。

| 部位サフィックス | 対象部位（キーワード） | 整復料（骨折） | 固定料（不全骨折） |
|-----------------|---------------------|--------------|-----------------|
| `鎖骨` | 鎖骨 | 5,500円 | 3,000円 |
| `肋骨` | 肋骨 | 5,500円 | 3,000円 |
| `指_趾` | 指, 趾, 中手骨, 中足骨, 基節骨, 末節骨 | 5,500円 | 3,000円 |
| `前腕` | 前腕, 橈骨, 尺骨 | 7,200円 | 4,100円 |
| `上腕` | 上腕 | 7,800円 | 4,600円 |

#### 医師同意
- 脱臼と同様、応急手当を除き担当医師の同意が必要

#### 特殊骨折の制限（追加コード不要）

制度上の制限:
- 頭蓋骨・脊椎・胸骨その他の複雑骨折は原則算定不可
- 特に医師から後療を依頼された場合に限り算定可（摘要欄に記載）
- 膝蓋骨骨折の後療も同様

GAS実装での扱い（2026-03-17 確認）:
- 頭蓋骨・脊椎・胸骨・膝蓋骨は `mapBuiToSettingKey_V3_` のキーワード一覧に存在しない
- キーワードにマッチしない部位は `base=0` として算定され、要確認理由に記録される（§12 実装安全弁）
- この動作は**意図的な安全弁**として機能しており、新規コード実装は不要
- 「医師から後療を依頼された場合に限り算定可」の特例ルートはUI設計・実装ともに未定。現状は算定可にする手段がないため、対象症例は手動で摘要欄に対応する

> ※ `mapBuiToSettingKey_V3_` に新たな部位キーワードを追加することは、上記の安全弁を意図せず解除するリスクがある。追加時は制度上の算定可否を必ず確認すること。

#### 金属副子等加算（Phase 1+2 実装済み・TC19a/b PASS 確認済み・TC20a/b/c clasp push 後 PASS 確認待ち）

制度の概要:
- 対象傷病: 骨折・脱臼の整復 / 不全骨折の固定（**C群＝打撲・捻挫・挫傷は対象外**）
- 金属副子等（金属副子・合成樹脂副子・副木・厚紙副子）を使用した場合に加算（1,000円）
- 使用・交換した日を申請書摘要欄と施術録に記載
- 金属副子等の**数・大きさに関わらず**1回の算定（数量に依らない）

算定回数（2026-03-17 確認済み）:

| 区分 | 算定回数 | 金額 |
|---|---|---|
| 初検時（整復料/固定料算定日） | 1回 | 1,000円 |
| 後療時（交換が必要な場合のみ） | 最大2回 | 各1,000円 |
| **通算上限** | **3回** | **最大3,000円** |

> 根拠: 保発0529第4号（令和6年5月29日）。大阪府柔道整復師会FAQ「交換した場合は2回まで加算できます」と明記。§18.3「後療時2回まで」と付録A「3回まで」は矛盾しない（初検1回 + 後療交換2回 = 計3回）。
> カウント単位: **caseKey全期間の通算**（月単位という記述は通知・ガイド資料に見当たらない）。

逓減対象（2026-03-17 条文確認済み）:
- **多部位逓減・長期逓減ともに対象外**
- 条文の逓減対象列挙は「後療料・温罨法料・冷罨法料・電療料」のみであり、加算（金属副子等加算）は含まれない
- 初検時加算も後療時交換加算も、逓減係数（coef・ltCoef）を乗じず `rowTotalOut` に直接加算する

GAS実装方針（Phase 1 / Phase 2 に分割）:

**Phase 1（基本加算・傷病種別チェックのみ）:**
1. UI: 部位入力行に `metalChk`（金属副子）列を追加（A群/B群の傷病のみ有効）
2. 設定シート: `金属副子等加算` = 1,000 を追加（`AM_SET_KEYS`）
3. `calcOnePartAmount_V3_`: 引数 `metalChk` 追加 → 傷病種別が骨折/不全骨折/脱臼なら `metalOut = settings['金属副子等加算']`、それ以外は `metalOut = 0` + 要確認
4. `metalOut` は逓減対象外 → `rowTotalOut += metalOut`（`coef`・`ltCoef` 乗算なし）
5. 戻り値に `metalOut` を追加。施術明細upsert・来院ヘッダへも反映

**Phase 2（回数制限・caseKey来院履歴参照）:**
- 初検日: 通算1回目（整復/固定料算定日のみ。それ以外の初検日は 0 + 要確認）
- 後療日: 通算2回目・3回目まで（caseKey内で `metalOut > 0` の来院数を集計）
- 超えた場合: `metalOut = 0` + 要確認理由 `"金属副子等加算 算定上限超（通算3回）"`
- 実装難度は `buildMonthlyVisitCounts_V3_` と同類。別タスクとして分離

#### 柔道整復運動後療料（未実装）

- 1日320円、負傷日から15日間を除き、週1回程度、月5回限度
- 負傷日が月の15日以前で当月16日以降に後療がない場合は月2回限度
- 負傷日が月の16日以降の場合は当月算定不可

## 19. 近接部位ルール

### 概要
同時に生じた近接部位の負傷は、上位の施術料のみで算定し、別途算定不可。

### ルール一覧

#### 骨折と捻挫
- 関節近接部位の骨折と同時に生じた当該関節の捻挫 → **骨折の所定料金のみ**

#### 捻挫と打撲・挫傷（背部上部/下部で区別）
- 頸部捻挫 + **背部上部**打撲/挫傷 → **捻挫の所定料金のみ**（背部下部は非近接）
- 腰部捻挫 + **背部下部**打撲/挫傷 → **捻挫の所定料金のみ**（背部上部は非近接）
- 肩関節捻挫 + **背部上部**打撲/挫傷（肩部含む） → **捻挫の所定料金のみ**（背部下部は非近接）
- 左右の肩関節捻挫と同時に生じた頸部捻挫/背部打撲 → **肩関節捻挫の所定料金のみ**

#### 骨折・脱臼と下位負傷
- 指・趾骨の骨折/脱臼と同時に生じた不全骨折/捻挫/打撲 → **骨折/脱臼の所定料金のみ**

#### 近接部位の範囲
- 原則: 当該捻挫部位から**上下2関節**までの範囲
- 例外（上部・下部限定）:
  - 手関節捻挫 ↔ 前腕部打撲（上部に限る）
  - 肘関節捻挫 ↔ 前腕部打撲（下部）/ 上腕部打撲（上部）
  - 肩関節捻挫 ↔ 上腕部打撲（下部に限る）
  - 足関節捻挫 ↔ 下腿部打撲（上部に限る）
  - 膝関節捻挫 ↔ 下腿部打撲（下部）/ 大腿部打撲（上部）
  - 股関節捻挫 ↔ 大腿部打撲（下部に限る）

#### GAS実装

- `checkProximityParts_V3_` が同一ケース内の部位1・部位2を判定
- **onEdit**: 部位名/傷病名セル(A12,A13,B12,B13,A27,A28,B27,B28)の編集時にtoastで即時警告
- **saveVisit_V3**: 近接部位が検出された場合は保存をブロック（throw Error）
- 部位名はキーワードマッチ（上部/下部限定の厳密判定は非対応、保守的に警告）

## 20. 長期施術継続理由書

- **対象傷病: 打撲・捻挫・挫傷（C群）および脱臼**
- 施術が初検日から**3か月を超えて**継続する場合
- 負傷部位、症状、施術継続が必要な理由を明記した「長期施術継続理由書」を申請書に添付
- 3か月超かつ月の施術回数が高頻度の場合は、部位ごとに頻度が高い施術が必要な理由も記載

> **重要（2026-03-19 バグ修正）**: 骨折・不全骨折は §20 継続理由書の対象外。`Ver3_amounts.js` の継続理由書アラート判定に `injuryType !== "骨折" && injuryType !== "不全骨折"` 条件を追加済み。TC25b で動作確認（needCheck=false）。

## 21. 申請書転記データを正本とする運用原則

### 目的

表示上の見え方と請求上の最終算定値を混同せず、返戻・査定・過誤請求を防止する。

### 正本の定義

| 位置づけ | 対象 |
|---|---|
| **第1正本** | 申請書転記データ（`申請書_転記データ` シート / JSON / NDJSON エクスポート） |
| **第2正本** | 申請書出力ファイル（xlsx 等、転記データから生成されたもの） |
| **参考資料** | 患者画面、来院ケース、来院ヘッダ、施術明細、初検情報履歴 |

- 参考資料シートは中間情報・内部確認用であり、それ単体で請求可否を確定しない
- シート表示と申請書転記データの金額が一致しない場合は、申請書転記データを優先する

### 確認の優先順位

1. 来院データ保存確認（保存後の来院ヘッダ・施術明細を目視）
2. 申請書転記データ確認（転記データ再生成 → 月次集計・金額を確認）
3. 申請書出力ファイル確認（xlsx 生成 → 記載金額・件数を確認）

### 実務判断ルール

以下のシート表示・部分情報だけで請求可否・金額を確定しない：

- 来院ヘッダの `区分` 列のみで請求区分を確定しない
- 施術明細の1行（1部位）の金額だけで月次請求額を判断しない
- Mixed 要約（`case1要約`/`case2要約`）だけで初検料・再検料の請求可否を判断しない
- 金額確認は必ず転記データの `当月合計` / `窓口負担額` / `請求金額` で行う

### 不一致時の追跡手順

1. 申請書転記データを暫定正本とする
2. 来院ケース・来院ヘッダ・施術明細を突合し、どの段階で値が変わるかを確認する
3. 意味変換が起きているレイヤー（保存時・金額計算時・転記集計時）を特定する
4. 必要ならロジック修正を行う
5. 修正後、転記データを再生成して再判定する

### 請求前チェックリスト

転記データ再生成後、以下の項目を目視確認してから請求確定する：

- [ ] 対象月の患者件数と case 件数が一致しているか
- [ ] 初検料 / 再検料 / 施療料 / 後療料の回数が来院実態と合っているか
- [ ] 温罨法 / 電療の回数が来院実態と合っているか
- [ ] 当月合計 / 窓口負担額 / 請求金額が正しく計算されているか
- [ ] 必須項目（患者名・保険者番号・負傷名等）に空欄がないか
- [ ] `初検取得モード` が `"caseKey"` になっているか（`"patientFallback"` は要注意）

### ロジック修正後の確認セット

ロジック修正を行った場合は、以下を1セットとして必ず実施する：

1. 保存テスト（saveVisit_V3 で対象パターンを再現）
2. 転記データ再生成（メニュー → 転記データ作成）
3. 申請書出力確認（xlsx 生成 → 修正前後の金額差分を確認）
4. 修正前後の差分確認（件数・金額・区分が期待値と一致するか）

上記が完了するまで、修正後データの請求確定を保留する。

- UI A12 から完成文字列を選択（`setAllowInvalid(true)` で自由入力も可）
- `V3TR_buildInjuryLabel_` = `bui + " " + byomei` でそのまま結合

**課題:**
- 左右/上下の全展開で候補数が膨大（100件超）
- 左右あり部位に「頸部」のような左右なし部位が混在し、入力者が混乱
- 申請書で「右」「左」が脱落しやすい

#### 完成文字列の生成ロジック

```javascript
// V3TR_buildInjuryLabel_（変更後）
function V3TR_buildInjuryLabel_(partAgg) {
  if (!partAgg) return "";
  const laterality = partAgg.laterality || "";  // 新フィールド: 左右/上下
  const bui = partAgg.bui || "";
  const byomei = partAgg.byomei || "";
  // "右足関節 捻挫" / "背部上部 打撲" / "頸部 捻挫"
  const buiLabel = laterality ? (laterality + bui) : bui;
  return (buiLabel + " " + byomei).trim();
}
```

### 設計案B — 最小変更方式（暫定案）

A案のUI列変更を避けたい場合の代替:

- 設定シートD列の部位候補を「左右なしの基本部位」のみに変更
- 部位入力時、ユーザーが「右」「左」を手入力で前置する運用に切り替え
- `V3TR_buildInjuryLabel_` は変更なし（`bui` に左右が含まれていれば正しく出力）
- **デメリット**: 左右の入力強制ができない（抜け漏れが起きやすい）
