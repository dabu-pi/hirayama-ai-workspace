# 柔整GASプロジェクト Ver3.1

柔道整復師（接骨院）の患者管理・保険請求業務を支援するGoogle Apps Scriptシステム。

## ステータス

稼働中（Ver3.1）

## 目的

- 来院受付・施術内容の登録
- 保険算定ルールに基づく金額自動計算
- 窓口会計（患者負担額の算出）
- 療養費支給申請書の自動生成
- 現場の日次入力の正本として、保険のみ / 保険+自費 / 自費のみを同じ来院管理の流れで扱う準備

## このブックの役割

- 【毎日記録】来店管理施術録ver3.1 = 日次入力の正本
- JBIZ-04 = 月次KPI / KPI逆算 / 経営判断の正本
- JBIZ-04 側に日次入力シートは作らず、このブックから月次値を渡す前提で進める

## 来院ヘッダ最小列追加の実装準備

### 追加位置案

- 追加先は `来院ヘッダ` の既存末尾、`caseIndex` の右側を第一案とする
- 追加順は `会計区分 / 自費メニュー区分 / 自費売上額 / 慢性候補フラグ / 次回予約あり / 新規区分`
- 既存処理は列番号固定ではなく見出し名参照のため、末尾追加なら既存フローへの影響を最小化しやすい

### 追加する正式列名

| 順序 | 正式列名 | 用途 |
|---|---|---|
| 1 | 会計区分 | `保険のみ / 保険+自費 / 自費のみ` の判定軸 |
| 2 | 自費メニュー区分 | 自費内容の集計軸 |
| 3 | 自費売上額 | 1来院単位の自費売上 |
| 4 | 慢性候補フラグ | 慢性患者移行率の分母候補 |
| 5 | 次回予約あり | 再来先行指標 |
| 6 | 新規区分 | 将来の `保険新規 / 自費直新規 / 再来` 分離用 |

### 選択肢マスタ案

- `設定` シートの `E:I` を新しい選択肢マスタ領域の第一案とする
- 既存の `A:B` は算定キー、`C` は傷病名一覧、`D` は部位名候補のため、右側追記なら既存用途と衝突しにくい

| マスタ項目 | 候補値 |
|---|---|
| 会計区分 | 保険のみ / 保険+自費 / 自費のみ |
| 自費メニュー区分 | 手技50分 / 運動療法 / セルフケア / ジム体験 / その他 |
| 慢性候補フラグ | TRUE / FALSE |
| 次回予約あり | TRUE / FALSE |
| 新規区分 | 空欄 / 保険新規 / 自費直新規 / 再来 |

`自費売上額` は数値入力列のため、選択肢マスタは持たない。

### 月次7項目との対応

| 月次項目 | 主な集計元列 | 集計メモ |
|---|---|---|
| 月次総保険売上 | 会計区分 | `保険のみ / 保険+自費` の来院を保険売上集計対象として切り出す起点 |
| 月次自費売上 | 会計区分 + 自費売上額 | `保険+自費 / 自費のみ` の `自費売上額` 合計 |
| 慢性候補人数 | 慢性候補フラグ | `TRUE` のユニーク患者数 |
| 手技自費患者数 | 会計区分 + 自費メニュー区分 | `手技50分` を受けたユニーク患者数 |
| 手技回数 | 自費メニュー区分 | `手技50分` の来院件数 |
| 自費平均単価 | 自費売上額 + 自費メニュー区分 | 当面は `手技50分` ベースで確認 |
| 次回予約あり件数 | 次回予約あり | `TRUE` の来院件数 |

`新規区分` は今回の7項目の直接集計元ではないが、次段階の `保険新規 / 自費直新規` 分離集計の受け皿として先行確保する。

### リスク整理

- `Ver3_core.js` の `buildHeaderColMap_` は見出し名ベースで列を解決するため、末尾追加なら既存列順依存を増やさない
- `ensureHeaderCols_` は不足列を右端へ追記する実装のため、今回の追加位置案と整合する
- 既存の `区分` 列があるため、新列は `会計区分` として明確に分離し、保険算定の `区分` と混同しない
- `自費売上額` は `保険のみ` 来院で `0` または空欄を許容し、現場入力を重くしすぎない
- `新規区分` は当面空欄運用を許容し、将来の `保険新規 / 自費直新規` 分離時に列追加をやり直さない構成にする

## 算定ロジック概要

1. 30日ルール判定（31日超で初検リセット）
2. 受傷日経過日数による加算可否判定
3. 区分確定（初検 / 再検 / 後療）
4. 患者×月 上限制御
5. 多部位逓減（3部位目60%、4部位目以降算定不可）
6. 長期減額（5か月超75%、長期+頻回50%）
