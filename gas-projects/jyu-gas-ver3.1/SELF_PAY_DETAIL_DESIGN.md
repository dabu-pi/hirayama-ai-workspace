# 自費明細保存設計書 — Phase 2 拡張設計

作成: 2026-03-22
対象: JREC-01（柔整毎日記録システム Ver3.1）+ JBIZ-04（接骨院経営戦略）
目的: Phase 1（単一メニュー手入力）から Phase 2（1来院×1メニュー1行 自費明細保存）への拡張設計を先行記録する
前提: 保険算定ロジック・来院ヘッダ・施術明細（保険専用）は変更しない

---

## 設計原則

| 原則 | 内容 |
|---|---|
| 保険と自費は分けて保存 | 来院ヘッダ = 保険用集計値。自費明細 = 経営分析用の別シート |
| Phase 1 との後方互換 | 来院ヘッダの selfPayAmount / selfPayMenuType 列は Phase 2 以降も保持（JBIZ-04 集計の連続性） |
| 1来院×1行を基本とする | Phase 2 の当面は 1来院 につき 1自費メニュー 1行 |
| 将来の複数行対応を前提 | 自費明細の visitKey + 連番（lineNo）でユニークキーを設計する |
| menu_id を軸とする | 価格マスタ・自費明細・来院ヘッダを menu_id で将来結合できるようにする |

## Phase 1 / Phase 2 の役割分担

### Phase 1（現在稼働中）
- **入力**: 患者画面 行7〜9（B8/D8/F8/B9/D9/F9/H9）
- **保存先**: 来院ヘッダ のみ（7列: 会計区分/自費メニュー区分/自費売上額/慢性候補/次回予約/新規区分/自費メニューコード）

- **入力**: 同上（患者画面 行7〜9 に変更なし）
- **追加保存先**: `自費明細` シート（1来院×1メニュー1行）
- **来院ヘッダ**: 引き続き更新（集計値として保持）
- **Phase 2 拡張時**: 行7〜9 に「メニュー追加ボタン」または行9〜11 に2行目入力欄追加

## 自費明細シート設計（Phase 2）

### シート名
`自費明細`（既存の `施術明細` とは別シート）

### 列定義

| # | 列名 | キー名 | 型 | 内容 | 空欄可 |
|---|---|---|---|---|---|
| 1 | selfPayDetailId | `selfPayDetailId` | 文字列 | visitKey_L{lineNo}（例: P001_2026-04-01_L1） | 不可 |
| 2 | visitKey | `visitKey` | 文字列 | 来院ヘッダとの結合キー | 不可 |
| 3 | lineNo | `lineNo` | 数値 | 1来院内の行番号（1始まり）当面は常に1 | 不可 |
| 4 | treatDate | `施術日` | 日付 | 来院ヘッダと同値 | 不可 |
| 5 | patientId | `患者ID` | 文字列 | 来院ヘッダと同値 | 不可 |
| 6 | accountingType | `会計区分` | 選択肢 | 保険のみ/保険+自費/自費のみ | ✅ |
| 7 | menuId | `menu_id` | 文字列 | M001等（Phase2で価格マスタと結合） | ✅ |
| 8 | menuName | `メニュー名` | 文字列 | 手技50分 等（UIから取得 or 価格マスタ参照） | 不可 |
| 9 | memberType | `会員区分` | 選択肢 | 一般/asago会員（Phase2で追加） | ✅ |
| 10 | unitPrice | `単価` | 数値 | 価格マスタ参照値 or 手入力値 | 不可 |
| 11 | qty | `数量` | 数値 | 当面は常に1 | 不可（デフォルト1） |
| 12 | subtotal | `小計` | 数値 | 単価 × 数量 | 不可 |
| 13 | chronicFlag | `慢性候補フラグ` | boolean | 来院ヘッダと同値 | ✅ |
| 14 | nextReservation | `次回予約あり` | boolean | 来院ヘッダと同値 | ✅ |
| 15 | firstVisitType | `新規区分` | 選択肢 | 来院ヘッダと同値 | ✅ |
| 16 | createdAt | `作成日時` | 日時 | 保存時刻 | 不可 |

### ユニークキー

```
selfPayDetailId = visitKey + "_L" + lineNo
例: "P001_2026-04-01_L1"（将来2行目: "P001_2026-04-01_L2"）
```

## 来院ヘッダ との関係

```
来院ヘッダ（1行/来院）
  ├─ visitKey          ← 結合キー
  ├─ 自費売上額         ← 自費明細.subtotal の合計（Phase2で集計更新）
  ├─ 自費メニュー区分    ← 自費明細の先頭行のmenuName（当面1行なので同値）
  └─ 会計区分 等        ← 同上

自費明細（1行/メニュー/来院 ← 将来複数行対応）
  ├─ visitKey          ← 来院ヘッダと結合
  ├─ lineNo = 1        ← 当面固定
  ├─ menu_id           ← 価格マスタと結合
  └─ subtotal          ← JBIZ-04 KPI集計の集計元
```

> **Phase 1 の来院ヘッダへの直接保存は Phase 2 以降も維持する。**
> 自費明細が保存されている場合は自費明細が正本。
> 自費明細が空の場合は来院ヘッダの自費売上額・自費メニュー区分をフォールバックとして扱う。

### Phase 2 の自動入力ロジック（onEdit）

```javascript
// D8（自費メニュー区分）変更時 onEdit で F8 に自動入力
// 判定: H9 に asago会員 の menu_id が入力済みか、または別途 会員フラグセル（未定）を参照
function onEdit(e) {
  if (e.range.getA1Notation() === UI.selfPay_menuType) {
    var menuName = e.value;
    var price = lookupMenuPrice_(menuName, isGymMember_());  // 価格マスタ参照
    if (price) e.source.getActiveSheet().getRange(UI.selfPay_amount).setValue(price);
  }
}
```

## JBIZ-04 KPI集計への対応

Phase 2 以降、以下の集計クエリが自費明細シートから実行できる:

| KPI | 集計クエリ | 集計元 |
|---|---|---|
| 月次手技自費売上 | SUM(subtotal) WHERE menuId=M001 AND month | 自費明細 |
| 月次全自費売上 | SUM(subtotal) WHERE month | 自費明細 |
| 保険延べ来院数 | COUNT(visitKey) WHERE accountingType IN (保険のみ,保険+自費) | 来院ヘッダ or 自費明細 |
| 自費のみ来店数 | COUNT(visitKey) WHERE accountingType=自費のみ | 自費明細 |
| 慢性候補移行率 | COUNT(慢性候補=TRUE) / COUNT(*) | 来院ヘッダ |
| 再来率 | COUNT(次回予約=TRUE) / COUNT(*) | 来院ヘッダ |
| asago会員割合 | COUNT(memberType=asago会員) / COUNT(*) | 自費明細 |
