## 2. 骨折 / 不全骨折 / 脱臼の算定可否ルール

### 2.1 医師同意要件（全3種共通）

| 場面 | 同意要否 | 根拠 |
|---|---|---|
| 初検（応急手当） | **不要** | 算定基準 §2 但し書き |
| 初検（応急手当以外） | **要（担当医師の同意）** | 算定基準 §2 |
| 後療（2回目以降） | **要（担当医師の同意）** | 算定基準 §2 |

**申請書記載要件:**
- 同意した医師の氏名・同意年月日を申請書「摘要」欄に記載必須
- 同意を求める医師は「当該負傷の診療を担当している医師」が原則

**現行GAS実装:** 医師同意チェック未実装（→ §6 未実装事項参照）

### 2.2 特殊骨折の算定制限

根拠: 留意事項通知 P42〜P43 相当

| 部位 | 整復料 | 後療料 | 備考 |
|---|---|---|---|
| 頭蓋骨 | **不可** | **不可（原則）** | 柔整施術範囲外 |
| 脊椎（頸椎・胸椎・腰椎・仙椎等） | **不可** | **不可（原則）** | 同上 |
| 胸骨 | **不可** | **不可（原則）** | 同上 |
| 膝蓋骨 | **不可（整復不可）** | **不可（原則）** | 整復が柔整師の技術範囲外 |
| 上記以外の「複雑骨折」 | **不可** | **不可（原則）** | 開放骨折・粉砕骨折等 |

**例外（医師依頼後療）:**
- 上記部位でも担当医師から後療を依頼された場合に限り算定可
- 申請書「摘要」欄に依頼した医師の氏名・依頼日を記載必須

**現行GAS実装:**
- 上記部位は `mapBuiToSettingKey_V3_` のキーワードリストに存在しない
- → `calcBaseFee_V3_` が `base=0` を返す
- → `calcOnePartAmount_V3_` L494-495 で reason 追加 → `needCheck=true`
- この動作が **意図的安全弁** として機能（SPEC.md §18.3 記載済み）
- **医師依頼後療の特例ルートは未実装（算定可にする手段がない）**

### 2.3 多部位骨折の整復料・固定料制限

根拠: 算定基準 多部位算定規定 / 留意事項通知 P41

#### 整復料・固定料の多部位での取り扱い

| 状況 | 整復料/固定料 | 後療料 | 根拠 |
|---|---|---|---|
| 1〜2部位 | 各部位を算定可（逓減なし） | 各部位を算定可 | §10 逓減対象外 |
| 3部位目 | 算定可（逓減なし） | 60%逓減 | §10 |
| 4部位目以降 | **算定可（逓減対象外のため）**※要確認 | **算定不可** | §10 |

> **※ 4部位以上の整復料・固定料について:**
> SPEC.md §10 の条文「施療料（初検日の基本料）、整復料、固定料は逓減対象外」から
> 逓減の適用がないことは確認済み。ただし4部位目以降の算定不可制限が整復料・固定料にも
> 適用されるかは制度原文を直接確認できていない。**実装前に通知原文で確認要。**

#### 同一骨の複数骨折（橈骨 + 尺骨 など）

| パターン | 算定単位 | 理由 |
|---|---|---|
| 橈骨 + 尺骨（前腕両骨折） | **1部位（前腕）** | `mapBuiToSettingKey_` → `"前腕"` に統合される |
| 脛骨 + 腓骨（下腿両骨折） | **1部位（下腿）** | 同上 |
| 鎖骨 + 肋骨（別の骨） | **2部位** | 別キーワードのため個別算定 |

> 現行実装では部位名のキーワードマッチが単一サフィックスを返すため、
> 橈骨と尺骨を別部位入力しても両方 `"前腕"` キーにマッチする。
> 同一来院の2部位入力で同じ整復料を2回算定するリスクあり → §6 未実装事項参照。

### 2.4 脱臼の特殊算定条件

| 条件 | 算定 | 根拠 |
|---|---|---|
| 通常脱臼（初検・応急手当）| 整復料（脱臼） | §18.2 |
| 通常脱臼（後療） | 後療料（脱臼）+ 長期減額対象 | §11 §18.2 |
| 顎関節脱臼 | 左右各1部位として算定可 | §18.2 特例 |
| 顎関節脱臼 + 同側顔面部打撲 | 脱臼料金のみ（打撲別途算定不可） | §18.2 特例 |
| 先天性脱臼 | **算定不可**（外傷性でないため） | 算定基準 対象外 |
| 習慣性脱臼 | **算定不可（原則）** | 外傷性の一時性が条件 |

> **先天性・習慣性脱臼の現行実装:**
> 傷病名プルダウンは「脱臼」の1択のみ（SPEC.md §15-2）。先天性・習慣性の区別はUIで入力不可。
> → 入力時点で制度上の請求可否を判定する手段がない。
> → 現状は算定されてしまう。将来的には「脱臼種別」フィールドの追加が必要。

---

### 2.5 骨折 × 脱臼の複合部位

| パターン | 算定 |
|---|---|
| 非近接部位の骨折 + 脱臼 | 各々算定可 |
| 近接部位の骨折 + 脱臼 | §19 近接部位ルール適用 |
| 指・趾骨の骨折/脱臼 + 不全骨折/捻挫/打撲 | 骨折/脱臼の料金のみ（§19） |

---

### 2.6 長期施術の取り扱い（傷病種別比較）

| 傷病 | 長期減額（§11）| 3か月超継続理由書（§20）|
|---|---|---|
| 骨折 | **対象外**（ltCoef=1.0 固定） | **対象外** |
| 不全骨折 | **対象外** | **対象外** |
| 脱臼 | **対象**（5か月超→75%/50%） | **対象**（3か月超で添付必須） |
| 打撲・捻挫・挫傷 | **対象** | **対象** |

## 7. fixture 実装時の注意点

### calcOnePartAmount_V3_ の引数確認

TC21a〜TC21d のような「部位あり・加算なし」の fixture を書く場合:

```js
// 最小構成例（TC21a 相当）
var result = calcOnePartAmount_V3_(
  settings,       // loadSettings_V3_ の返り値
  "初検",          // kubun
  "骨折",          // byomei（detectInjuryType_ の入力）
  new Date("2026-03-01"),  // injuryDate
  new Date("2026-03-01"),  // treatDate（受傷日 = 来院日: dayDiff=0）
  false,          // coldChk
  false,          // warmChk
  false,          // elecChk
  1,              // partOrder
  [],             // reasons（空配列で渡す）
  "膝蓋骨",        // bui
  {},             // monthlyVisitCounts（長期減額計算用）
  false,          // metalChk
  0,              // metalPriorCount
  false,          // exerciseChk
  0               // exercisePriorCount
);
// 確認: result.base === 0, result.needCheck === true
// result.reasons に "整復料/固定料 取得不可（膝蓋骨" を含む
```
