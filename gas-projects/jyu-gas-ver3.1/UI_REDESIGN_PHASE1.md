## 案A / 案B / 案C 比較

### 案A（推奨）: 行7〜9 を会計ブロックとして活用

| 項目 | 評価 |

- Phase 0 の Sheets 設置前なので、B55→B8等の変更がもっともコストゼロ
- 関数本体（readSelfPayFromUI_V3_・clearSelfPayUI_V3_等）は UI.selfPay_* を参照しているためセル番地変更は UI const の1箇所のみ

## 推奨案（案A）の具体的セル配置

### レイアウト設計（行7〜9）

```
Row 7  : A7:H7 「─── 会計・経営情報 ───────────────────────」（見出し結合、ダークグレー背景・白文字）
Row 8  : A8:「会計区分」    B8:[dropdown] 保険のみ/保険+自費/自費のみ
         C8:「自費メニュー」 D8:[dropdown] 手技50分/運動療法/セルフケア/ジム体験/その他
         E8:「自費金額（円）」F8:[数値入力]
         G8:「会計合計」    H8:[数式 =E3+F8]  ← 窓口負担+自費金額の合算表示
Row 9  : A9:「慢性候補」   B9:[チェックボックス]
         C9:「次回予約」   D9:[チェックボックス]
         E9:「新規区分」   F9:[dropdown] 保険新規/自費直新規/再来
         G9:「メニューコード」H9:[テキスト入力]  ※将来拡張用
```

> **H8 の会計合計** は Sheets 数式で実装（GASは書かない）:
> `=IF(F8="",E3,E3+F8)` — 自費なし時は窓口負担のみ表示、自費あり時は合計表示

### E列（表示エリア）の扱い

| セル | 現状 | 変更後 |
|---|---|---|
| E2 | 来院合計（保険） | **変更なし** — 保険算定の来院合計を表示し続ける |
| E3 | 窓口負担額 | **変更なし** |
| E4 | 保険請求額 | **変更なし** |
| E5 | 要確認フラグ | **変更なし** + 条件付き書式追加（TRUE時: 背景橙） |
| E6 | 要確認理由 | **変更なし** + 条件付き書式追加（内容あり時: 橙） |

**要確認/要確認理由の扱い決定:**
→ **E5:E6 に保持する。削除・移動しない。**
理由: 保険算定コンプライアンスの最重要表示。文字を小さく/グレーにするのではなく、
`要確認=TRUE` のとき条件付き書式で**橙背景** にすることで「ふだんは目立たず、問題あり時に強調」を実現する。
GASコード変更不要（Sheets 条件付き書式のみ）。

## 価格マスタ連動方針

### Phase 1（今回: 手入力）
- F8（自費金額）はユーザーが直接入力
- 選択メニューに応じて「参考価格表」をコメントや隣接セルに表示（オプション）

### Phase 2（将来: VLOOKUP自動入力）

```
設定シート「価格マスタ」から参照:
  メニュー名 | 一般料金 | ジム会員料金 | menu_id
  手技50分   | 5500     | 4700         | M001
  運動療法   | 3300     | 2800         | M002
  ...

F8 の数式案（将来）:
  =IF(D8="","", IFERROR(VLOOKUP(D8, 設定!$A$xx:$D$yy, IF(isGymMember, 3, 2), FALSE), ""))
```

> **会員フラグ（isGymMember）** は将来 JBIZ-04 の患者マスタと連携。
> Phase 1 では手入力で問題なし。将来拡張しやすいよう H8 数式を `=E3+F8` にしておく。
