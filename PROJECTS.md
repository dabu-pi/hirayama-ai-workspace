## 1. freee見積自動化プロジェクト

**ディレクトリ:** `freee-automation/`
**ステータス:** 開発中

### 目的

freee APIを使用して、見積書の作成・送信を自動化する。
手入力による作業時間の削減と、入力ミスの防止が主な目標。

### 対象業務

- 設備販売・施工に関する見積書作成
- 顧客ごとの見積データ管理
- 承認後の請求書への転換

### 技術構成

| 要素 | 内容 |
|---|---|
| 言語 | Python |
| API | freee API v1 |
| 認証 | OAuth2 |
| 実行環境 | Windows / ローカル実行 |

### 主要機能（設計）

- 見積データのJSON構築
- freee APIへのPOSTリクエスト
- 作成結果の確認・ログ出力

### 今後の拡張

- 請求書自動化
- 会計仕訳連携
- スプレッドシートからの一括入力

## 2. 柔整GASプロジェクト

**ディレクトリ:** `gas-projects/jyu-gas-ver3.1/`
**ステータス:** 稼働中（Ver3.1）

### 目的

柔道整復師（接骨院）の患者管理・保険請求業務を支援するシステム。
スプレッドシートをUIとして、施術録の記録から療養費支給申請書の作成までを一貫して管理する。

### 対象業務

1. 患者の来院受付・施術内容登録
2. 保険算定ルールに基づく金額自動計算
3. 窓口会計（患者負担額の算出）
4. 療養費支給申請書の生成（公的機関への提出書類）
5. 保険請求データの集計・管理

### 技術構成

| 要素 | 内容 |
|---|---|
| 言語 | Google Apps Script (JavaScript) |
| プラットフォーム | Google スプレッドシート |
| バージョン管理 | clasp + GitHub |

### システム設計

#### データキー体系

| キー | 形式 | 意味 |
|---|---|---|
| visitKey | `患者ID_yyyy-MM-dd` | 1来院日の一意キー |
| caseKey | `患者ID_初検日_C{n}` | 1受傷エピソードの一意キー |

#### 金額計算ロジック（判定優先順位）

1. 30日ルール判定（31日超で初検リセット）
2. 受傷日経過日数による加算可否判定
3. 区分確定（初検 / 再検 / 後療）
4. 患者×月 上限制御（初検料・再検料・相談支援料は月1回）
5. 要確認フラグ付与
6. 多部位逓減（3部位目60%、4部位目以降算定不可）
7. 長期減額（5か月超75%、長期+頻回50%）
8. 来院合計算出

#### 傷病種別と算定体系

| 分類 | 傷病 | 初検日料金 | 後療日料金 |
|---|---|---|---|
| A | 骨折 | 整復料（部位別） | 後療料850円 |
| A | 不全骨折 | 固定料（部位別） | 後療料720円 |
| B | 脱臼 | 整復料 | 後療料720円 |
| C | 打撲・捻挫・挫傷 | 施療料760円 | 後療料505円 |

#### 加算（冷罨法・温罨法・電療）

受傷日からの経過日数で算定可否を自動判定。

| 加算 | 条件 |
|---|---|
| 冷罨法（85円） | 打撲・捻挫は受傷後1日以内、骨折は7日以内 |
| 温罨法・電療（75/33円） | 打撲・捻挫は受傷後6日目以降、骨折は8日目以降 |

### 設計方針

- **レセプト事故ゼロ** — 返戻・査定を最小化
- **算定不可でも金額0・要確認フラグで記録**（自動却下しない）
- **単価はすべて設定シートで管理**（コード内に固定値を持たない）
- 令和6年6月施行の単価に対応済み

## 4. hirayama接骨院戦略AI（開発予定）

**ディレクトリ:** `hirayama-jyusei-strategy/`（未作成）
**ステータス:** 企画段階

### 目的

平山接骨院の経営戦略立案・患者集客・サービス改善をAIでサポートする。
データ分析と提案生成を自動化し、経営判断を加速する。

| 要素 | 内容 |
|---|---|
| 言語 | Python / Node.js |
| AI | Claude API (claude-sonnet-4-6) |
| データソース | スプレッドシート・柔整GASシステム |
| 出力形式 | Markdown レポート / PDF |

### 主要機能（設計）

- 経営データの読み込みと集計
- Claude APIによる分析・提案文の自動生成
- 月次戦略レポートの出力
- KPI管理ダッシュボード（将来）
