## 1. 柔整GASプロジェクト

**ディレクトリ:** `gas-projects/jyu-gas-ver3.1/`
**ステータス:** 稼働中（Ver3.1）
**最終確認日:** 2026-03-05

### 目的

柔道整復師（接骨院）の患者管理・保険請求業務を支援するシステム。
スプレッドシートをUIとして、施術録の記録から療養費支給申請書の作成までを一貫して管理する。

### 対象業務

1. 患者の来院受付・施術内容登録
2. 保険算定ルールに基づく金額自動計算
3. 窓口会計（患者負担額の算出）
4. 療養費支給申請書の生成（公的機関への提出書類）
5. 保険請求データの集計・管理

### 技術構成

| 要素 | 内容 |
|---|---|
| 言語 | Google Apps Script (JavaScript) / Python |
| プラットフォーム | Google スプレッドシート |
| バージョン管理 | clasp + GitHub |

### 主要ファイル

| ファイル | 役割 |
|---|---|
| `Ver3_core.js` | 来院登録・区分判定・施術明細upsert・算定ロジック中枢 |
| `Ver3_amounts.js` | 金額計算 |
| `Ver3_transferData.js` | 申請書データ転記 |
| `Ver3_patientPicker.js` | 患者選択UI |
| `write_application.py` | 療養費支給申請書生成（ローカル実行Pythonスクリプト） |

### 算定ロジック（判定優先順位）

1. 30日ルール判定（31日超で初検リセット）
2. 受傷日経過日数による加算可否判定
3. 区分確定（初検 / 再検 / 後療）
4. 患者×月 上限制御（初検料・再検料・相談支援料は月1回）
5. 要確認フラグ付与
6. 多部位逓減（3部位目60%、4部位目以降算定不可）
7. 長期減額（5か月超75%、長期+頻回50%）
8. 来院合計算出

### 傷病種別と算定体系

| 分類 | 傷病 | 初検日料金 | 後療日料金 |
|---|---|---|---|
| A | 骨折 | 整復料（部位別） | 後療料850円 |
| A | 不全骨折 | 固定料（部位別） | 後療料720円 |
| B | 脱臼 | 整復料 | 後療料720円 |
| C | 打撲・捻挫・挫傷 | 施療料760円 | 後療料505円 |

### 設計方針

- **レセプト事故ゼロ** — 返戻・査定を最小化
- **算定不可でも金額0・要確認フラグで記録**（自動却下しない）
- **単価はすべて設定シートで管理**（コード内に固定値を持たない）
- 令和6年6月施行の単価に対応済み

## 2. freee見積自動化プロジェクト

**ディレクトリ:** `freee-automation/`
**ステータス:** 開発中（フェーズ2で詰まり中）
**最終確認日:** 2026-03-05

### 目的

Gmailで受信した見積依頼メールを起点に、freee見積書作成・PDF返信下書き・スプレッドシート記録を自動化する。

### 対象業務

- `hawk@pop13.odn.ne.jp`（長谷川さん）からの見積依頼メールを検知
- freeeで見積書を自動作成
- PDF添付の返信メールを下書き作成（**自動送信はしない**）
- 案件管理スプレッドシートへの進捗自動記録

### 技術構成

| 要素 | 内容 |
|---|---|
| 言語 | Google Apps Script (JavaScript) |
| 連携 | Gmail / freee API / Google スプレッドシート |
| 認証 | OAuth2 (freee) |

### 主要ファイル

| ファイル | 役割 |
|---|---|
| `src/freee請求書作成.js` | freee API連携・見積書作成 |
| `src/hawkメール自動貼り付け.js` | Gmail受信検知・内容抽出 |
| `src/phase3_下書き作成.js` | 返信メール下書き作成 |
| `spec.md` | 詳細仕様書 |

### 現在の課題

フェーズ2: freee本番OAuth で `invalid_grant / 401` エラーが発生中。
`redirect_uri` をGAS WebアプリURL（`/exec`）と完全一致させて再認証が必要。

## 3. 患者管理Webアプリ

**ディレクトリ:** `patient-management/`
**ステータス:** 開発中（プロトタイプ）
**最終確認日:** 2026-03-05

### 目的

接骨院の患者住所録をGoogle スプレッドシートと連携して管理するFlask Webアプリケーション。

### 対象業務

- 患者情報（氏名・生年月日・住所等）の登録・閲覧
- 郵便番号からの住所自動補完
- 年齢の自動計算

### 技術構成

| 要素 | 内容 |
|---|---|
| 言語 | Python (Flask) |
| データストア | Google スプレッドシート（gspread） |
| 認証 | Google Service Account |
| 実行環境 | ローカル（localhost:5000） |

### セキュリティ注意事項

- `service_account.json` は gitignore 済み。各PC に個別配置する
- パスは `.env` 経由で管理する（現在整備中）

### 今後の拡張

- 柔整GASシステムとの患者IDスキーマ共通化
- 患者検索・フィルタリング機能
- 来院履歴の可視化

## 4. hirayama接骨院戦略AI

**ディレクトリ:** `hirayama-jyusei-strategy/`
**ステータス:** ドキュメント作成済み・実装予定
**最終確認日:** 2026-03-05

### 目的

平山接骨院の経営戦略立案・患者集客・サービス改善をAIでサポートする。
慢性疼痛特化の整骨院×トレーニングジムモデルの経営戦略を軸に、データ分析と提案生成を自動化する。

### 対象業務

- 患者数・売上トレンドの分析
- 新患集客施策の立案
- 既存患者のリテンション戦略
- 競合分析・差別化ポイントの整理

### 技術構成（予定）

| 要素 | 内容 |
|---|---|
| 言語 | Python |
| AI | Claude API (`claude-sonnet-4-6`) |
| データソース | Google スプレッドシート（柔整GASシステム） |
| 出力形式 | Markdown レポート / PDF |

### ドキュメント構成

| フォルダ | 内容 |
|---|---|
| `strategy/` | ビジネスモデル・収益・患者フロー・KPI |
| `menu/` | 施術メニュー・料金・会員プラン |
| `operations/` | 日次業務・スケジュール・スタッフ導線 |
| `marketing/` | ポジショニング・患者獲得・ジム転換 |
| `finance/` | コスト構造・利益シミュレーション・設備投資 |

### 次のアクション

`finance/` 配下の数値（固定費・変動費・設備投資額・月次売上）を実数値で入力してから Claude API 実装に着手する。
