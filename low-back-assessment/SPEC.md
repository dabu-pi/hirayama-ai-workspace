# SPEC.md — 腰痛評価シートシステム仕様書 (JEVAL-01)

最終更新: 2026-03-23

---

## 1. プロジェクト目的

接骨院の腰痛初回・再評価業務において：

1. **評価の標準化** — 施術者が変わっても同じ評価軸で記録できる
2. **評価者差の低減** — プルダウン・チェックボックス中心で主観ブレを最小化
3. **評価→方針の一貫化** — 入力すると評価まとめ・注意点・施術優先順位・セルフケア案が自動表示される
4. **将来AI連携の基盤** — 施術者判定 / ルールベース / AI の3層判定構造を前提にしたデータ設計
5. **施術者育成の基盤** — 何を評価すべきかが可視化され、評価の「考え方」が身につく設計

## 2. 評価項目の根拠

### 採用した評価ツールとエビデンス根拠

| 評価ツール | 略称 | エビデンス概要 | 採用判断 |
|---|---|---|---|
| Numeric Rating Scale | NRS | 痛みの数値評価。感度・再現性ともに高い（VASと同等）。患者・施術者双方に簡便 | ✅ 採用 |
| Roland Morris Disability Questionnaire（短縮版） | RMDQ-10 | 腰痛機能障害の国際標準指標。感度高・臨床変化の検出に優れる（Ostelo 2008） | ✅ 採用（10項目版） |
| STarT Back Screening Tool（日本語版相当） | STarT-J簡易版 | 慢性化リスクの早期スクリーニング。RCTで有効性確認（Hill 2011, Lancet） | ✅ 採用（9項目簡易版） |
| Patient-Specific Functional Scale | PSFS | 患者個人の機能目標に対する達成度評価。治療目標の共有に有用（Stratford 1995） | ✅ 採用（施術者聞き取り版） |
| 赤旗（Red Flags） | RF | 重篤疾患除外のスクリーニング。診療ガイドライン共通必須項目（NICE 2016等） | ✅ 採用 |
| 神経症状評価 | 神経 | 神経根障害・馬尾症候群除外。SLR、デルマトーム参照 | ✅ 採用（簡易版） |
| 動作評価（前屈・後屈・側屈） | 動作 | 機能障害の客観的記録。繰り返し評価で改善を可視化 | ✅ 採用 |
| 移乗動作評価 | 移乗 | 高齢者・ADL低下例で必須。転倒リスクと連動 | ✅ 採用 |

### 採用しなかったもの・理由

| ツール | 除外理由 |
|---|---|
| FABQ（恐怖回避思考質問票） | 初版では複雑すぎる。STarT-J簡易版で心理社会的因子はカバー |
| ODI（Oswestry Disability Index） | 10セクション×6択で煩雑。RMDQより現場負担が大きい |
| SLR角度の精密計測 | 角度計が不要なように「陽性/陰性/疼痛誘発」の3段階で代替 |

## 3. システム構成

### シート一覧

| # | シート名 | 役割 |
|---|---|---|
| 1 | 設定 | 施設情報・プルダウン選択肢マスタ・スコア閾値 |
| 2 | 患者マスタ | 患者基本情報の一元管理 |
| 3 | 腰痛評価入力 | 施術者が評価を入力するメインシート |
| 4 | 評価履歴 | 評価結果の時系列記録（自動転記） |
| 5 | 判定ロジック | ルールベース判定の中間計算シート（非表示可） |
| 6 | コメントマスタ | 条件別コメントテンプレート一覧 |
| 7 | AI判定用出力 | AI連携時のJSON出力用シート（将来用） |
| 8 | スタッフマスタ | 施術者一覧・資格情報 |

### 判定3層構造（将来完成形）

```
入力値
  └─ Layer 1: ルールベース判定（GAS）
       └─ Layer 2: 施術者判定（人間が入力）
            └─ Layer 3: AI判定（Claude API）
                 └─ 最終コメント出力
```

初版（Phase 1〜4）は Layer 1 + Layer 2 のみで完全稼働を目指す。

## 4. データフロー

```
施術者が「腰痛評価入力」シートに入力
    ↓
onEdit トリガー（GAS）が発火
    ↓
「判定ロジック」シートで自動計算
    ↓
評価コメント・注意点・方向性が入力シートに自動表示
    ↓
評価完了後「記録確定」ボタン押下
    ↓
「評価履歴」シートへ転記（1行追記）
    ↓
（将来）「AI判定用出力」シートへJSON出力
    ↓
（将来）Claude API呼び出し → AI判定コメント返却 → 入力シートに表示
```

## 5. 設計原則

| 原則 | 具体的実装 |
|---|---|
| 入力負荷最小 | プルダウン・チェックボックス中心。テキスト自由入力は所見欄のみ |
| 評価者差の低減 | すべての項目に「判定基準メモ」をセルコメントで表示 |
| 将来拡張性 | AI判定用列を初版から確保。列順を変えない |
| 単価・閾値の外部管理 | スコア閾値は「設定」シートで管理。コードにハードコードしない |
| 安全ネット | 赤旗・馬尾症候群項目は入力時に自動アラート（背景色変化） |
| 患者説明連動 | 評価結果から「患者向け説明文」を自動生成 |

## 7. 用語定義

| 用語 | 定義 |
|---|---|
| 赤旗（Red Flags） | 重篤疾患（骨折・腫瘍・感染・馬尾症候群等）を示唆する症状・所見 |
| 慢性化リスク | 腰痛が慢性化・遷延化する心理社会的・身体的リスク因子 |
| 機能評価 | ADL・日常動作への影響度の評価 |
| 施術者判定 | 施術者が総合的に入力する治療優先順位・方向性 |
| ルールベース判定 | GASのif/switch処理で自動生成される判定・コメント |
| AI判定 | Claude APIが評価データをもとに生成する判定・コメント（将来実装） |
| PSFS | 患者が困っている具体的な動作を3つ挙げ、できる度合いを0〜10でスコア化 |
| STarT簡易版 | 慢性化リスクを9項目でスクリーニング（Yes/Noで回答、5点以上=高リスク） |
