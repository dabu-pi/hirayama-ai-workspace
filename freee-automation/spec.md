# freee 見積自動化 仕様書

作成日: 2026-03-03

---

## 1. 目的

`hawk@pop13.odn.ne.jp`（長谷川さん）からの見積依頼メールを起点に、以下を自動化して業務を省力化する。

1. メール内容を確認し必要情報を抽出
2. freee の見積書で見積を作成
3. 見積ができたら PDF を添付した返信メールを「下書き」作成（送信しない）
4. 案件管理用の Google スプレッドシートに進捗を自動記録（「2024長谷川さん」ページ）

---

## 2. 現行業務フロー

- 長谷川さん（hawk）が顧客に営業を行い、弊社へ見積・請求依頼メールが届く
- 弊社は見積書作成（freee）／請求書作成／入金管理などの事務を担当
- 進捗はスプレッドシートで管理

---

## 3. 使用システム

| システム | 用途 |

| Gmail | 見積依頼の受信、下書き作成 |
| freee | 見積書（freee見積書）を発行 |
| Google スプレッドシート | 案件進捗管理 |

## 4. 記録先シート仕様（「2024長谷川さん」ページ）

| 列 | 項目 | 自動/手動 | 内容 |
|----|------|----------|------|
| B | お客様名 | 自動 | 件名の（）内、または本文から抽出 |
| C | 発生日 | 自動 | 受信日（原則）または本文記載日 |
| D | 取引の内容 | 自動 | 案件が分かる短い内容（例: KEISER M3 バイクマウント） |
| E | 長谷川 | 自動（固定値） | 常に「長谷川」 |
| G | 見積をした日 | 自動 | freee で見積作成成功時 |
| H | 受注した日 | 手動 | - |
| I | 請求書を発行した日 | 手動 | - |
| J | お客様から入金確認した日 | 手動 | - |
| K | 領収書を発行した日 | 将来自動化可 | - |
| L | 外注をした日 | 手動 | - |

### 重複防止

- 各行に Gmail の **Message-ID** を保存し、同一メールの二重処理を防ぐ
- 処理済みメールには Gmail ラベルを付与

## 5. 自動化トリガー（2-5確定 2026-03-06）

- 送信元: `hawk@pop13.odn.ne.jp`
- 件名フィルタ: 「見積」「見積依頼」「作成お願いします」のいずれかを含む件名のみ処理
  - 対象外の件名はラベルのみ付与してスキップ（台帳には記録しない）
- 時間トリガー: 5〜15 分ごとに GAS が未処理メールを確認

## 6. メールからの情報抽出

### 件名ルール

- 例: `見積もりの作成お願いします（ASRE様）`
- 括弧（）内の文字列を **お客様名**（B列）として優先使用

### 抽出対象フィールド

| フィールド | 用途 | 必須/任意 |
|-----------|------|----------|
| 取引先名（会社名/担当者名） | freee 見積の宛先 | 必須 |
| 見積内容（作業内容・機種・型番等） | D列 / 明細 | 必須 |
| 明細（品目/数量/単価 or 一式金額） | freee 明細行 | 必須 |
| 送料・出張費・消費税の扱い | freee 調整行 | 任意 |
| 納期/作業予定日 | 備考欄 | 任意 |

### メールテンプレ（推奨フォーマット）

長谷川さんへの依頼：メール本文をできる限りテンプレ化し、抽出精度を向上させる。

## 7. 見積の注意書き（freee 備考欄）

以下の文言を見積書に必ず含める:

> メンテ中に修理不能の問題が発見された場合、取引を中止する可能性がある

---

## 8. 事故防止（絶対条件）

1. **二重処理を絶対に避ける**
   - Gmail ラベル付与（処理済みマーク）
   - シートに Message-ID を保存し冪等性を担保
2. **勝手に送信しない**
   - 自動化は必ず「下書き保存」止まり
   - 人が最終確認して送信する

---

## 9. 実装フェーズ

### フェーズ 1（最優先・安定）: メール → シート記録

**目的**: 案件の取りこぼしゼロ

- 対象メール検知（時間トリガー 5〜15分）
- 「2024長谷川さん」へ追記: B/C/D/E/O 列
- 処理済みラベル付与
- Message-ID 保持で重複防止

**成果**: 台帳に自動で積まれる状態（見積作成はまだ）

**現状**: 動作し始めている ✅

---

### フェーズ 2（半自動 OK）: freee 見積作成

**目的**: 見積作成工数削減

- メールから情報抽出 → freee で見積作成
- 成功時 G 列に見積日を自動記録
- freee 見積 ID / URL をシートに保存（列追加 or メモ）
- 抽出不完全な場合は「要確認」フラグを立てて残す

**現状**: freee 本番 OAuth の `invalid_grant / 401` エラーで詰まり中 → 再構築中 🔧

---

### フェーズ 3: Gmail 下書き作成（PDF 添付）

**目的**: 返信作業を「確認だけ」に

- 見積作成できたら:
  - 見積 PDF を添付
  - 返信文を生成
  - 下書きに保存（送信しない）
- PDF 添付が難しい場合の暫定案: freee 見積 URL を本文に貼る

---

### フェーズ 4: 運用強化

- エラー通知（Slack / メール等）
- 取引先マスタ整備（顧客名 → freee 取引先 ID）
- 品目マスタ整備（定型品目）
- 重複検知（同一顧客・同一件名）

## 10. freee 本番 OAuth 再構築メモ

| ポイント | 内容 |
|---------|------|
| `redirect_uri` | GAS WebアプリURL `/exec` と完全一致が必要 |
| `code` | 使い捨て（再利用不可）|
| Client Secret 更新 | GAS 側の ScriptProperties も必ず更新する |
| Token 保存場所 | ScriptProperties（Token / RefreshToken）|
| リセット手順 | `reset()` 関数で ScriptProperties の OAuth 情報を全削除してからやり直す |

---

## 11. 技術スタック

- **実行環境**: Google Apps Script（GAS）
- **認証**: OAuth 2.0（freee）、Google APIs（Gmail / Sheets）
- **主要 API**:
  - Gmail API（メール取得・ラベル付与・下書き作成）
  - Google Sheets API（スプレッドシート書き込み）
  - freee API（見積書作成・PDF 取得）

## 12. 確定仕様（2026-03-06 確定）

### 旧未決事項 → 確定

| 項目 | 確定内容 |
|---|---|
| 対象メール | `hawk@pop13.odn.ne.jp` 送信 かつ 件名に「見積」「見積依頼」「作成お願いします」のいずれかを含むもの |
| 発生日（C列）ルール | Gmail 受信日時を使用（本文日付は参照しない） |
| 見積日（G列）ルール | freee POST 成功時の当日日付 |
| 取引先未解決時 | T列に `要確認（取引先未解決：理由）` を記録してスキップ |
| 明細不足時 | freee POST しない・Gmail下書きも作成しない・T列に `要確認（明細不足）` を記録 |

### T列（要確認フラグ）仕様

| 値 | 意味 | 対応 |
|---|---|---|
| `要確認（明細なし：Q列手入力要）` | Phase1で明細自動抽出失敗 | Q列にlines_jsonを手入力 |
| `要確認（顧客名不明：B列確認要）` | Phase1で顧客名抽出失敗 | B列・P列を手動確認 |
| `要確認（取引先未解決：〇〇）` | Phase2で partner_id 解決失敗 | freeeで取引先確認またはP列手入力 |
| `要確認（明細不足：Q列にlines_jsonを入力）` | Phase2でQ列が空 | Q列にlines_jsonを入力 |
| `（空）` | 問題なし or 見積作成済み | - |
