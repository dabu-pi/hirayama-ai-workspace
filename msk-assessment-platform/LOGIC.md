## 共通ロジック（全症状で使う）

以下は腰痛以外のモジュールでも同様に使われる共通判定ロジック。

### 共通: 赤旗チェック（最優先・全症状共通）

すべての症状評価で最初に実行される共通判定。

```
FLAG_CAUDA（馬尾）, FLAG_REDFLAG（赤旗1件以上）
→ 詳細は腰痛モジュール内に実装（他モジュールでも同フラグ名を使う）
```

### 共通: NRS強度判定

```
NRS 0〜3 = 軽度 / 4〜6 = 中等度 / 7〜10 = 重度
（腰痛・頸肩・膝すべてで同一）
```

### 共通: 転倒リスク判定

移乗動作評価から算出。高齢者評価モジュールでも流用。

### 共通: PSFS変化判定

```
PSFS +2以上 = 有意な改善（MCID達成）
（全症状共通のアウトカム指標）
```

## 判定3層構造の概念

```
Layer 1: ルールベース判定（GAS自動）  ← Phase 1〜2 で実装
Layer 2: 施術者判定（人間入力）        ← Phase 1 から運用
Layer 3: AI判定（Claude API）         ← Phase 5 で実装
         ↓
最終コメント出力
```

初版では Layer 1 + Layer 2 で完全に稼働できるよう設計する。

## ルールベース判定フロー（Layer 1）

### Step 1: 赤旗チェック（最優先）

```
if (赤旗22行「排尿・排便障害」== "あり" || 赤旗23行「会陰部感覚異常」== "あり") {
    判定 = "【緊急】馬尾症候群疑い — 即日医療機関紹介を検討"
    アラート背景 = 赤
    以降の評価は参考値として記録するが、施術は原則中止
}
else if (赤旗スコア >= 1) {
    判定 = "赤旗あり — 医療連携を検討してから施術方針を決定"
    アラート背景 = オレンジ
}
else {
    赤旗クリア
}
```

### Step 2: 神経症状レベル判定

```
if (SLRテスト == "両側陽性" || 下肢筋力低下 == "あり") {
    神経レベル = "重度"
    → 神経所見を最優先。医療連携の検討を促す
}
else if (SLRテスト == "陽性（右）" || "陽性（左）") {
    神経レベル = "中等度"
    → 神経根障害の可能性。施術方針に反映
}
else if (下肢放散痛 == "片側" || 両側) {
    神経レベル = "軽度"
    → 神経刺激症状あり。悪化監視が必要
}
else {
    神経レベル = "なし"
}
```

### Step 3: 慢性化リスク判定（STarT）

```
if (STarTスコア >= 6) {
    リスク = "高"
    → 心理社会的因子への介入必須
    → 説明・行動変容・患者教育を施術と同等に重視
}
else if (STarTスコア >= 4) {
    リスク = "中"
    → 心理社会的因子に注意。セルフエフィカシー向上を意識した説明を
}
else {
    リスク = "低"
    → 身体的アプローチ中心で対応可
}
```

> **【重要】未入力の扱い（2026-03-25 確定）:**
> - `C65`（STarT合計）数式: `=IF(COUNTA(C56:C64)=0,"",COUNTIF(C56:C64,"はい(1)"))`
> - Q1〜Q9 が全て空欄の場合、C65 は `""（空文字）` を返す
> - `logic_engine.js` の `toNum("")` は `null` を返し、`START_LOW = false` とする
> - **未入力 ≠ 低リスク確定** — 未評価として扱い、branch12（運動療法開始）ではなく branch14（慢性期汎用方針）へ分岐させる
> - 根拠: `toNum()` のコメント「null を返すことで『未入力』と『0点』を区別する」

### Step 4: 痛み強度判定（NRS）

```
if (NRS現在 >= 7) { 強度 = "高強度" }
else if (NRS現在 >= 4) { 強度 = "中等度" }
else { 強度 = "軽度" }
```

### Step 5: 機能障害レベル判定（RMDQ-10）

```
if (RMDQ >= 8) { 機能 = "重度障害" }
else if (RMDQ >= 4) { 機能 = "中等度障害" }
else { 機能 = "軽度障害" }
```

> **【重要】未入力の扱い（2026-03-25 確定）:**
> - `C52`（RMDQ合計）数式: `=IF(COUNTA(C42:C51)=0,"",COUNTIF(C42:C51,"はい(1)"))`
> - Q1〜Q10 が全て空欄の場合、C52 は `""（空文字）` を返す
> - `toNum("")` は `null` → `RMDQ_MILD / MOD / SEVERE` は全て `false`
> - **未入力 ≠ 軽度確定** — 未評価として扱う（STarT と同じ方針）

### Step 6: 動作評価まとめ判定（重症度ベース）

4方向すべて（前屈・後屈・右側屈・左側屈）を対象に重症度を集計する。
パターン分類（屈曲優位など）は施術者判定欄（セクションJ）で補記する。

```
// 判定対象: 前屈(C76), 後屈(C77), 右側屈(C78), 左側屈(C79)

if (著明制限 が1つ以上) {
    動作評価まとめ = "重度制限型"
    → 患者説明: 「動作が大きく制限されています」
}
else if (中等度制限 が1つ以上 || 軽度制限 が2つ以上) {
    動作評価まとめ = "中等度制限型"
    → 患者説明: 「複数方向または中程度の制限があります」
}
else if (軽度制限 が1つ) {
    動作評価まとめ = "軽度制限型"
    → 患者説明: 「一部の方向に軽い制限があります」
}
else {
    動作評価まとめ = "正常"
    → 患者説明: 「動作範囲に特に問題は見られません」
}

// GAS数式（C81）:
// =IF(COUNTIF(C76:C79,"著明制限")>=1,"重度制限型",
//    IF(OR(COUNTIF(C76:C79,"中等度制限")>=1,COUNTIF(C76:C79,"軽度制限")>=2),"中等度制限型",
//       IF(COUNTIF(C76:C79,"軽度制限")>=1,"軽度制限型","正常")))
```

**設計根拠:**
- 全項目 正常 → 明示的に "正常" を返す（旧仕様は else="軽度制限型" のバグあり）
- 著明制限1つで即 重度制限型（1項目でも著明なら臨床的に重大）
- 複数異常 or 中等度1つ → 中等度制限型（患者説明上の自然な閾値）
- 軽度1項目 → 軽度制限型（経過観察レベル）
- 将来モジュール（頸部・膝）でも同一ロジック・同一COUNTIF構造を流用可能

### Step 7: 転倒リスク判定（移乗動作）

```
if (立ち上がり != "自立" || 歩行 == "中等度/著明障害") {
    転倒リスク = "高"
    → 転倒予防・ADL視点を方針に追加
}
else if (段差昇降 == "困難/不可" || 床からの立ち上がり == "困難/不可") {
    転倒リスク = "中"
}
else {
    転倒リスク = "低"
}
```

### Step 8: フラグ集計（慢性期・既往含む全フラグ）

発症期間・既往歴から慢性期フラグを追加する。Step 9 の判定マトリクスで使用。

```
// Step 8 固有フラグ（Steps 1〜7 のフラグに追加）

FLAGS.ACUTE     = 発症からの期間 == "2週未満"
FLAGS.SUBACUTE  = 発症からの期間 == "2〜6週" || "6週〜3か月"
FLAGS.CHRONIC   = 発症からの期間 == "3か月以上"
FLAGS.RECURRENT = 既往歴（腰部）== "あり（複数回）" || "手術歴あり"

// 実装: logic_engine.js > readAndComputeFlags()
```

### Step 9: 総合方針判定（判定マトリクス完全版）

Step 1〜8 のフラグを優先順位に従って評価し、治療優先方針テキストを C95 に出力する。

```
優先順位:
 1. CAUDA          → 【緊急】馬尾症候群疑い
 2. REDFLAG        → 赤旗あり — 医療連携を優先
 3. NERVE_SEVERE   → 神経症状重度 — 医療連携・神経所見優先
 4. NERVE_MOD      → 神経根障害疑い — 神経所見優先
 5. START_HIGH + NRS中〜高  → 行動変容・説明優先
 6. START_HIGH + NRS低      → 行動変容優先・段階的機能改善
 7. FALL_HIGH      → ADL・転倒予防優先
 8. NRS_HIGH + RMDQ中〜重度 → 疼痛管理優先 → 機能改善
 9. NRS_HIGH       → 疼痛管理優先
10. RMDQ_SEVERE    → 機能障害対応優先
11. ACUTE + NRS_MID → 急性期管理優先
12. (SUBACUTE|CHRONIC) + START_LOW + !NRS_HIGH → 機能改善・運動療法開始
13. CHRONIC + START_HIGH → 生活指導・行動変容優先
14. CHRONIC        → 機能改善・セルフケア習慣化
15. else           → 機能改善・セルフケア優先（標準方針）

// 実装: logic_engine.js > judgeOverallPolicy()
```

### Step 10: 自動生成コメント（K. セクション C99〜C106）

フラグと生値から 8 種のコメントを生成し K. セクションに書き込む。

```
出力先:
  C99  — 評価まとめ（施術者向けサマリー：判定・スコア・主要所見）
  C100 — 注意すべき所見（フラグベース：緊急 > 赤旗 > 神経 > STarT ...）
  C101 — 初回説明の方向性（患者への説明方針）
  C102 — 施術の優先順位（施術手順の提案）
  C103 — セルフケア・運動療法の方向性
  C104 — 再評価時に見るべきポイント（数値目標付き）
  C105 — 患者さんへの説明文（患者向け要約）
  C106 — 医療連携を考えるべき条件（★今回の評価 + 一般基準）

// 実装: logic_engine.js > generateComments()
// 将来モジュール: 頸部・膝固有フラグを追加するだけで流用可能
```

## 総合判定マトリクス（治療優先方針）

| 条件組合せ | 自動推奨方針 |
|---|---|
| 赤旗 or 馬尾 | **医療連携優先** |
| 神経レベル=重度 | **神経所見優先（医療連携検討）** |
| 神経レベル=中等度 | **神経根対応優先** |
| STarT高 + NRS中〜高 | **行動変容・説明優先** |
| STarT高 + NRS低〜中 | **行動変容・段階的機能改善** |
| 機能重度障害 + 転倒リスク高 | **ADL・転倒予防優先** |
| NRS高 + 機能中等度以上 | **疼痛管理優先 → 機能改善** |
| 発症2週未満 + NRS中 | **急性期管理（安静・保護）** |
| 発症6週以上 + STarT低 | **機能改善・運動療法開始** |
| 発症3か月以上 + STarT低〜中 | **機能改善+セルフケア習慣化** |
| 発症3か月以上 + STarT高 | **生活指導+行動変容+段階的運動** |

## 判定優先順位（競合時）

複数フラグが同時に立つ場合、以下の優先順位でコメントを選択する。

```
1. FLAG_CAUDA           （最優先・緊急）
2. FLAG_REDFLAG         （医療連携必須）
3. FLAG_NERVE_SEVERE    （神経所見優先）
4. FLAG_NERVE_MOD       （神経所見注意）
5. FLAG_START_HIGH      （心理社会的介入優先）
6. FLAG_FALL_HIGH       （転倒予防優先）
7. FLAG_NRS_HIGH        （疼痛管理優先）
8. FLAG_RMDQ_SEVERE     （機能障害対応優先）
9. FLAG_MOTION_SEVERE   （動作改善優先・重度）
9b. FLAG_MOTION_MOD    （動作改善優先・中等度）
10. FLAG_CHRONIC        （慢性期管理）
11. その他              （標準方針）
```

## 再評価時の変化判定

| 評価 | 前回比較 | 判定 |
|---|---|---|
| NRS | -2以上 | 有意な改善 |
| NRS | +2以上 | 悪化傾向（方針見直し） |
| RMDQ | -3以上 | 有意な改善 |
| PSFS | +2以上 | 有意な改善（MCID達成） |
| STarT | 低下 | 慢性化リスク低減 |
| STarT | 上昇 | 心理社会的因子悪化（注意） |

## Phase 5 向け AI判定連携設計

Layer 3（Claude API）に渡すデータ構造案：

```json
{
  "evaluation_id": "E0001",
  "patient_age": 52,
  "sex": "女性",
  "onset_duration": "3か月以上",
  "nrs": { "current": 6, "worst": 8, "best": 3 },
  "rmdq_score": 6,
  "start_score": 5,
  "start_risk": "中リスク",
  "psfs_avg": 4.5,
  "red_flags": 0,
  "nerve_level": "なし",
  "motion_pattern": "屈曲優位制限型",
  "fall_risk": "低",
  "flags": ["FLAG_START_MID", "FLAG_NRS_HIGH", "FLAG_CHRONIC"],
  "rule_based_judgment": "行動変容・疼痛管理優先",
  "clinician_judgment": "機能改善優先",
  "free_text": "デスクワーク8時間。寝返りで目が覚める。"
}
```

このJSONを `AI判定用出力` シートに書き込み、GASからClaude APIを呼び出す。
