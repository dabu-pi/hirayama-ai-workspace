## 全体臨床フロー（症状共通）

```
受付 → 患者マスタ登録（初回のみ）
  ↓
症状確認 → 使用する評価モジュールを選択
  Phase 1: 腰痛評価モジュール
  （将来 Phase 2〜: 頸肩 / 膝 / 姿勢 / 高齢者機能）
  ↓
【共通共通】赤旗スクリーニング → NRS → PSFS
  ↓
【症状別】症状固有の評価（腰痛ならRMDQ・STarT・動作評価等）
  ↓
自動生成コメントを確認 → 患者に説明
  ↓
記録確定 → 評価履歴に保存
  ↓
施術 → セルフケア指導 → 再評価予定を伝える
```

## 1. 初診での使い方（腰痛モジュール）

### フロー概要（所要時間：10〜15分）

```
受付 → 問診票記入（紙）
  ↓
施術者がシートを開く
  ↓
患者マスタに患者登録（〜2分）
  ↓
評価入力シートを開き、基本情報入力（〜2分）
  ↓
赤旗スクリーニング（口頭確認しながら入力：〜3分）
  ↓
NRS・RMDQ・STarT・PSFS を聞き取り入力（〜5分）
  ↓
動作評価・神経症状確認（〜3分）
  ↓
自動生成コメントを施術者が確認
  ↓
コメントを参照しながら患者に説明
  ↓
「記録確定」ボタン → 評価履歴に保存
```

### 初診で特に重視すること

| ポイント | 理由 |
|---|---|
| 赤旗を最初に確認する | 施術禁忌・紹介が必要な状態を早期に発見する |
| STarTを必ず入力する | 慢性化リスクが高い患者への対応を初回から変える |
| PSFSの活動名を患者の言葉で書く | 「どうなりたいか」を患者と共有する最初のステップ |
| 患者向け説明文を参照しながら話す | 評価結果→説明の一貫性を保つ |
| 再評価予定日をその場で決める | 「また来る理由」「継続の見通し」を共有する |

## 2. 再診での再評価の使い方

### 再評価のタイミング目安

| 発症からの期間 | 再評価タイミング |
|---|---|
| 急性期（2週未満） | 1〜2週後（症状変化が速いため） |
| 亜急性期（2〜6週） | 2〜3週後 |
| 慢性期（6週〜） | 3〜4週後、または症状変化があった時 |
| 最終評価（卒業前） | 卒業判断の直前 |

### 再評価での確認優先事項

1. NRS・RMDQ・PSFS の数値変化（前回比）
2. STarTの変化（慢性化リスクが下がっているか）
3. 新たな神経症状がないか（必ず毎回確認）
4. セルフケア・運動療法の実施状況
5. 患者の「できたこと」「困っていること」の変化

### 再評価時の施術方針変更トリガー

| 状況 | 対応 |
|---|---|
| NRS +2以上 / 下肢症状新出 | 施術内容を見直し・医療連携検討 |
| RMDQ -3以上改善 | 機能改善フェーズへ移行 |
| STarTスコア上昇 | 心理社会的介入を強化 |
| PSFS +2以上 | 次の目標活動の設定 |
| 4〜6週で改善なし | 施術方針の見直し + 医療連携検討 |

## 3. 卒業設計での使い方

### 卒業判断基準（目安）

| 指標 | 卒業の目安 |
|---|---|
| NRS | 2以下 または 患者が満足している状態 |
| RMDQ-10 | 3点以下（軽度障害以下） |
| PSFS | 設定した目標スコアに到達（通常7〜8点） |
| 動作 | 主要動作が「正常」または「軽度制限」以下 |
| 本人の自己評価 | 「自分で管理できる」と感じている |

### 卒業評価シートの使い方

- 評価区分を「最終評価」に設定して記録確定
- コメント「再評価時に見るべきポイント」を患者に渡す形で印刷 or 口頭説明
- 「再燃した時のサイン」と「その時の対処法」を必ず伝えて終了

### 卒業後の関係維持（接骨院メニューへの接続）

| 卒業後のシナリオ | 推奨アクション |
|---|---|
| 腰痛は改善・再発予防希望 | ジム会員コース / 月1回メンテナンス施術へ誘導 |
| 不安が残っている | 「困ったらいつでも来てください」と伝え再診入口を明示 |
| 完全回復・継続不要 | セルフケアパンフを渡し、再燃時の受診方法を伝える |

## 4. 接骨院メニューへの組み込み方

### 評価シートと施術メニューの連動

| 評価結果 | 推奨メニュー |
|---|---|
| 急性期 + NRS高 | 保険施術 + 急性期集中ケア |
| 亜急性〜機能改善期 | 保険施術 + 手技自費（慢性ケア50分） |
| 慢性期 + STarT低 | 手技自費 + ホームエクササイズ指導 |
| 慢性期 + STarT高 | 手技自費 + 行動変容プログラム |
| 機能改善後 | ジム誘導（再発予防コース） |
| 高齢者 + 転倒リスク | バランス・ADL改善コース |

### 施術時間ごとの活用

| タイミング | 評価シートの活用 |
|---|---|
| 施術前（2〜3分） | 評価コメント確認 → 今日の施術の優先順位を頭に入れる |
| 施術中 | 動作評価・セルフケア指導の参考にする |
| 施術後（2〜3分） | セルフケアコメントを参照して指導 / 次回評価予定を伝える |

## 5. 紙運用 → タブレット運用への移行計画

### フェーズ別移行計画

| フェーズ | 形態 | 内容 |
|---|---|---|
| Phase 1〜2 | PC入力 | 施術者がPCでスプレッドシートを入力 |
| Phase 3〜4 | PC入力 + 印刷 | 評価コメントをA4で印刷し患者に渡す |
| Phase 5 | タブレット入力 | 施術者がタブレットで入力（Google Sheetsアプリ） |
| Phase 6 | タブレット入力 + 患者入力 | 待合室でPSFS・RMDQ・STarTを患者が直接入力 |

### タブレット入力最適化のための設計原則（Phase 6向け準備）

- すべての入力欄をプルダウン or チェックボックスにする（テキスト入力を極限まで減らす）
- 1画面に収まる項目数にする（縦スクロールを最小化）
- セクションごとにタブ分け（基本情報→赤旗→NRS→RMDQ→STarT→動作→確認）
- 「次へ」ボタンで順番に進む構造にする（ユーザビリティ）

### タブレット入力移行時の注意点

- 患者データは施設のGoogleアカウントで管理（個人アカウント不可）
- セッションタイムアウトの設定（プライバシー保護）
- 入力内容の確認ステップを必ず設ける

## 6. 評価者間差を減らすための設計

### 判定基準表（施術者向け教育資料）

各評価項目について、「この状態をどう入力するか」を明確化する。

| 評価項目 | 基準の明確化方法 |
|---|---|
| 動作評価（前屈等） | 角度目安をセルコメントに記載 + 写真・動画リンク（将来） |
| NRS | 「今の痛みを0=痛みなし、10=想像できる最大の痛み」として統一説明文を用意 |
| RMDQ-10 | 「今日の状態」で回答する（過去や平均ではない）ことをセルに注記 |
| STarT簡易版 | 「過去1週間のことについて」と明記。迷ったら「はい」に倒す |
| SLRテスト | 「下肢へ走る痛み・しびれが出た場合=陽性」と定義を統一 |
| 転倒リスク | 「立ち上がりに補助が必要」=見守り要として統一 |

### スタッフ教育時の使い方

1. **入職時:** SPEC.md + SHEET_DESIGN.md を読み、評価の目的・各項目の意味を理解する
2. **ロールプレイ:** 患者役・施術者役を交代しながら評価入力の練習（5症例以上）
3. **ケースカンファレンス:** 週1回、評価結果を施術者間で共有し認識合わせ
4. **評価基準の更新:** 現場で迷った判定基準は月1回まとめてシート更新

## 8. 安全管理とリスク管理

### 評価シートにおける安全ネット

| リスク | 対処設計 |
|---|---|
| 馬尾症候群見逃し | 排尿障害・会陰部感覚異常を赤旗最上位に配置。入力時に赤色アラート |
| 施術禁忌見逃し | 赤旗スコア>=1 で「医療連携を検討してから施術」の注意が自動表示 |
| 評価入力忘れ | 必須項目（NRS・赤旗・評価区分）未入力時は「記録確定」ボタンを無効化 |
| データ消失 | 評価履歴シートへの転記はコピー動作。元データは入力シートに保持 |
| 誤入力 | クリアボタンで再入力可。転記前に確認ダイアログ表示 |

### 医療・法的リスクへの配慮

- 評価シートは「接骨院での評価ツール」であり、医師の診断に代わるものではない
- 柔道整復師の業務範囲内の評価項目に限定する
- 「医療連携を考えるべき条件」は常時表示し、施術者の判断責任を明確にする
- 患者データの保管・管理はGoogleアカウントの利用規約・個人情報保護法に準拠する
