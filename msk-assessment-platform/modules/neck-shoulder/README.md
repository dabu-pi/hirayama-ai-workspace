## 1. このモジュールの役割

頚部痛・肩こりを主訴とする患者に対して、**入口整理から初回方針決定**までを担う。

腰痛モジュールと同様に「入口評価」として機能し、必要に応じて運動療法初回評価へ接続する。

軽量版を優先する理由：
- 初回来院時に過剰な評価をしない（CLINICAL_FLOW.md §3 参照）
- 日常臨床で継続使用できる項目数に絞る
- 実運用しながら項目を追加・修正していく

## 5. 将来の拡張方針

### 5-1. 症状分類の細分化（将来）

現時点は「頚肩こり」として一括管理する。
実臨床テストを経て、以下の分岐拡張を検討する。

| 分類 | 分岐条件 | 拡張内容 |
|---|---|---|
| 首痛優位 | 主訴=頚部痛 + ROM制限あり | 頚部痛特化のコメントセット |
| 肩こり優位 | 主訴=肩こり + ROM正常 | 筋・筋膜・姿勢中心の評価追加 |
| 頭痛併発 | 主訴に頭痛含む | 頭痛特化スクリーニング項目追加 |
| 上肢症状あり | FLAG_NS_RADICULOPATHY | 神経根評価の詳細版に分岐 |

### 5-2. 頚部神経症状の詳細版（将来）

FLAG_NS_RADICULOPATHY が立った場合、以下の詳細評価に分岐できる構造を想定する。

- 頚椎レベル別デルマトーム確認（C5/C6/C7/C8）
- 反射テスト（上腕二頭筋腱反射・腕橈骨筋反射）
- 筋力テスト（肘屈曲・手関節伸展・握力）

これは初期評価の軽量版とは別の「神経根詳細評価シート」として追加する予定。

### 5-3. シートが重くなった場合の分割方針

- 現時点は1シートで完結（軽量版）
- 評価項目が30行を超えたら「基本版 / 詳細版」で条件分岐して非表示を活用
- さらに重くなった場合は「頚肩こり_詳細評価」シートを別途追加

### 5-4. 他部位追加時の共通入口の使い方

共通_初期評価シートは変更せず、以下のみ追加する。
1. `modules/<部位>/README.md` で仕様記述
2. `SHEET_DESIGN.md` に該当シートの列設計を追加
3. `EVIDENCE_MAP_<部位>.md` で根拠を整理
4. 初期評価サマリーに「部位モジュール種別」として行追加
