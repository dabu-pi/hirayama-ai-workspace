## 背景

平山接骨院は現在、`gas-projects/jyu-gas-ver3.1/` で保険請求管理を運用している。
戦略目標（自費転換率・ジム入会率・KPI）を計測するには、
**患者ごとの「ステップ通過状況」を記録するデータ構造が必要**。

このファイルでは、「何を記録すれば何が測れるか」を整理する。

---

## 追加すべき管理項目

以下の項目を患者単位で記録できる仕組みを作る。
（現行の保険請求システムに追加するか、別シートで管理するかは実装時に判断）

---

### 1. 保険来院

| 記録項目 | 内容 | 型 |
|---|---|---|
| 初回保険来院日 | 患者との接触開始日 | 日付 |
| 累計来院回数（保険） | 自費提案タイミング判断に使う | 数値 |
| 主訴 | 慢性腰痛・肩こり等 | テキスト |
| 慢性化フラグ | 初診時に慢性疼痛対象かどうか | 真偽値 |

**KPI計測への使い方:**
- 来院患者のうち「慢性化フラグON」の人数 = 自費提案候補数
- 月次でカウントし、提案率の分母として使う

---

### 2. 自費提案日

| 記録項目 | 内容 | 型 |
|---|---|---|
| 自費提案日 | 初回の自費メニュー案内日 | 日付 |
| 提案したメニュー | 8回プログラム / 単回 / 運動療法 | テキスト |
| 提案時の反応 | 興味あり / 保留 / 断り | 選択肢 |

**KPI計測への使い方:**
- 「提案した患者数 / 慢性化フラグ患者数」= 提案率
- 提案後の反応分布を見て、トークスクリプト改善に使う

---

### 3. 自費成約

| 記録項目 | 内容 | 型 |
|---|---|---|
| 自費成約日 | 実際に申込・支払いを受けた日 | 日付 |
| 成約メニュー | 何を購入したか | テキスト |
| 成約金額 | 実際の売上額 | 数値 |

**KPI計測への使い方:**
- 「成約件数 / 提案件数」= 成約率
- 「成約金額の月次合計」= 自費収益KPI（目標: +20万円/月）

---

### 4. 運動療法開始

| 記録項目 | 内容 | 型 |
|---|---|---|
| 運動療法開始日 | 初回の運動療法セッション日 | 日付 |
| セッション形式 | パーソナル / グループ / セルフ指導 | 選択肢 |

**KPI計測への使い方:**
- 自費患者のうち運動療法に移行した割合 = 運動療法転換率
- ジム体験案内の前段階として実績を追う

---

### 5. ジム体験

| 記録項目 | 内容 | 型 |
|---|---|---|
| ジム体験案内日 | ジム利用を案内した日 | 日付 |
| 体験実施日 | 実際に体験した日（未実施は空白） | 日付 |
| 体験後の反応 | 入会希望 / 検討中 / 不要 | 選択肢 |

**KPI計測への使い方:**
- 「体験した患者数 / 運動療法患者数」= ジム体験率
- 体験 → 入会の転換率を追う

---

### 6. ジム入会

| 記録項目 | 内容 | 型 |
|---|---|---|
| ジム入会日 | 実際に会員契約した日 | 日付 |
| 入会プラン | スタンダード / プレミアム | 選択肢 |
| 月額費用 | 実際の会員費 | 数値 |
| 退会日 | 退会した場合に記録 | 日付（空白可） |
| 継続月数 | 在籍月数を自動計算 | 数値（計算式） |

**KPI計測への使い方:**
- 「在籍会員数 × 月額費用の合計」= ジム月次定期収益
- 「退会月 - 入会月」= 継続月数 → 平均継続月数の計算に使う
- 月次退会率 = 「退会件数 / 月初在籍数」

---

## KPIダッシュボードとの接続イメージ

上記の記録項目を揃えると、以下のKPIが自動集計できる。

```
月次保険来院数
    ↓（慢性化フラグ÷来院数）
慢性化患者割合
    ↓（提案数÷慢性化患者数）
自費提案率
    ↓（成約数÷提案数）
自費成約率
    ↓（成約月次合計）
自費月次収益 → KPI目標（+20万円）との比較
    ↓（ジム体験率）
ジム体験率
    ↓（入会数÷体験数）
ジム入会率
    ↓（在籍数×月額）
ジム月次定期収益 → 将来の安定収益柱
```

## 実装ステップ（優先順位）

| ステップ | 内容 | 前提 |
|---|---|---|
| 1 | 現行GASシステムのシート構造を確認 | 既存システムとの整合確認 |
| 2 | 患者マスタに「慢性化フラグ」「主訴」を追加 | 最小コストで始める |
| 3 | 「自費提案記録シート」を追加（手入力） | 日次オペレーションに組み込む |
| 4 | 「ジム会員シート」を追加（入会・退会管理） | ジム会員費が確定してから |
| 5 | 月次KPIサマリーシートを自動生成 | 各シートの記録が安定してから |
