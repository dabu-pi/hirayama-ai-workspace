## このプロジェクトの一言要約

まずは **慢性腰痛・首肩こり** の患者導線を、
**手技50分前後・標準5,500円 / ジム会員優待4,700円** の自費メニュー中心で作る。
機械は当面 **保険治療の補助用** に限定し、
その後に **セルフケア → 運動療法 → ジム継続** へつなげる。
最初の収益目標は月次自費収益 **+20万円**。

基準仮説は **案C**
（慢性患者移行率 50% / 月平均来院回数 3.5回 / 主力手技価格 5,500円）。
到達目標は **案A**
（慢性患者移行率 50% / 月平均来院回数 4.0回 / KPI逆算は 5,500円基準）。

正式価格は **標準価格 5,500円 / ジム会員優待価格 4,700円** とし、
`KPI逆算` は当面 **5,500円基準** のまま運用する。

日常の数値更新は live スプレッドシートで行い、
ローカルには `scripts/sync-hirayama-inputs.mjs` で最新の手入力値を pull 同期する。
構造変更は Claude Code、数値変更はユーザー手入力の分担で運用する。

スプレッドシートは
`保険・来院前提 → 数値前提 → 価格設定 → KPI逆算 → KPI目標`
の順で役割を分け、`KPI逆算` は考えるシート、`KPI目標` は見るシートとして運用する。
JBIZ-04 側に日次入力シートは新設せず、日次入力の正本は `gas-projects/jyu-gas-ver3.1/` の【毎日記録】来店管理施術録ver3.1 に寄せる。
JBIZ-04 は月次KPI / KPI逆算 / 経営判断の正本として使う。

## JBIZ-04へ渡す月次7項目の計算定義案

| 月次項目 | 計算定義案 | 集計元 | JBIZ-04転記先 |
|---|---|---|---|
| 月次総保険売上 | 当月の `保険のみ + 保険+自費` 来院に紐づく保険請求額合計 + 窓口保険負担分 | 毎日記録ver3.1 の来院ヘッダ | `KPI目標` の `月次総保険売上_実績値` |
| 月次自費売上 | 当月の `自費売上額` 合計 | 毎日記録ver3.1 の来院ヘッダ | `KPI目標` の `自費売上_実績値` |
| 慢性候補人数 | 当月に `慢性候補フラグ=TRUE` となったユニーク患者数 | 毎日記録ver3.1 の来院ヘッダ | `保険・来院前提` 見直し用の月次確認値 |
| 手技自費患者数 | 当月に `自費メニュー区分=手技50分` を1回以上受けたユニーク患者数 | 毎日記録ver3.1 の来院ヘッダ | `KPI逆算` の移行率・患者数比較用月次値 |
| 手技回数 | 当月に `自費メニュー区分=手技50分` で計上された来院件数 | 毎日記録ver3.1 の来院ヘッダ | `KPI逆算` の `手技回数` 実績比較 |
| 自費平均単価 | `月次自費売上 ÷ 手技回数` を基本とし、当面は主力手技中心で確認 | 毎日記録ver3.1 の来院ヘッダ | `KPI目標` の `自費平均単価_実績値` |
| 次回予約あり件数 | 当月来院のうち `次回予約あり=TRUE` の件数 | 毎日記録ver3.1 の来院ヘッダ | 月次レビュー補助指標（当面はメモ管理） |

補足:
- `慢性候補人数` と `手技自費患者数` は件数ではなく**ユニーク患者数**を基本にする。
- `手技回数` と `次回予約あり件数` は**来院件数ベース**で集計する。
- `自費平均単価` は将来、メニュー別集計が安定したら `主力手技のみ` と `自費全体` を分けて持つ。
