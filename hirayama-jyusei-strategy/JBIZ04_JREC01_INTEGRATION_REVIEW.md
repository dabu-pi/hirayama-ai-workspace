**来院ヘッダに既に追加済みの経営管理列**

| 列名 | 値 | 状態 |
|---|---|---|
| 会計区分 | 保険のみ / 保険+自費 / 自費のみ | ✅ 実装済み |

**保険と自費の現行分離状況**

- 来院ヘッダの「来院合計」= **保険金額のみ**（初検料+再検料+相談支援料+明細合計）

- 施術明細シート（27列）= **保険専用** の部位別内訳データ
- 「自費明細」シート = **未実装**

**申請書連携**

- B案申請書生成: 本番稼働可能（2026-03-20確認済み）
- データソース: 施術明細（保険専用）→ NDJSON → Cloud Run → Excel申請書
- 来院ヘッダの自費列は申請書生成に**無関係**（分離されている）

## Claude Code が考える最適方針

### 基本姿勢

> **既存実装を壊さずに「積み上げる」設計。保険算定は一切触らない。**

現行の3層は**すでに2/3が実装済み**：

| 層 | 内容 | 現状 |
|---|---|---|
| 層1: 来院ヘッダ_統合 | 全来店記録（保険+自費フラグ付き）| ✅ 実装済み |
| 層2: 施術明細_保険 | 療養費支給申請書用の部位別明細 | ✅ 実装済み |
| 層3: 自費明細 | 自費売上分析用の明細 | ❌ 未実装 |

→ **今すぐ必要なのは「層3の追加」と「JBIZ-04 KPI細分化」の設計確定のみ。**

### 優先判断マトリクス

| 課題 | 緊急度 | 影響範囲 | 推奨 |
|---|---|---|---|
| 層1の入力UI（GAS患者画面への自費入力欄）確認 | 高 | 運用開始前提 | 最優先確認 |
| 層3（自費明細シート）新設 | 中 | 自費分析のみ | Phase 1 |
| JBIZ-04 KPI「延べ来院数」分離 | 中 | 将来予防 | Phase 1 |
| JBIZ-04 KPI「手技/その他」自費分離 | 中 | KPI精度 | Phase 1 |
| JREC-01→JBIZ-04 自動集計 | 低 | 運用効率 | Phase 2 |
| menu_id共通化 | 低 | 将来拡張 | Phase 3以降 |

## 危険点・見落とし

### 危険点（既存ロジック保護）

| リスク | 内容 | 対策 |
|---|---|---|
| 来院合計への自費混入 | `来院合計 = 保険金額のみ` を維持。自費は「自費売上額」列に分離 | **絶対禁止。コードを変更しない** |
| 療養費支給申請書への影響 | 申請書生成は施術明細（保険専用）を参照。来院ヘッダの経営管理列は無視 | 施術明細の列構成・データを変更しない |
| 延べ来院数の計算式変更 | JBIZ-04の `KPI逆算` で延べ来院数を使った計算式がある。列名変更・値変更は式を壊す | 列を**追加**する（既存列を変更・削除しない）|
| JBIZ-04シートの大改修 | 既存シートに大きな構造変更を加えるとユーザーの手入力が壊れる | 補助列・補助シートの追加で対応 |

### 見落としていた点（現行実装の確認が必要）

1. **来院ヘッダの経営管理列は「設計上ある」が、GAS患者画面（UI）で実際に入力できるか？**
   - Ver3_core.jsの `saveVisit_V3` が「自費売上額」「会計区分」「自費メニュー区分」を書き込む実装があるか要確認
   - UIに入力欄がなければ、院長は毎日スプレッドシートを直接編集する必要がある

2. **UI入力欄の追加は「保険算定UIに追記」か「別サイドパネル設置」か** — 患者画面（B2〜行52）はすでに保険入力で密な構成。自費入力欄を追加するには患者画面の行拡張か別エリア設置の設計が必要。

3. **「自費明細」シートを追加する場合、どのタイミングで書き込むか**
   - `saveVisit_V3` のステップ⑤として追加するのが最も整合的
   - ただし「自費売上額」が1来店で複数メニューある場合の設計が未決（例: 手技50分 + セルフケア指導 を同日実施した場合）

3. **KPI集計のデータソース問題**
   - JBIZ-04の `KPI目標!D6`（自費売上）は院長の手動入力
   - これを自動化するには、来院ヘッダの「自費売上額」を月次集計するスクリプトが必要
   - 自動化しないなら現行の手動運用を維持し、列名変更・KPI細分化だけ対応すれば十分

## 採用すべき結論

### 方針: ChatGPT案を採用。ただし「段階的実装」で現行を壊さない

**理由:**
- ChatGPT案の大筋（3層保存・KPI細分化・延べ来院数分離）は現行設計と整合している
- 層1・層2はすでに動いている。変更は最小限（層3追加 + JBIZ-04 KPI調整）
- 保険算定ロジック・申請書生成には一切手を加えない

### 今すぐ確定すべき点

1. **GAS患者画面（UI）に自費入力欄があるか** → Ver3_core.js で確認（今回のタスク外）
2. **自費明細の粒度** → 1来店1行（会計区分単位）か 1来店×メニュー行か
3. **「延べ来院数」列の扱い** → JBIZ-04で「保険延べ来院数」に改名 or 別列追加か

### 今すぐ実施しなくてよいこと

- menu_id共通化（将来課題）
- JREC-01→JBIZ-04の自動集計（Phase 2）
- KPI実績履歴の大規模リデザイン（列追加のみで対応）
