# スプレッドシート管理表 使い方ガイド

> 対応スクリプト: `gas_spreadsheet_setup.js`
> 対応設計書: `SHEET_DESIGN.md`
> 最終更新: 2026-03-10

---

## 1. セットアップ手順（最初の1回だけ）

1. **Googleスプレッドシートを新規作成する**
2. メニューから **拡張機能 > Apps Script** を開く
3. エディタに表示されているデフォルトコードを全選択して削除する
4. `gas_spreadsheet_setup.js` の内容を全文コピーして貼り付ける
5. 上部の関数プルダウンで **`createHirayamaSpreadsheet`** を選択して「▶ 実行」
6. 権限承認のダイアログが出たら「許可」する
7. 完了ダイアログにURLが表示されたら → そのURLを開く

> 実行には30秒〜1分ほどかかります。

## 2. シートの説明と入力場所

| シート名 | 役割 | ユーザーが入力するもの |
|---|---|---|
| **全体ダッシュボード** | 現在地の一覧確認 | 最終更新日・今週のメモのみ |
| **ロードマップ進捗** | Phase1〜6の進み具合 | ステータス（4択）・進捗率（%） |
| **数値前提** | 固定費とベースライン収入 | 固定費9項目・保険売上・患者数 |
| **価格設定** | 全12メニューのマスタ管理 | 一般料金・ジム会員料金・KPI集計対象・確定状況 |
| **KPI目標** | 転換率の目標と必要件数の逆算 | 保険来院数・慢性割合・各転換率 |
| **未確定項目** | 決めるべき事項の管理 | 決定状況（3択）・確定値・備考 |

## 3. セルの色の意味

| 色 | 意味 |
|---|---|
| 🟡 黄色 | **入力してください**（ここだけ触ってOK） |
| 🔵 水色 | 自動計算（触らない） |
| ⬜ グレー | 見出し・ラベル（触らない） |
| 🟢 緑文字 | 確定済み |
| 🔴 赤文字 | 未確定・要対応 |

---

## 4. 使い始めの入力順序

```
Step 1: 「数値前提」シートに固定費・保険ベースラインを入力
         → ダッシュボードの固定費カバー率が動き始める

Step 2: 「価格設定」シートで各メニューの一般料金・ジム会員料金を確認・修正
         → KPI目標シートのブロックBに単価が自動反映される

Step 3: 「ロードマップ進捗」シートでステータスと進捗率を更新
         → ダッシュボードの戦略完成度が更新される

Step 4: 「未確定項目」シートで確定したものを「確定済み」に変える
         → ダッシュボードの数値確定度が上がる

Step 5: 「KPI目標」シートに転換率の目標を入力
         → 必要成約件数・達成ギャップが逆算される
```

## 5. よくある問題と対処

**Q: `#NAME?` エラーが出る（特にダッシュボードのTop3欄）**
FILTER 関数が使えない旧バージョンのスプレッドシートで発生します。
以下の代替数式をセルに **Ctrl+Shift+Enter** で入力してください（Top1の例）:

```
=IFERROR(INDEX('未確定項目'!B$4:B$12,
  SMALL(IF(('未確定項目'!D$4:D$12="高")*('未確定項目'!F$4:F$12="未確定"),
    ROW('未確定項目'!B$4:B$12)-ROW('未確定項目'!B$4)+1), 1)), "（なし）")
```
Top2は末尾の `1` を `2` に、Top3は `3` に変えてください。
> `gas_spreadsheet_setup.js` の208〜213行目にもコメントで記載しています。

---

**Q: 行を間違えて追加・削除してしまった**
スクリプトを再実行すると新しいシートが追加されますが、既存のシートは上書きされません。
既存シートを手動で削除してから再実行するか、元の数式を `SHEET_DESIGN.md` を参照して手動で直してください。

---

**Q: 数式が「要入力」と表示される**
入力が必要なセルが空白のままです。黄色セルに値を入力してください。

## 7. Claude Code との運用分担ルール

### 基本方針

| 役割 | 担当 |
|---|---|
| Sheets は**運用画面**（データ入力・確認） | ユーザー |
| Sheets の**構造変更**（列追加・数式・入力規則・注意書き） | **Claude Code 主導** |
| ローカルドキュメントの更新・commit・push | Claude Code |

### ユーザー直編集の最小化ルール

- 黄色セル（手入力欄）への値入力はユーザーが行う
- **列追加・行削除・数式変更・シート追加はユーザーが単独で行わない**
  → 必ず Claude Code に依頼し、GAS または手順提示で実施する
- やむを得ずユーザーが手動変更した場合は、次セッション開始時に必ず申告する

### 手動変更が発生した場合の手順

```
1. 次セッション開始時に「手動でXXを変更した」と申告
2. Claude Code が Sheets 実物を確認して差分を把握
3. ローカルドキュメント（STATUS.md / SHEET_DESIGN.md）に反映
4. commit / push して整合済みにしてから次タスクへ進む
```

> 差分がある状態で次タスクへ進まない。これが Sheets ↔ docs 整合の基本ルール。
