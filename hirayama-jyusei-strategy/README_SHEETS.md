## シートの考え方

- `保険・来院前提`: 来院構造と保険前提を入れる
- `数値前提`: 固定費と売上前提を入れる
- `価格設定`: 単価前提と主力手技を決める
- `KPI逆算`: 設計用。必要件数と導線を考える
- `KPI目標`: 当月管理用。目標と実績を比較する
- `KPI実績履歴`: 月次保存用。1か月=1行で残す
- `KPI逆算履歴`: 前提変更履歴用。節目だけ残す

## 色分け

- 青系: 手入力セル
- 緑系: 自動計算セル
- 灰系: 固定説明セル
- 黄系: 仮値・未確定値
- 赤系: 警告

## 保護方針

- `全体ダッシュボード`: 主要表示セルは warning-only 保護
- `保険・来院前提`: 自動計算セルのみ warning-only 保護
- `数値前提`: 自動計算セルのみ warning-only 保護
- `価格設定`: 集計・警告セルのみ warning-only 保護
- `KPI逆算`: 逆算と警告セルを warning-only 保護
- `KPI目標`: 目標値列と差異・達成率列を warning-only 保護
- `KPI実績履歴`: 見出しと説明行を warning-only 保護
- `KPI逆算履歴`: 見出しと説明行を warning-only 保護
- `未確定項目`: 集計セルのみ warning-only 保護

## 入力順

1. `保険・来院前提`
2. `数値前提`
3. `価格設定`
4. `KPI逆算`
5. `KPI目標`
6. 月末に `KPI実績履歴`
7. 前提変更時に `KPI逆算履歴`

## 主力手技フラグ

- `価格設定` の `K列` が `主力手技フラグ`
- `TRUE` は1行のみ想定
- `価格設定` の警告セルは `B18`
- `KPI逆算` の警告セルは `C7`
- 0件 / 複数件のときは価格参照を `5,000円` にフォールバックしつつ警告を出す

- `保険・来院前提`
- `数値前提`
- `価格設定`

- `未確定項目`

`KPI逆算 / KPI実績履歴 / KPI逆算履歴 / 全体ダッシュボード` は基本的に見るシートで、主要数式セルは誤編集しにくいよう warning-only 保護を使う前提です。

## live と local の役割

- スプレッドシートは日常数値の正本として使う
- ローカルは構造・コード・設計書の正本として使う
- 数値更新はユーザーが live スプレッドシートへ手入力する
- 構造変更は Claude Code 側で行う
- live の最新手入力値は `scripts/sync-hirayama-inputs.mjs` で local へ pull 同期する
- snapshot JSON は最新状態の控えとして Git 管理できる形で保存する

## 入力値同期

- スクリプト: `scripts/sync-hirayama-inputs.mjs`
- 定義ファイル: `scripts/hirayama-input-sync-config.mjs`
- 保存先: `hirayama-jyusei-strategy/data/inputs_snapshot.json`
- 同期方式: `Sheets -> local` の pull 専用
- 同期対象: `保険・来院前提 / 数値前提 / 価格設定 / KPI目標 / 未確定項目` の手入力セルのみ

### 実行例

```powershell
$env:AIOS_SERVICE_ACCOUNT_PATH='C:\hirayama-ai-workspace\workspace\secrets\credentials.json'
node .\scripts\sync-hirayama-inputs.mjs --dry-run
node .\scripts\sync-hirayama-inputs.mjs
```

- `--dry-run`: 保存せずに同期内容を標準出力へ表示
- 通常実行: snapshot JSON を上書き保存
- 将来は Windows タスクスケジューラで定期実行しやすいよう、コマンド 1 本で完結する形にしている

## 保険関連の定義

- `窓口単価` = 患者が窓口で支払う平均額
- `総保険売上単価` = 窓口負担 + 保険者支払分の1来院あたり合計
- `月次総保険売上` = 月の総保険収入（窓口 + 保険者）

## KPI目標 の月次運用

### 8項目

| KPI | 目標値 | 実績値 | 月末確定タイミング |
|---|---|---|---|
| 月間売上 | 自動 | 自動 | 月次総保険売上と自費売上が確定した時点 |
| 月次総保険売上 | `数値前提` 参照 | 手入力 | レセプト見込みと窓口集計が固まった時点 |
| 自費売上 | `KPI逆算` 参照 | 手入力 | 月内の自費売上集計後 |
| 自費売上比率 | 自動 | 自動 | 月間売上と自費売上が確定した時点 |
| 新患数 | 手入力 | 手入力 | 月末集計時 |
| 延べ来院数 | `保険・来院前提` 参照 | 手入力 | 月末集計時 |
| 保険平均単価 | 自動 | 自動 | 月次総保険売上と延べ来院数が確定した時点 |
| 自費平均単価 | `KPI逆算` 参照 | 手入力 | 自費売上と実績ベース単価が固まった時点 |

### 転記ルール

- `KPI目標` は当月の管理シートとして使う
- 月末または翌月初に `KPI実績履歴` へ値貼り付けで保存する
- 自動転記は後段対応でよく、当面は手動転記で運用する

## KPI逆算履歴 の保存ルール

- 毎月保存しない
- 価格方針変更、導線変更、KPI定義変更など節目だけ1行追加する
- `KPI逆算` は常に最新、`KPI逆算履歴` は比較用のスナップショットと考える
