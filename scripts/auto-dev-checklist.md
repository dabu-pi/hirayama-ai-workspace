# auto-dev-checklist.md — 自動開発モード チェックリスト

各フェーズで Claude が確認すべき項目の一覧。
STOP 条件に該当する場合は即座に実装を中断して人間に確認を求める。

---

## Phase 1: 着手前（PLAN フェーズ）

### 環境確認
- [ ] `git pull origin master` を実行した
- [ ] 作業ディレクトリが正しい（`workspace/` or `claude-sandbox/`）
- [ ] `ROADMAP.md` で今回のタスクを確認した
- [ ] `CLAUDE.md` で仕様・禁止事項を確認した

### タスクの明確化
- [ ] タスクの完了条件が明確である
- [ ] 変更するファイルが1〜3ファイル以内に収まる見込みがある
- [ ] 仕様が不明な部分がない（不明なら REPORT & STOP）
- [ ] このタスクは `ROADMAP.md` に記載されている（未記載なら REPORT & STOP）

### セキュリティ確認
- [ ] 認証情報（APIキー・パスワード・トークン）を扱わない
- [ ] 外部 API への本番 POST を行わない
- [ ] 今回の変更が `workspace/` であれば機能ブランチを作成した

**ブランチ名:**
```
git checkout -b [feature|fix|docs|refactor|hotfix]/[短い説明]
```

---

## Phase 2: 実装中（DO フェーズ）

### 実行ルール（Phase によって使うコマンドが異なる）

**Phase1（rwl 直接）:**
- [ ] スクリプト・テストの実行は `rwl` を経由している
- [ ] `python main.py` を直接実行していない

**Phase2（adr 経由）:**
- [ ] スクリプト・テストの実行は `adr -Cmd "..."` を経由している
- [ ] `-AutoNote` で自動 note が生成されている（手動 note は補足のみ）

**共通:**
- [ ] 変更は1ファイルずつ `git diff` で差分確認している

### 進捗チェック（1ファイル変更ごと）
- [ ] 変更内容が PLAN の意図と一致している
- [ ] 変更範囲が想定外に広がっていない（広がったら REPORT & STOP）
- [ ] エラーが出た場合、原因を特定してから次へ進んでいる

### STOP 判定（いずれか該当したら即停止）
- [ ] **PASS**: 認証情報の操作が発生していない
- [ ] **PASS**: 本番 API POST が発生していない
- [ ] **PASS**: 同じエラーが3回繰り返していない
- [ ] **PASS**: 仕様不明な箇所がない

---

## Phase 3: 仕上げ（CHECK → ACT フェーズ）

### ログ確認
- [ ] `logs/run/` に実行ログが生成された
- [ ] `logs/error/` にエラーファイルが存在しない（存在する場合: `aerr` で確認）
- [ ] テストが存在する場合、すべてパスした

**Phase2 補足:**
```powershell
dstat       # ログ件数・エラー件数・最新メモを一覧表示
aerr        # エラー内容を整形表示（失敗時）
aerr -ListAll  # 全エラーログの一覧表示
```

### git 確認
- [ ] `git diff` で変更差分が意図通りである
- [ ] コミット対象に認証情報が含まれていない

**Phase1:**
- [ ] `git add .` を使わず、ファイルを個別指定した
- [ ] `git commit -m "..."` でコミットした

**Phase2（gsc 経由）:**
- [ ] `gsc -Message "..." -Files @(...) -Push` で安全コミットした
- [ ] `gsc` のセキュリティチェックが全て PASS した

### コミットメッセージ確認
```
[プレフィックス]: [変更内容の要約]

プレフィックス一覧:
feat     新機能
fix      バグ修正
docs     ドキュメント
refactor リファクタリング
test     テスト
chore    設定・依存・整理
```

### 記録
- [ ] `note` コマンドで今サイクルの記録を保存した（最低1件）
- [ ] 重要な決定・バグ原因は `-Tag decision` or `-Tag bug` で記録した
- [ ] `ROADMAP.md` の完了タスクのステータスを ✅ に更新した

### 次サイクルへの引き継ぎ
- [ ] `## NEXT` セクションに次のタスクを明記した
- [ ] 残課題・懸念点は `-Tag todo` or `-Tag risk` で note に記録した

---

## STOP 時の出力テンプレート

```
## STOP — 理由: [条件名]

**状況:** （現在何をしようとしていたか）

**問題:** （なぜ止まったか、具体的に）

**確認事項:**
1. 〇〇について教えてください
2. △△を実施してよいですか？

**再開手順:** （確認後にどうすればループを再開できるか）
```
