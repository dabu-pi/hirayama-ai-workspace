# 700x700派生画像仕様

最終更新: 2026-04-05

## 対象

- 公開商品向けの表示用画像
- 入力は `data/output/public_image_manifest.csv`
- 元画像は `data/raw-images/wordpress-public/` 配下の回収済み画像

## 出力サイズ

- 700 x 700 px

## 正方形化ルール

- 元画像の縦横比は維持する
- 歪ませる拡大縮小はしない
- 700x700 の正方形キャンバスに `contain` で収める
- 収まりきらない部分の切り抜きはしない

## 余白ルール

- 不足する余白は白で埋める
- 余白は上下左右に中央寄せで付与する
- 元画像がすでに正方形比率なら余白なしで 700x700 にする

## EXIF回転

- EXIF orientation を反映してからリサイズする

## 透過PNGの扱い

- 透過は白背景へ合成する
- 透過PNGも出力は JPEG に統一する

## 出力形式ルール

- 出力形式は JPEG 統一
- 拡張子は `.jpg`
- 品質は `quality=90`
- `optimize=True`
- `progressive=True`

## ファイル名ルール

```text
<sd_product_code>-<image_seq:02d>-700x700.jpg
```

例:

```text
HYCY26924AT-01-700x700.jpg
HYOT23888AT-05-700x700.jpg
```

## 保存先

```text
data/derived-images/public-700x700/<sd_product_code>/
```

## manifest ルール

- 生成結果は `data/output/public_derived_image_manifest.csv` にまとめる
- `image_seq=1` を primary 候補とする
- 将来の `derived_url_candidate` は一旦ストレージ相対パスとして持つ

## 上書きルール

- デフォルトでは既存出力を上書きしない
- `--overwrite` 指定時のみ再生成する

## 再生成ルール

- 次の場合は再生成対象
  - 画像処理ルールの変更
  - 背景色の変更
  - 出力品質の変更
  - 元画像の差し替え

## 今回の制約

- 対象は公開商品のみ
- 非公開・売却済み・下書き相当は未対応
- ローカル派生画像生成までで、Drive 本番移行や WordPress 再アップロードは含めない

## 目視確認メモ

- 横長・縦長画像とも歪みは目立たない
- 白余白は横長 / 縦長で視認しやすいが、商品全体を収めるという目的には合っている
- 透過PNGを白背景へ合成したケースも、販売表示用途としては大きな違和感は見られなかった
