# baseImageUrl 候補比較

最終更新: 2026-04-05

## 結論

第一候補は **同一サイト配下での静的配信** とする。

想定例:

```text
https://machine-group.net/images/machine-sales/
```

この `baseImageUrl` に JSON の相対パス

```text
public-700x700/<sd_product_code>/<file>.jpg
```

を結合して配信する。

## 比較候補

### 1. 同一サイト配下で静的配信

メリット:

- 実装しやすい
- フロント側は `baseImageUrl + displayUrl` だけでよい
- 同一オリジンで扱えるため CORS や mixed content の懸念が少ない
- 公開サイトとの整合が取りやすい
- 将来 CDN 化しても `baseImageUrl` 差し替えで吸収しやすい

デメリット:

- 画像配置先の静的ディレクトリ設計が必要
- 将来の本番サーバー構成に合わせた配置作業は別途必要

評価:

- 暫定運用にも本命運用にも一番向いている

### 2. 外部ストレージ配信

メリット:

- 画像だけ独立管理しやすい
- 配信容量を本体サイトから分離しやすい

デメリット:

- URL 管理、公開設定、CORS を別途考える必要がある
- 先に配信先実装が必要

評価:

- 将来候補にはなるが、今回の最小前進としては少し重い

### 3. Google Drive 直接配信または共有URL流用

メリット:

- 正本候補と近い
- 既存の Drive 方針と整合しやすい

デメリット:

- 共有URLは配信URLとして安定性が弱い
- 画像配信の本番URLとしては扱いづらい
- キャッシュ、公開制御、表示速度の不確実性がある

評価:

- 正本管理候補としては有力
- 配信先の第一候補にはしない

### 4. WordPress メディア流用

メリット:

- 既存公開資産に近い
- 短期的には回収元として使える

デメリット:

- 今回は WordPress 依存を減らしたい方針と逆行する
- 旧運用資産への依存が残る
- 将来の切り離しがしにくい

評価:

- 回収元としては使える
- 配信先の本命にはしない

### 5. CDN / オブジェクトストレージ系

メリット:

- 将来的な拡張性が高い
- キャッシュや配信性能を取りやすい

デメリット:

- 現時点では構成が重い
- 最初の公開商品フローには過剰

評価:

- 中長期候補
- 今回の第一候補にはしない

## 採用しない候補の理由

- Google Drive 直接配信:
  - 正本管理と配信URLを同一視しない方が安全
- WordPress メディア流用:
  - 旧運用依存を増やしたくない
- CDN / オブジェクトストレージ:
  - 今回は準備コストが大きい

## 暫定運用と本命運用

暫定運用:

- preview はローカル相対パス
- `frontend/public-preview/config.js` で `baseImageUrl` を切り替える

本命運用:

- 同一サイト配下の静的ディレクトリへ配置
- 例: `https://machine-group.net/images/machine-sales/`
- JSON は相対パス維持
- フロント側で `baseImageUrl` を前置
