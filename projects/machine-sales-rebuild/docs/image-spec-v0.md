# 商品画像仕様 v0

## 目的

新統合スプレッドシート v0、サイト出力ビュー、`products.json` で共通に使う商品画像ルールを定義する。  
「元画像は正本として保持し、サイト表示用の正方形画像は派生生成物として別管理する」ことを前提にする。

## 画像ルール

| 項目 | 仕様 |
|---|---|
| 登録枚数 | 1商品あたり最低1枚、最大10枚 |
| 元画像の縦横比 | 制限しない。縦長/横長/正方形のまま登録できる |
| 表示用画像サイズ | 700 x 700 px の正方形を基本とする |
| 表示用画像の生成方針 | 商品全体ができるだけ収まることを優先し、過度な自動トリミングは避ける |
| 余白の扱い | 正方形へ合わせる際に不足する余白は背景で埋める |
| 代表画像 | `main_image_index` または配列先頭を代表画像として扱う。未指定なら1枚目を代表画像にする |
| v0の運用上限 | 将来の配列拡張は許容するが、v0 の登録上限は10枚で固定する |

## 元画像と表示用画像の役割

| 区分 | 役割 | 主な保持先 |
|---|---|---|
| 元画像 | 登録時にアップロードされた原本。縦横比や元サイズを残し、将来の再生成や差し替えの正本にする | `商品マスタ.source_image_urls` |
| 表示用画像 | 新サイト表示用に生成した 700x700 正方形画像。派生生成物として扱い、元画像とは別URLで管理する | `サイト出力ビュー.display_image_urls`, `products.json.images[].displayUrl` |
| 代表画像 | 一覧カードやSNSプレビューなどで優先表示する1枚 | `商品マスタ.main_image_index`, `サイト出力ビュー.main_display_image_url`, `products.json.images[].isMain` |

## 700x700 を基本サイズにする理由

- 一覧カードと詳細ページで画像表示比率を統一しやすい
- 横長/縦長の元画像が混在しても、フロント側レイアウトを崩しにくい
- 商品詳細の視認性を確保しつつ、過剰に大きい派生画像を避けやすい
- 将来サムネイルや高解像度派生を追加する場合も、基本表示サイズの基準を固定しやすい

## 最大10枚にする理由

- 現行の `画像1〜3` より拡張しつつ、中古機器の状態説明に必要なカット数を確保しやすい
- 登録作業が重くなりすぎない上限に抑え、実務運用しやすくする
- `products.json` やサイト詳細表示の配列が過度に大きくなるのを防ぐ

## 実装時に決めること

| 論点 | 現時点の扱い |
|---|---|
| 表示用画像の保存先 | Drive派生画像、オブジェクトストレージ、CDNのどれに置くかは実装フェーズで決める |
| 背景色/余白色 | 白背景を第一候補としつつ、サイトデザイン確定時に最終決定する |
| 派生画像の再生成タイミング | 元画像差し替え時の即時再生成、バッチ再生成、手動再生成のどれにするか実装で決める |
| サムネイル派生 | v0では必須にしない。必要になったら `thumbnailUrl` などを追加する |
| 画像メタデータの厳密管理 | v0では表示用の `width/height` を 700 固定想定で持てば足りる。元画像のEXIFや実寸管理は後段で検討する |
