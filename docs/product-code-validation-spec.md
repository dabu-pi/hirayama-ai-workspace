# 商品コードバリデーション仕様

## 目的

既存 `sd_product_code` の互換を維持しながら、新規採番時の事故と旧データ例外を制御する。  
ここでは「生成仕様」と「検証仕様」を分け、現行GASの仕様を引き継ぐ部分と、新システムで厳格化すべき部分を明示する。

## 商品コード構造

現行仕様は以下の連結形式。

```text
sd_product_code = 店舗コード + メーカーコード + 仕入年コード + 通し番号3桁 + 部位コード
```

例:

| 商品コード | 分解 | 意味 |
|---|---|---|
| `OOCY16003LG` | `OO` + `CY` + `16` + `003` + `LG` | 大阪本部 / CYBEX / 2016年仕入 / 通番3 / 脚 |
| `OOB116001AT` | `OO` + `B1` + `16` + `001` + `AT` | 大阪本部 / B1 / 2016年仕入 / 通番1 / その他 |
| `HYIC24909AT` | `HY` + `IC` + `24` + `909` + `AT` | 兵庫 / ICARIAN / 2024年仕入 / 通番909 / その他 |

注意: 現行メーカーコードは2文字固定ではなく、`HOISUT` や `KOMATSU` のような長さ例外がある。したがって、検証・分解では固定位置だけに依存しない。

## 生成仕様

| 要素 | 生成元 | 生成ルール | 未設定時の扱い |
|---|---|---|---|
| 店舗コード | 商品マスタ `store_code` | 設定マスタ `store.legacy_code` を採用 | 旧互換では `AT` を許容するが、新規採番では未設定をエラーにする |
| メーカーコード | 商品マスタ `maker_code` | 設定マスタ `maker.legacy_code` を採用 | 旧互換では `OT` を許容するが、新規採番では未設定をエラーにする |
| 仕入年コード | 商品マスタ `purchase_year` または `purchase_year_code` | `2024` → `24` のように下2桁へ変換。メーカー直送は `MD` | 未設定は新規採番ではエラー。旧データの空欄 `AT` は互換値として保持可 |
| 通し番号3桁 | 商品マスタ `serial_no` | 0埋め3桁。例: `1` → `001` | 未設定はエラー。`1000` 以上は現行仕様と衝突するため採番ルール見直し対象 |
| 部位コード | 商品マスタ `part_code` | 設定マスタ `part.legacy_code` を採用 | 旧互換では `AT` または空文字を許容するが、新規採番では空文字禁止 |

## 新規採番ルール

| ルール | 内容 |
|---|---|
| 既存コード優先 | 既存行に `sd_product_code` が入っている場合は再生成しない |
| 新規行のみ採番 | `sd_product_code` 空欄かつ、必須要素が揃っている行だけ採番する |
| 通番採番 | 現行互換を優先するなら `serial_no` を人が確定し、GAS は0埋めだけ行う。自動採番にする場合は店舗/年単位ではなく全体一意性を先に検証する |
| 衝突検出 | 生成後に既存 `sd_product_code` 全件と重複チェックし、重複時は書き込まずエラー行として返す |
| 未登録マスタ | 設定マスタに存在しない `maker_code/store_code/part_code` は採番停止し、`validation_message` で理由を残す |
| 例外コード | `首` の旧空コード、`MC` 衝突など旧例外は「既存値保持は許可」「新規採番は原則禁止または明示選択」に分ける |

## 検証仕様

## 行単位チェック

| チェック項目 | 判定ルール | NG時の扱い |
|---|---|---|
| 空欄チェック | `sd_product_code` が既存行で空なら、新規採番に必要な `store_code`, `maker_code`, `purchase_year`, `serial_no`, `part_code` が揃っているか確認 | 不足項目を列挙してエラー |
| 一意性チェック | `sd_product_code` がシート内で重複していないか確認 | 重複行番号と既存 `internal_id` をエラー表示 |
| 文字種チェック | 半角英数字のみを原則とする | 全角・空白混入はエラー |
| 店舗コード妥当性 | コード先頭が設定マスタ `store.legacy_code` と一致するか | 未一致はエラー |
| メーカーコード妥当性 | 店舗コードと年コードの間にあるメーカーコード片が、当該商品の `maker_code` に紐づく `legacy_code` と一致するか | 未一致はエラー |
| 仕入年コード妥当性 | 年コード片が `purchase_year` 下2桁または `MD` と一致するか | 未一致はエラー |
| 通し番号妥当性 | 通番部が3桁数字で、`serial_no` と一致するか | 不一致はエラー。3桁超は別途例外扱い |
| 部位コード妥当性 | コード末尾が設定マスタ `part.legacy_code` と一致するか | 未一致はエラー。旧 `首` 空コードは互換例外として警告扱い可 |
| 公開状態整合 | `condition_code=sold` なのに `inquiry_enabled=true` など矛盾がないか | 原則警告。公開仕様によってはエラーへ格上げ |

## 固定長パースを避ける分解ルール

現行はメーカーコード長が可変なので、`sd_product_code` を単純に `2+2+2+3+2` の固定長で切ると壊れる。  
新システムでは以下の順で分解する。

| 順番 | 方法 |
|---|---|
| 1 | 先頭2文字を店舗コード候補として設定マスタ `store.legacy_code` と突合 |
| 2 | 末尾側から、登録済み `part.legacy_code` のうち最長一致するものを部位コード候補として突合。旧 `首` 空コード例外は末尾コードなしとして扱う |
| 3 | 店舗コードと部位コードを除いた残り文字列の末尾3桁を通し番号候補、直前2文字または `MD` を仕入年コード候補として読む |
| 4 | 残余部分をメーカーコード候補として、当該商品の `maker_code` に紐づく `legacy_code` と照合 |
| 5 | 複数メーカーが同じ `legacy_code` を持つ場合は曖昧として警告/エラーにする |

## 旧データ例外の扱い

| 例外 | 現行の問題 | v0 方針 |
|---|---|---|
| `首` の部位コードが空 | 商品コード末尾が部位コードなしになる可能性 | 既存コードは保持し、分解時は「旧 `neck` 空コード例外」として許容。新規採番では `part_code=NECK` に対して非空の新旧対応方針を別途決める |
| メーカー `MC` 衝突 | `MAXICAM` と `MUSCLE CLAMP` が同じ `MC` | 既存コードは保持。新規採番では `maker_code` を明示選択させ、`legacy_code=MC` 衝突がある場合は自動推定しない |
| メーカーコード長が2文字超 | 固定長パースが破綻 | 分解は設定マスタ突合ベースで行う。正規表現1本で切らない |
| 大小文字混在 | `Bm`, `Hs`, `Ig`, `Lf` など表記ゆれ | 既存コードは原文保持。検証時は設定マスタの `legacy_code` と大文字小文字区別あり/なしの方針を明示し、表示上の正規化とは分ける |
| 通し番号が999超 | 現行 `slice(-3)` だと `1000` → `000` で衝突 | 新規採番では `serial_no > 999` をエラーにするか、4桁化の影響調査後に別仕様へ移行する |
| 未登録マスタ値 | 現行 `createProductCode()` は前行のコード値を引きずる可能性 | 新システムでは未登録値を即エラーとし、前行値流用を絶対にしない |

## バリデーション結果の持ち方

商品マスタ自体に以下の補助列を追加するか、別の検証ログタブを用意する。

| 列名 | 型 | 内容 |
|---|---|---|
| `code_validation_status` | string | `ok`, `warning`, `error` |
| `code_validation_message` | string | NG理由、例外内容、重複相手など |
| `code_validated_at` | datetime | 最終検証日時 |

v0 の商品マスタ本体列にはまだ必須化しないが、初回移行スクリプトを作る段階でこの3列を追加すると安全。

## テスト観点

| テスト | 期待結果 |
|---|---|
| 既存コード `OOCY16003LG` | 正常分解でき、各要素が設定マスタと一致する |
| 既存コード `HYIC24909AT` | 正常分解できる |
| メーカーコード `MC` 衝突行 | 自動推定で確定できない場合は警告/エラーになる |
| `首` 空コード旧データ | 旧例外として警告つき許容、または既存保持のみ許可 |
| `serial_no=1000` 新規採番 | 衝突リスクとしてエラーになる |
| 未登録メーカー入力 | 採番せずエラーになる |
| 同一 `sd_product_code` 重複 | 2件目以降をエラーとして検出する |
