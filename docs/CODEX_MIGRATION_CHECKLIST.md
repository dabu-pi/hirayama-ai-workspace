# CODEX_MIGRATION_CHECKLIST.md

Claude Code 前提で回していた現在の運用を、Codex 前提でも同じように安全に継続するためのチェックリスト。

目的は次の3つ。

1. 3台のPCで同じリポジトリを安全に同期する
2. AIが変わっても途中から再開できるようにする
3. Claude 固有運用を Codex でも読める形に整理する

---

## 0. 前提方針

- GitHub を **正本（唯一の基準）** にする
- 各PCのローカル状態ではなく、**コミット済みの状態** を信頼する
- AIごとの会話履歴に依存せず、**状態はファイルに残す**
- 認証情報・ローカル設定は GitHub に上げない

---

## 1. 一度だけやる移行作業

### 1-1. Codex が最初に読むファイルを明確化する

- [ ] `README.md` にワークスペース全体の入口がある
- [ ] `PROJECTS.md` に各プロジェクトの役割がまとまっている
- [ ] `ROADMAP.md` に全体優先順位がある
- [ ] `docs/PROJECT_STATUS.md` に workspace 全体の現在地がある
- [ ] 各主要プロジェクトに `README.md` または `PROJECT_STATUS.md` がある

### 1-2. Claude 専用ルールを Codex でも読める形に寄せる

- [ ] `CLAUDE.md` の内容を「Claude 専用メモ」ではなく「AI 共通の一次情報」として使う
- [ ] Codex 用の実務ルールは `AGENTS.md` または `docs/` に寄せる
- [ ] 「最初に読む順番」を文書に固定する
- [ ] 会話でしか伝わっていない運用ルールを文書へ移す

### 1-3. 再開用ファイルを統一する

- [ ] 各プロジェクトに「現在地」「次の作業」「STOP理由」を書くファイルがある
- [ ] 重要判断は `logs/notes/` または `PROJECT_STATUS.md` に残す
- [ ] 「今どこまで進んだか」をコミット前に必ず更新する

---

## 2. 各PCでやるセットアップ

対象PC:

- ノートPC
- ジムPC
- 自宅PC

### 2-1. Git / GitHub

- [ ] Git がインストールされている
- [ ] GitHub リポジトリを同じ場所に clone している
- [ ] `git remote -v` が同じ `origin` を指している
- [ ] `git config user.name` / `git config user.email` が正しい
- [ ] `push` できる認証状態になっている

### 2-2. 実行環境

- [ ] PowerShell が使える
- [ ] Python が使える
- [ ] 必要なツールが各PCで揃っている
- [ ] `SETUP.md` の内容が最新化されている

### 2-3. ローカル専用ファイル

- [ ] `.env`
- [ ] `service_account.json`
- [ ] `.clasp.json`
- [ ] `.claude/settings.local.json`

上のようなPC固有ファイルは:

- [ ] `.gitignore` 済み
- [ ] GitHub に push しない
- [ ] 必要なら各PCへ手動配布する

## 3. Codex 前提の起動ルール

### 3-1. Codex を使うときの最初の確認

- [ ] 現在の作業ディレクトリが正しい
- [ ] `git status` を確認する
- [ ] `git pull` して最新化する
- [ ] 未コミット変更がある場合は、他PCの作業残りかどうか確認する

### 3-2. Codex に最初に読ませるファイル

毎回、次を先に確認する運用にする。

1. `README.md`
2. `PROJECTS.md`
3. `ROADMAP.md`
4. `docs/PROJECT_STATUS.md`
5. 対象プロジェクトの `README.md` / `PROJECT_STATUS.md` / `spec.md`

### 3-3. Codex への定型指示

Codex の冒頭指示は次の形に揃える。

```text
まず README.md、PROJECTS.md、ROADMAP.md、docs/PROJECT_STATUS.md と、
今回触るプロジェクトの README / PROJECT_STATUS / spec を読んでから作業してください。
作業前に git status を確認し、最後に変更点・検証結果・次の作業を整理してください。
```

## 4. 3台で衝突しにくくする日常運用

### 4-1. 作業開始時

- [ ] `git status`
- [ ] `git pull`
- [ ] 今回どのプロジェクトを触るか決める
- [ ] 対象の `PROJECT_STATUS.md` を読む

### 4-2. 作業中

- [ ] なるべく1テーマずつ作業する
- [ ] 別PCと同時に同じファイルを編集しない
- [ ] 重要な判断はメモまたは状態ファイルに残す
- [ ] 認証情報や一時生成物を誤って追加しない

### 4-3. 作業終了時

- [ ] 必要なテストまたは確認を実行する
- [ ] `PROJECT_STATUS.md` や関連メモを更新する
- [ ] `git status` で意図しない差分がないか確認する
- [ ] `git add` → `git commit`
- [ ] `git push`

### 4-4. 別PCへ移る前

- [ ] push が終わっている
- [ ] 次の作業内容がファイルに書かれている
- [ ] 「会話にしかない情報」が残っていない

---

## 5. Codex 向けに整えておくと良い点

### 5-1. ルール置き場を一本化する

- [ ] AI運用ルールを `docs/` と `AGENTS.md` に寄せる
- [ ] `CLAUDE.md` だけに重要ルールを閉じ込めない
- [ ] プロジェクトごとの再開順序を文書化する

### 5-2. 状態ファイルを厚くする

- [ ] 現在地
- [ ] 完了済み
- [ ] 次アクション
- [ ] 保留事項
- [ ] テスト状況
- [ ] 直近コミット

この6点が書いてあれば、Codex でも別PCでも再開しやすい。

### 5-3. PC依存を減らす

- [ ] 絶対パス依存を減らす
- [ ] 手順を `SETUP.md` に寄せる
- [ ] エディタ設定がなくても再開できる状態にする

---

## 6. やらない方がいいこと

- [ ] push 前の変更を別PCで続きを始める
- [ ] AIとの会話だけに判断理由を残す
- [ ] `.env` や秘密鍵をコミットする
- [ ] 同じブランチで別PCから同時に大きな編集をする
- [ ] 状態ファイルを更新せずに PC を切り替える

---

## 7. おすすめの最小運用

毎回これだけ守れば、Codex でもかなり安定する。

1. 作業前に `git pull`
2. 対象プロジェクトの `PROJECT_STATUS.md` を読む
3. 作業後に状態ファイルを更新する
4. `commit` と `push` を終えてから PC を変える

---

## 8. このワークスペースでの優先対応

- [ ] Codex でも参照すべきファイル順を固定する
- [ ] `CLAUDE.md` のうち AI 共通ルールを `docs/` または `AGENTS.md` へ移す
- [ ] 各主要プロジェクトの `PROJECT_STATUS.md` を継続更新する
- [ ] `SETUP.md` を3台運用前提で再確認する
- [ ] PC固有ファイルの一覧を `.gitignore` と運用文書で明文化する
