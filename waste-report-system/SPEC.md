## 0. 目的

Googleスプレッドシートの入力（収集運搬一覧）から、**提出事故ゼロ**で以下を自動作成し、毎月の提出業務を省力化する。

- 日報（帳票型・案件別）
- 月報（帳票型・提出用：日報参照で生成）
- 年度積算（提出用：年度開始〜指定月分までの累計、月報レイアウト）
- PDF出力（指定フォルダ）
- Gmail下書き作成（PDF添付、送信は手動）

---

## 1. 最優先方針（事故ゼロ）

1. 入力の欠損・不正・重複がある場合は **停止（throw Error）** する
2. 提出済み（SENT）データは **上書き禁止**
3. テンプレ破損・結合崩れ・数式欠損など「帳票事故」は **強制復旧** してから生成する

---

## 2. シート構成（確定・稼働中）

| 種別 | シート名 | 役割 |
|---|---|---|
| 入力（UI） | 収集運搬一覧 | 1行＝1回収集の記録（集計元） |
| テンプレ | 日報 | 帳票テンプレ（コピーして 日報_YYYY-MM を作成） |
| テンプレ | 月報 | 帳票テンプレ（コピーして 月報_YYYY-MM / 年度積算_YYYY-MM月分 を作成） |
| 設定 | 設定 | PDF出力先/年度設定等 |
| ログ | 提出済み確認 | 対象年月ごとの生成状態・提出状態を管理 |
| 実績 | 年度積算_YYYY-MM月分 | 自動生成される月次累計帳票（例: 年度積算_2026-04月分） |

---

## 3. 用語定義

| 用語 | 定義 |
|---|---|
| 対象年月（targetYm） | YYYY-MM |
| 一般ごみ（general） | 燃やすごみ + 不燃ごみ |
| 資源ごみ（recycle） | 資源ごみ |
| 案件（Case） | 日報に出力する単位。排出者の氏名 + 排出先の住所 で統合 |

---

## 4. 入力仕様（収集運搬一覧）

### 必須ヘッダー（存在しない場合は停止）

日付 / 排出者の氏名 / 排出先の住所 / 収集運搬の車種 / 車番 / 燃やすごみ / 不燃ごみ / 資源ごみ

### 入力検証ルール

| 項目 | ルール |
|---|---|
| 排出者の氏名 | 空行は無視（スキップ） |
| 排出先の住所 | 空欄は停止（事故防止） |
| 日付 | Date に変換できない場合は停止 |
| 数量 | 非数値は停止（空欄は0扱い） |

---

## 5. 提出済み管理（提出済み確認シート）

### ステータス

| 値 | 意味 |
|---|---|
| DRAFT | 作業用シート生成済み |
| PDF_READY | PDF生成済み（下書き作成済み含む） |
| SENT | 提出完了（以後更新禁止） |

### ガード（最重要）

対象年月が **SENT** の場合、日報/月報/年度積算/自動下書き作成をすべて停止する。

---

## 6. 日報仕様（作業用：日報_YYYY-MM）

### 生成
- シート名: `日報_YYYY-MM`
- 日報テンプレからコピー
- 同名がある場合は削除して作り直す

### ヘッダー（帳票事故対策）
- 対象範囲: K4:AO6
- 処理: breakApart → clear → 再セット

| セル | 内容 |
|---|---|
| K4:AO4 | YYYY年 M月 |
| K5:AO5 | 1〜31（日数超は空） |
| K6:AO6 | 曜日（日/月/火/水/木/金/土） |

### 案件ブロック
- 4行1ブロック（先頭7行目、4行刻み）
- 氏名: E列 / 住所: F列 / 車種・車番: I列（上段/次段）
- 日別入力: K〜AO
- 合計: AP列（=SUM(K:AO)）を必ず再設定
- 固定ラベル: J列（燃やす/不燃/資源）が空の場合は自動補修

### 案件統合（Case化）
- キー: `排出者の氏名 | 排出先の住所`
- 同一キーは日付ごとに数量を加算
- キー矛盾（同一排出者で住所揺れ等）は停止

---

## 7. 月報仕様（作業用：月報_YYYY-MM）

### 生成
- シート名: `月報_YYYY-MM`
- 月報テンプレからコピー
- **前提: 日報_YYYY-MM が存在しない場合は停止**

### タイトル・提出日
- A4:N4（結合セル）: 一般廃棄物収集運搬業務実績報告書（YYYY年M月分）
- L7:N7（結合セル）: 対象月末日

### 初期化（帳票事故対策）
- B30:N49 を 0 初期化
- A30:A47（事業所名欄）をクリア
- A48:A49 は breakApart→merge→「合計」を必ずセット

### 月列マッピング（4月始まり）

| 列 | 月 |
|---|---|
| B | 4月 |
| C | 5月 |
| D | 6月 |
| E | 7月 |
| F | 8月 |
| G | 9月 |
| H | 10月 |
| I | 11月 |
| J | 12月 |
| K | 1月 |
| L | 2月 |
| M | 3月 |
| N | 年間合計（=SUM(B:M) を毎回設定） |

### 転記元（日報から抽出）
- 日報の案件ブロックから排出者ごとの月合計（AP列）を取得
- 1排出者: 上段=一般（燃やす＋不燃） / 下段=資源
- 同名重複が日報内に存在する場合は停止

---

## 8. 年度積算仕様（月次提出用：年度積算_YYYY-MM月分）

### 生成
- シート名: `年度積算_YYYY-MM月分`
- 月報テンプレからコピー

### 集計期間（"○月分提出"の定義）
- 対象 YYYY-MM の **翌月1日（排他）** までを上限として年度累計する
- 例: 2026-02 → 〜 2026/03/01 未満

### タイトル・提出日
- A4:N4: 一般廃棄物収集運搬業務実績報告書（M月分）
- L7:N7: 対象月末日

### 集計ロジック
- キー: `排出者の氏名 | 排出先の住所`
- 月別 B..M = 4月..3月（上段=一般、下段=資源）
- 合計行 A48:A49: 列別合計をセット
- N列（年計）: =SUM(B:M) を毎回セット

### 枠制約
- **9社超えの場合は停止**（現仕様: 月報テンプレ1枚で提出する運用）

## 9. PDF出力仕様

### 出力先
- 設定シートの PDF出力フォルダID があればそのフォルダ
- なければスプレッドシートの親フォルダ（なければルート）

### 出力形式
- export URL 方式
- A4 / portrait / fitw=true / gridlines=false

## 10. 自動月次処理（毎月1日）

### スケジュール
- **毎月1日 21:00（JST）**
- 関数名: `WR_AUTO_runMonthlyFiscalDraft`

### 処理手順

1. SENT ガード（提出済みなら停止）
2. 年度積算（`年度積算_YYYY-MM月分`）作成 ← 前月が対象
3. 対象シートを PDF 出力（blob 取得）
4. Gmail 下書き作成（送信はしない）
5. 提出済み確認ログを PDF_READY 更新

### Gmail 下書き設定

| 項目 | 値 |
|---|---|
| To | tanigaki-toshihiko@nantan.hyogo.jp |
| 件名 | 廃棄物実績報告書（YYYY-MM分） |
| 添付 | 年度積算 PDF |
| 本文署名 | 廃棄物収集運搬（自動作成） |

### メール本文（固定テンプレ）

```
南但クリーンセンター
谷垣 様

いつもお世話になっております。
廃棄物収集運搬業務の実績報告書（YYYY年M月分）をお送りいたします。

本メールには、PDFファイルを添付しております。
ご査収のほどよろしくお願い申し上げます。

――――――――――――――――――
（自動作成）
```

### 下書き量産防止（任意）
- in:drafts で同条件の下書きが見つかった場合は作成をスキップしてログに注記

## 11. GAS ファイル構成（実装済み・clasp clone 確認済み）

| ファイル | prefix | 役割 |
|---|---|---|
| `appsscript.json` | — | V8 runtime / Asia/Tokyo タイムゾーン設定 |
| `WR_main.gs.js` | `WR_` | 日報作成・月報作成（B案: 日報参照）・PDF出力・ログ・ユーティリティ |
| `WR_main.gs.js` | `WR2_` | 年度積算（指定YYYY-MM月分）: 年度開始〜指定月の累計を月報テンプレ1枚で生成 |
| `WR_main.gs.js` | `WR2V2_` | 年度積算 v2（設定の年度選択）: 9社/ページ分割・ページ別PDF対応 |
| `WR_autoSubmit.js` | `WR_AUTO_` | 月1回トリガー・前月分年度積算→PDF→Gmail下書き作成 |

> **注意（旧 SPEC との差異）:** 仕様書 v2026-02-25 では `WR_core.gs` と記載していたが、実際のファイル名は `WR_main.gs.js`。

**関数一覧（主要）:**

| 関数名 | 用途 |
|---|---|
| `onOpen` | スプレッドシート開時にメニュー追加 |
| `WR_buildDailySheet(ym)` | 日報_YYYY-MM 生成 |
| `WR_buildMonthlyReport(ym)` | 月報_YYYY-MM 生成 + PDF出力 |
| `WR2_buildFiscalMonthlyReport(ym)` | 年度積算_YYYY-MM月分 生成（手動） |
| `WR2V2_buildFiscalReport_fromSettings()` | 年度積算 v2（設定シートの年度選択から自動） |
| `WR_AUTO_runMonthlyFiscalDraft()` | 月1回自動: 前月分年度積算→PDF→Gmail下書き |
| `WR_AUTO_installMonthlyTrigger()` | 毎月1日21:00トリガーを設定（1回だけ実行） |

**名前空間ルール:**
- Apps Script は .gs 全体が同一グローバル名前空間
- `WR_` / `WR2_` / `WR2V2_` / `WR_AUTO_` の prefix で衝突を防止
- 新機能追加時は必ず別 prefix を使うこと（例: `WRP_` / `WRY_`）

## 13. 運用手順（確定）

### 通常（月次・手動）

1. 必要なら 日報 作成（指定YYYY-MM）
2. 月報 作成（指定YYYY-MM）（日報が必須）
3. 年度積算 作成（指定YYYY-MM）（提出用）
4. PDF を確認
5. 自動生成された Gmail 下書きを確認し、手動送信
6. 送信後、提出済み確認のステータスを SENT に更新

### 自動（月1）

毎月1日 21:00 に「前月分」の年度積算 PDF を添付した下書きが自動作成される。
