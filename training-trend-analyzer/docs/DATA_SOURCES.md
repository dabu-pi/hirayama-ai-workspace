## 収集ソース一覧

### S-01: Google Trends

| 項目 | 内容 |
|---|---|
| 用途 | 機種名・ブランド名の検索ボリューム推移を取得 |
| 取得方法 | pytrends（非公式Pythonライブラリ）または手動ダウンロード |
| データ形式 | 相対スコア（0〜100）。絶対数ではない |
| 更新頻度 | 日次〜週次（pytrends はリクエスト制限あり） |
| 利点 | 無料。実際の検索行動を反映。長期履歴あり（2004〜） |
| 制約 | 絶対数が取れない。地域・期間設定で値が変わる。比較対象が変わると再比較が必要 |
| 法的注意 | 非公式ライブラリ使用に伴うリスクあり。利用規約はGoogle利用規約に準ずる |
| MVPでの採用 | **採用（優先）** |

**実装メモ:**
```python
# pytrends の基本使用例
from pytrends.request import TrendReq
pytrends = TrendReq(hl='ja-JP', tz=540)
pytrends.build_payload(['TECHNOGYM', 'Life Fitness'], timeframe='today 12-m', geo='JP')
df = pytrends.interest_over_time()
```
- 1回のリクエストで比較できるキーワードは5つまで
- 過剰リクエストで一時的にブロックされる場合あり。インターバルを設ける

### S-02: Amazon（PA-API / 手動）

| 項目 | 内容 |
|---|---|
| 用途 | 機種別のレビュー件数・平均評価・ランキング順位・価格 |
| 取得方法 | Amazon PA-API 5.0（要アソシエイト審査）または手動スクレイピング |
| データ形式 | JSON（PA-API）/ HTML パース（スクレイピング） |
| 更新頻度 | 日次〜週次 |
| 利点 | 購買行動に近い指標。評価数・レビュー内容が豊富 |
| 制約 | PA-API は審査が必要（売上実績が必要な場合あり）。スクレイピングは利用規約で禁止 |
| 法的注意 | Amazon のスクレイピングは利用規約（Conditions of Use）で禁止。PA-API のみ使用可 |
| MVPでの採用 | **保留（手動確認は可。自動は PA-API 審査後）** |

**実装メモ:**
- MVP では手動で商品ページを確認し、CSVに記録して取り込む方式
- PA-API 審査が通り次第、`collectors/amazon.py` を実装する

### S-03: 楽天市場（楽天API）

| 法的注意 | 楽天デベロッパーAPI利用規約に準ずる。商用利用は確認が必要 |
| MVPでの採用 | **Phase 2 で採用予定** |

## 法的・運用上の注意（共通）

| 注意事項 | 内容 |
|---|---|
| robots.txt 確認 | スクレイピング前に必ず `robots.txt` を確認し、禁止されているサイトには接触しない |
| 利用規約遵守 | 各プラットフォームの利用規約・API ToS を読んでから実装する |
| 著作権 | 収集したテキストデータをそのまま公開しない。集計・スコア化したデータのみを公開 |
| 個人情報 | ユーザー名・レビュアー名等の個人識別情報は取得しない |
| レート制限 | API のレート制限を守る。過負荷をかけない |
| キャッシュ | 同一データの再取得は避け、DBにキャッシュする |
| ソース明示 | 社内レポートにはデータソース・取得日を必ず記載する |
