# PROJECT_STATUS.md — トレーニングマシントレンド分析基盤

最終更新: 2026-04-08

---

## プロジェクト目的

トレーニングマシンのトレンドを複数ソースから継続収集し、ブランド・カテゴリ・機種単位でデータベース化し、時系列変化をスコア化してランキング提示する。

**短期目標（MVP）:** 自社仕入れ判断を数字で支援するローカルツール
**中期目標:** ジムオーナー・仕入れ担当者向け参照サイトとして公開

## 次アクション（優先順）

1. **モックデータ作成** — `data/mock/` にサンプルJSONを用意し、DB不要でロジックを動かせるようにする
2. **DB初期化スクリプト** — `scripts/init_db.py` でSQLiteにスキーマを作成
3. **正規化エンジン雛形** — `src/normalizer/` にブランド・カテゴリのマスタ辞書を作る
4. **手動収集CSVインポーター** — まず手動データをCSVで入れてランキングを確認する
5. **スコア計算の最小実装** — 言及数ベースの単純スコアで動かす

## リスク

| リスク | 深刻度 | 対策 |
|---|---|---|
| 表記ゆれによる同一機種の重複登録 | 高 | 正規化エンジンを最初から作る。aliases テーブルで吸収 |
| 家庭用/業務用の混在 | 高 | categories に use_type カラムを持たせる |
| データソースの利用規約違反 | 中 | DATA_SOURCES.md で各ソースの法的注意を明記 |
| スクレイピング禁止サイトへのアクセス | 中 | robots.txt 確認 + 公式API優先 |
| ランキングの恣意性 | 中 | 複数指標の合成スコアを採用。計算式を透明化 |
| モデルの廃番・改称 | 低 | aliases テーブルで旧名を保持 |
