# DATA_POLICY — データ・個人情報管理方針

**プロジェクトID:** PMOD-08  
**最終更新:** 2026-04-10  
**位置づけ:** 症例データ・患者情報の取り扱いに関する絶対ルール

---

## 基本原則

> **「患者さんのデータを使わせてもらっている」という認識を常に持つ。**

このプロジェクトは治療家の学習・情報発信を目的とするが、
そのために患者の個人情報・医療情報を扱う。
適切な匿名化・管理をせずにデータを使うことは、
患者の信頼を裏切る行為であり、法的にも問題になる。

---

## 個人情報の定義と注意事項

以下の情報は「個人情報」に該当し、**このプロジェクトのどのファイルにも含めない**。

| 情報の種類 | 具体例 |
|---|---|
| 氏名 | 本名・ニックネーム・イニシャル（特定可能なもの） |
| 住所 | 都道府県以下の具体的な住所 |
| 生年月日 | 完全な生年月日（年齢は可） |
| 電話番号 | 自宅・携帯 |
| 保険証番号 | 社会保険・国民健康保険 等 |
| 職業（特定可能なもの） | 具体的な会社名・役職 |
| 来院日時（特定可能な組み合わせ） | 特定人物と紐づく記録 |

## 匿名化の手順

症例をデータベースに登録する前に、必ず以下を実施する。

### Step 1: 識別子の除去

- 氏名 → 削除（「患者A」「Case-001」等の連番IDに置き換える）
- 生年月日 → 年代に変換（例：「1975年3月15日」→「40代後半」）
- 住所 → 削除
- 保険証番号・患者番号 → 削除
- 来院日時 → 季節・月のみ（例：「2025年11月5日」→「2025年秋」）

### Step 2: 特定性の排除

- 珍しい職業・生活状況は一般化する（「○○工場勤務」→「肉体労働」）
- 複数の情報を組み合わせて特定できる場合は追加で削除・変更する

### Step 3: 匿名化確認

- 「このデータだけを見て、誰のことかわかるか？」を問いかける
- わかるならさらに匿名化する

## 院内用と公開用の分離

| 用途 | 保存場所 | 内容 |
|---|---|---|
| 院内用（詳細記録） | **Git管理外** の安全な場所（暗号化推奨） | 詳細な患者情報・経過記録 |
| データベース用（匿名化済み） | `data/cases/anonymized/` | 匿名化済み。Git管理内 |
| note・商品化用 | `note/` または `product/` | さらに簡略化・一般化済み |

**院内用の詳細記録は絶対にこのリポジトリに含めない。**

---

## `data/cases/anonymized/` の運用ルール

- **このフォルダに入れるのは匿名化済みのデータのみ**
- Git管理内に置くため、push 前に必ず個人情報の有無を確認する
- ファイル名は `case_YYYYMM_NNN.md` 形式（例: `case_202601_001.md`）
- ケースIDと実際の患者の対応表は **院内のみで管理**（このリポジトリには含めない）

## note・商品化での使用ルール

### 使用できる情報

- 年代（例：50代男性）
- 主訴（例：慢性腰痛・半年以上）
- 職業の種別（例：デスクワーク中心）
- 使用した機器・設定・期間
- 治療経過（VASスコアの変化など数値ベース）
- 「うまくいった点・うまくいかなかった点」の考察

### 使用できない情報

- 特定の個人を連想させるエピソード
- 「○○市の○○さん」的な記述
- 患者の写真・動画（許可があっても原則使用しない）

### 使用前のチェックリスト

- [ ] 匿名化を Step 1〜3 の手順で確認したか
- [ ] 院内に照会して特定可能性がないか確認したか
- [ ] 「この記事を読んだ患者が、自分のことだと気づく可能性」を考慮したか
- [ ] AI検証プロンプトでの確認を実施したか

## 法的・倫理的リスクを避けるルール

| リスク | 回避策 |
|---|---|
| 個人情報保護法違反 | 個人情報を含む症例データの公開は絶対禁止 |
| 医療広告ガイドライン違反 | 「治った」「必ず効く」等の誇大表現をしない |
| 名誉毀損 | 特定可能な患者の情報を使う場合は事前同意が必要 |
| 著作権侵害 | 論文・書籍から引用する場合は出典を明記し適切な引用範囲を守る |
| 医師法・柔道整復師法 | 適応外の使用・診断に類する表現を避ける |

### 医療広告ガイドラインに関する注意

note・商品での記述においては以下に注意する。

- 「〜は完治します」「必ず効きます」→ **禁止**
- 「〜に効果があるとされています」「〜で改善した例があります」→ OK
- 「この方法なら手術を避けられます」→ 証拠なしには **禁止**
- 「治療家向けの学習資料です」という立ち位置を明確にする

## データ保護の技術的対策

- 院内用の詳細記録は `.gitignore` で除外する、またはこのリポジトリ外に保管する
- `secrets/` や `.env` ファイルと同様に、個人情報ファイルは Git 管理対象外にする
- 万一 push してしまった場合は即座に `git history` から削除する手順を踏む（連絡先: IT担当）
