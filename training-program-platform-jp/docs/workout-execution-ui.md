# workout-execution-ui

## 位置づけ

この画面は、アプリの最重要画面である。
プログラムライブラリで選んだ内容を、実際のトレーニング行動に変える中心であり、ここが弱いと継続利用が成立しない。

特に `前回記録` と `履歴` は補助機能ではなく、この画面の価値を支える中核要素として扱う。

## 1画面で完結すべき理由

- トレーニング中は操作回数と視線移動を減らす必要がある
- 別ページで前回記録を探させると入力の流れが切れる
- 初心者ほど「今どこを見ればよいか」で迷いやすい
- 中級者は前回値と今回目標を見比べながら即判断したい

そのため、実行中に必要な主要情報は 1 画面で完結する前提を置く。

## 必須表示項目

- 上部固定エリア
  - `Rest`
  - `Calc`
  - タイマー
  - `Finish`
- プログラム情報
  - プログラム名
  - `Week / Day`
  - `Progression Guide`
  - `Notes`
- 種目名
  - タップで当該種目の単体履歴へ遷移
- 種目タイプ
  - `T1 / T2 / T3` など
- Set 番号
- Previous
  - 前回の重量・回数
  - データがない場合は `-`
- Target
  - 目標回数や目標条件
- Kg 入力欄
- Reps 入力欄
- 完了チェック
- 休憩タイマー
- 記録完了導線
- 履歴参照導線

## 必須操作

- Kg 入力
- Reps 入力
- 完了チェック
- 種目単位の記録確認
- `Add Set`
- `Swap`
- `Add Exercise`
- 履歴画面への移動
- 記録完了

## 実行中にユーザーが迷わないための UI 原則

- `Previous` は各セット行で即見える
- `Previous` は参照用表示であり、Kg 入力の初期値ではない
- `Target` は曖昧にせず、そのセットで何を目指すかを短く示す
- 上部固定エリアはスクロール中でも主要操作を失わないようにする
- プログラム情報は読み物ではなく、進行判断に必要な最小情報だけを置く
- 入力欄は左から右、上から下へ自然に埋められる
- 補助情報より入力と確認を優先する
- 実行中は不要な説明文を増やしすぎない
- 履歴は別画面で深掘りできるが、導線は常に近くに置く

## モバイル前提の情報優先順位

1. 種目名
2. 各セット行の `Previous`
3. 各セット行の `Target`
4. Kg / Reps 入力
5. 完了状態
6. タイマー
7. Add Set / Swap / Add Exercise
8. 履歴導線

## 各セット行の例

```text
Set 1 | Previous 80kg x 3 | Target 3 reps | Kg [   ] | Reps [   ] | 完了
```

## Previous がない場合の例

```text
Set 1 | Previous - | Target 5 reps | Kg [   ] | Reps [   ] | 完了
```

## AMRAP 例

```text
Set 3 | Previous 82.5kg x 2 | Target 3+ reps | Kg [   ] | Reps [   ] | 完了
```

## Kg 入力の基本ルール

- Kg 欄の初期値は空欄とする
- `Previous` をそのまま入力値として自動セットしない
- ユーザーが 1 セット目の Kg を入力したら、同一種目の後続セットの Kg 欄へ同じ重量を自動反映する
- 自動反映後も、後続セットは必要に応じて個別上書きできる前提とする

## セット行の状態遷移

### 1. 未入力

- Kg / Reps が未入力
- `Previous` と `Target` を参照しながら入力を始める状態

### 2. 入力済み

- Kg / Reps のいずれか、または両方が入力済み
- まだ完了チェック前なので編集可能

### 3. チェック済み・ロック

- 完了チェックを入れた状態
- その行の入力値はロックされる
- 誤タップや進行中の値変更を防ぐ

### 4. ロック解除後再編集

- 編集が必要な場合のみロック解除する
- ロック解除後は再び入力済み状態として編集できる
- 再チェックで再度ロックされる

## 種目名タップの導線

- 種目名を押すと、その種目単体の履歴画面へ遷移する
- 遷移先はプログラム全体履歴ではない
- 実行中に必要な `Previous` はその場で見せつつ、詳細確認だけを種目単体履歴へ逃がす

## 画面遷移の最小導線

`ライブラリ → 詳細 → 今日のワークアウト → 記録完了 → 履歴`

補足:

- 実行画面から履歴へ途中移動できる導線を持つ
- 種目名タップでは種目単体履歴へ移動できる
- ただし `前回記録` 自体は履歴画面へ飛ばなくても読める状態を維持する
