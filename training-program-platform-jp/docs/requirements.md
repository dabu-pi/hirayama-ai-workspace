# requirements

## 前提

- 初期対象は初心者から中級者
- 初期運用は管理者 1 名
- 初期は一般ユーザー投稿機能なし
- 将来の複数ユーザー登録 / 利用を前提に要件を置く

## MVP の中心要件

- ワークアウト実行画面は MVP の中核である
- `前回記録` と `履歴` は補助機能ではなく、継続利用を支える中心機能として扱う
- 前回記録は別ページ参照ではなく、ワークアウト実行画面の各セット行で即見えることを前提にする
- 履歴ページは別途必要とし、実行画面から過去履歴へ移動できる導線を持つ

## 機能要件

### 1. 認証・権限

- ユーザー登録 / ログイン
  - 将来対応前提として要件だけ明記する
  - 初期 MVP では本実装を急がない
- 管理者ログイン
  - 初期運用で必要
  - 管理画面へのアクセス制御を行う

### 2. プログラム閲覧

- プログラム一覧を表示できる
- 条件絞り込みができる
  - 目的
  - レベル
  - 週あたり頻度
  - 期間
- プログラム詳細を表示できる
  - 概要
  - 想定対象
  - 進め方
  - 週構成 / 日構成

### 3. プログラム開始と実行

- プログラム開始ができる
- 当日ワークアウトを表示できる
- ワークアウト実行画面は 1 画面で必要情報が完結することを優先する
- 種目名を押すと、その種目単体の履歴画面へ遷移できる
  - プログラム全体履歴ではなく、当該種目だけの履歴を参照する導線とする
- 各セット行に次の情報を表示できる
  - Set 番号
  - Previous
    - 前回の重量・回数
  - Target
    - 目標回数や目標条件
  - Kg 入力
  - Reps 入力
  - 完了チェック
- 前回記録は各セット行で即見えること
- 前回記録のために別ページへ移動させないこと
- `Previous` が存在しない場合は空欄ではなく `-` を表示する
- `Previous` は参照用表示であり、Kg 入力の初期値にはしない
- Kg 欄は初期状態では空欄とする
- ユーザーが 1 セット目の重量を入力したら、同一種目の後続セットの Kg 欄へ同じ重量を自動反映できること
  - ただし後続セットは必要に応じて個別に上書きできる前提を置く
- 当日の種目ごとに次の情報を記録できる
  - セット
  - 回数
  - 重量
  - RPE
  - メモ
  - 完了状態
- 各セット行は完了チェックを持つ
- チェック後、その行の入力値はロックされる
- 編集する場合はロック解除が必要
- AMRAP 表記を扱えること
  - 例: `3+ reps`
- Reps 入力挙動は固定ロジックに寄せすぎず、プログラム側設定に従える前提を置く
- `Target` 側で固定回数だけでなく AMRAP や `3+ reps` のような表記を扱えること
- 画面項目として次の操作を置ける前提を持つ
  - `Add Set`
  - `Delete Set`
  - `Swap`
  - `Add Exercise`
- `Delete Set` は追加セットだけでなく通常セットも対象にできること
- セット行は左スワイプで `Delete` を出せること
- セット削除は基本操作として扱い、例外操作にしないこと
- ロック済みセットはそのままでは削除不可とすること
  - 削除前に `Unlock`（ロック解除）が必要
- セット削除時は確認ダイアログ（削除前確認）を出す方針を持つこと
- セット削除後は表示上の Set 番号を詰め直すことを推奨する
- 実行画面に休憩タイマーを置く前提とする

### 4. 履歴管理

- ワークアウト履歴を表示できる
- 実行済みセッションの記録詳細を確認できる
- 継続中プログラムの進行状況を確認できる
- 実行画面から過去履歴へ移動できる
- 履歴は次回負荷判断の材料として参照しやすいこと

### 5. 管理機能

- 管理者によるプログラム登録 / 編集
- 作成者情報登録
- 種目マスタ登録 / 編集
- 公開 / 非公開管理

## 非機能要件の方向性

- スマホ優先 UI
- 初期は Web 寄り
- 将来の PWA 化を妨げない構成
- 単一管理者運用から将来の複数ユーザー運用へ拡張しやすいこと
- 実行中に迷わない視認性と入力速度を優先すること

## 今回未確定のまま置く論点

- 一般ユーザー向け認証の実装時期
- 管理画面と利用画面の技術分離レベル
- オフライン対応の実装範囲
- プログラム進行ロジックの自動化粒度
- `Add Set` と `Swap` を MVP で実装まで入れるか、構造準備に留めるか
- `Delete Set` の Undo（取り消し）を MVP に含めるか
- `Add Exercise` を MVP で実装まで入れるか、構造準備に留めるか
- 休憩タイマーの詳細仕様
- ロック解除 UI の見せ方
- 後続セット自動反映後の個別上書きルールの細部
