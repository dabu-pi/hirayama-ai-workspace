# Claude 共通前提テンプレート（パフォーマンス重視版）

このテンプレートは、training-program-platform-jp を含む全開発タスクで毎回プロンプト冒頭に付与する共通前提です。

---

## ① 実行ポリシー（承認不要範囲）

以下は事前承認済みとして、承認なしで実行してよい：

- ファイル閲覧・検索
- ローカルファイル編集
- テスト実行
- ログ確認
- git status / diff / add / commit / push
- 必要な build / lint / typecheck

以下のみ実行前に停止して報告すること：

- 本番DBの破壊的変更
- データ削除
- 秘密情報・認証情報の変更
- 外部サービスへの送信
- 課金や irreversible な操作

---

## ② パフォーマンス前提（最重要）

すべての実装・変更は以下を必ず満たすこと：

- パフォーマンス悪化を避けること
- 既存より体感速度を落とさないこと
- 不要な JOIN / query / render を増やさないこと
- 可能な限り最小変更で実装すること

### 禁止・注意事項

- 不要なテーブルJOINの追加
- 全件取得（over-fetching）
- 同一データの多重取得
- layout / root での重いクエリ実行
- client 側での過剰 state 管理

### 推奨方針

- 必要なデータのみ取得（select最小化）
- lazy load / conditional fetch の活用
- server component 側での適切なデータ取得
- キャッシュ可能なものはキャッシュ前提で設計
- 重い処理は分離（非同期・段階読み込み）

---

## ③ 設計原則

- 既存構造を壊さない（backward compatible）
- データモデルは可能な限り維持する
- 表示層で吸収できる変更はUI側で解決する
- 新しい概念は「レイヤー追加」で解決する

---

## ④ 実装優先順位

1. 正しく動く
2. 速い（体感速度）
3. 軽い（負荷・構造）
4. 拡張しやすい

※「動くが遅い」は未完成とみなす

---

## ⑤ 出力時の必須項目

すべての回答に以下を含めること：

- STATUS
- SUMMARY
- CHANGES
- UPDATED_FILES
- COMMIT / PUSH
- NEXT

### パフォーマンス補足（必須）

- PERFORMANCE_IMPACT:
  - 変更による負荷増減
  - クエリ数の変化
  - レンダリング影響

---

## ⑥ 判断基準

複数案がある場合：

- より軽い案
- よりシンプルな案
- より既存に影響が少ない案

を優先する

---

## ⑦ このテンプレートの役割

- 開発のブレ防止
- パフォーマンス劣化の予防
- 設計の一貫性維持
- 将来のB2B2C拡張に耐える構造維持

---

## ⑧ 運用ルール

- すべてのClaudeプロンプトの冒頭に貼る
- 新しい問題が出たらこのテンプレートを更新する
- ローカルにも保存し、毎回参照する
