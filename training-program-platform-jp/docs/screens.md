# screens

## 画面一覧

### 1. ホーム

- アプリの入口
- 継続中プログラム、ライブラリ導線、最近の記録への導線を置く
- スマホで最初に見る前提の構成にする

### 2. ライブラリ

- プログラム一覧を表示する
- レベル、目的、頻度などの条件絞り込みを行う
- 初心者が選びやすい説明を優先する

### 3. プログラム詳細

- プログラム概要
- 想定対象
- 期間、頻度、構成
- 作成者情報
- 開始ボタン

### 4. 今日のワークアウト

- アプリの最重要画面
- 当日実行する種目一覧
- 前回記録と今回目標を見ながら、その場で入力完了できる構成にする

#### 主要構成

- 上部固定エリア
  - `Rest`
  - `Calc`
  - タイマー
  - `Finish`
- プログラム情報エリア
  - プログラム名
  - `Week / Day`
  - `Progression Guide`
  - `Notes`
- 種目セクション
  - 種目名
  - 種目タイプ表示
  - 補助情報
  - 種目名タップで種目単体履歴へ移動
- セット一覧テーブル
  - `Set`
  - `Previous`
  - `Target`
  - `Kg`
  - `Reps`
  - `完了`
- 種目ごとの補助操作
  - `Add Set`
  - `Swap`
- 履歴への導線
- 画面下部
  - `Add Exercise`

#### 操作仕様

操作挙動の詳細（タップ・入力・ロック・自動反映・履歴遷移）は以下を参照。

- [workout-screen-interaction-spec.md](./workout-screen-interaction-spec.md)

#### 画面要件

- `Previous` は各セット行で即見える
- `Previous` がない場合は `-` を表示する
- `Target` は目標回数や `3+ reps` のような条件をその場で読める
- Kg 欄の初期状態は空欄とする
- `Previous` は参照用であり、Kg 欄への自動初期入力には使わない
- 1 セット目に入力した重量は、同一種目の後続セットの Kg 欄へ自動反映できる
- 自動反映された後続セットの Kg 値は、必要に応じて個別上書きできる
- 各セット行は完了チェック後にロックされる
- 再編集にはロック解除が必要
- 入力のために別画面へ飛ばさない
- 実行画面から履歴画面へ移動できる
- 種目名タップ時の遷移先はプログラム全体履歴ではなく、当該種目の単体履歴とする

### 5. 記録完了

- 当日の完了結果サマリ
- 次回への導線
- 履歴確認への導線

### 6. 履歴

- 過去のワークアウト一覧
- セッション詳細
- 継続中プログラムの進捗確認
- 実行画面で見ている `Previous` の根拠を後から確認する画面
- 次回負荷判断のための参照先

#### 「今日のワークアウト」との関係

- 実行画面では各セット行に前回記録を即表示する
- 履歴画面では過去セッション全体を確認する
- 実行画面から履歴へ移動できる導線を必ず置く
- 種目名タップ時は種目単体履歴へ移動する
- `前回記録` は実行画面で使う表示、
  `履歴` は過去全体を振り返る画面として役割を分ける

### 7. 管理画面トップ

- 管理メニュー一覧
- プログラム管理
- 種目マスタ
- 作成者マスタ
- 公開状態の確認

### 8. プログラム登録 / 編集

- 基本情報入力
- 週構成 / 日構成編集
- 種目割り当て
- 公開 / 非公開設定

### 9. 種目マスタ

- 種目一覧
- 和名 / 英名 / 表記ゆれ管理
- カテゴリやメモ管理

### 10. 作成者マスタ

- 作成者一覧
- 表示名、説明、出典メモなどの管理

### 11. 将来のログイン / マイページ想定欄

- ユーザー登録 / ログイン画面
- マイページ
- 参加中プログラム一覧
- 自分の履歴 / プロフィール / 設定

## 備考

- 初期 MVP はスマホ利用画面を優先する
- PC 画面は管理・登録補助が中心
- ログイン / マイページは将来欄として先に場所だけ想定しておく
- 特に `今日のワークアウト` と `履歴` の往復導線は MVP の中心として扱う
