# user-stories

## 1. 初心者ユーザーがプログラムを探して開始する

### ストーリー

トレーニング経験が浅いユーザーとして、
自分の目的とレベルに合うプログラムを見つけて、
前回記録を見ながら迷わず入力できる形で開始したい。

### 期待する体験

- プログラム一覧で候補を比較できる
- 難しすぎるプログラムを避けやすい
- 詳細を見て自分に合うか判断できる
- 開始後は今日やる内容がすぐ分かる
- 各セット行で前回記録を見ながら入力できる
- 別ページを行き来せず進められる

## 2. 中級者ユーザーが前回記録を見ながら進める

### ストーリー

継続的にトレーニングしているユーザーとして、
前回重量・前回回数・今回目標を比較しながら、
今日のワークアウトをスムーズに進めたい。

### 期待する体験

- 当日画面の各セット行で `Previous` と `Target` が見える
- セットごとに入力しやすい
- 重量、回数、RPE、メモを残せる
- AMRAP のような可変目標も読み違えず進められる

## 3. 継続ユーザーが履歴から次回負荷を判断する

### ストーリー

継続して使うユーザーとして、
自分の履歴を振り返って、
次回の重量や回数の判断材料を得たい。

### 期待する体験

- 履歴ページへ必ず到達できる
- 過去セッションを後から確認できる
- 前回セット内容が次回実行画面にも反映される
- 次回負荷判断に必要な最低限の記録が欠けない

## 4. 管理者が実行 UI で破綻しないようプログラムを管理する

### ストーリー

管理者として、
新しいプログラムや作成者情報を登録しつつ、
その構造が実行 UI で破綻しないよう管理したい。

### 期待する体験

- プログラム基本情報を登録できる
- 週構成と日構成を整備できる
- 種目マスタと作成者マスタを参照できる
- 公開前に非公開で下書き管理できる
- 各セット行に `Previous` と `Target` を出しても崩れない構造で管理できる
- 実行画面から履歴導線を持っても迷わない情報構成にできる

## 補足

今回は初期設計文書の整備が目的なので、ストーリーは代表シナリオに絞る。
ログイン本実装や複数ユーザー権限の細部は後続で追加する。
