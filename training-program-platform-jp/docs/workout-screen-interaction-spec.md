# workout-screen-interaction-spec

## 位置づけ

この文書は「今日のワークアウト」画面の interaction spec（操作仕様書）である。
見た目ではなく、**タップ・入力・自動反映・ロック・履歴遷移など操作時の挙動**を固定することが目的。

前提文書:
- `docs/workout-screen-wireframe.md`（画面設計図）
- `docs/workout-execution-ui.md`（実行UI要件）
- `docs/screens.md`（画面一覧・要件）
- `docs/requirements.md`（機能要件）

---

## 1. 画面の目的

「今日のワークアウト」は、プログラムの当日内容を確認し、前回記録と今回目標を見比べながら、その場で入力を完了させる画面である。

ユーザーはこの画面で次の3つを同時に行う。

- 今日やる内容を確認する
- 前回記録（Previous）と今回目標（Target）を見比べる
- Kg / Reps を入力し、完了チェックで記録を確定する

---

## 2. ユーザーの基本操作フロー

```
画面を開く
  → プログラム情報・種目一覧を確認する
  → 種目ごとに以下を繰り返す
       1. Previous / Target を確認する
       2. Kg を入力する（1セット目入力で後続セットへ自動反映）
       3. Reps を入力する
       4. 完了チェックを入れる（その行がロックされる）
       5. 次のセットへ進む
  → 全種目が終わったら Finish を押す
  → 記録完了画面へ遷移する
```

種目名タップで種目単体履歴へ移動することは、上記フローの任意の時点で可能。

## 3. UI要素別 操作仕様

操作仕様の列構成:

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|

---

### 3-1. 種目名

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| 種目名 | タップ | 種目単体履歴画面へ遷移する | なし | プログラム全体履歴ではなく、当該種目の単体履歴へ移動する |

**補足詳細:**
- 種目名は入力欄ではなく遷移トリガーである
- 遷移先で実行中セッションの入力は失われない
- 戻り操作で実行画面へ戻れることを前提とする（画面遷移の実装詳細は screens.md 参照）

---

### 3-2. Kg 入力欄

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| Kg欄 | タップ | テンキー（数字入力キーボード）が開く | なし（入力中は保存しない） | 画面表示は即時更新する |
| Kg欄（1セット目） | 入力確定 | 同一種目の後続セットの Kg 空欄へ同値を自動反映する | なし（まだ保存しない） | 後続セットに手入力済みの値がある場合は上書きしない（→ 4節 自動反映ルール参照） |
| Kg欄（2セット目以降） | 入力 | 入力値をその行に表示する | なし | 自動反映後の値を個別に上書きできる |
| Kg欄（ロック中） | タップ | 反応しない（入力不可） | なし | ロック解除後のみ入力可能（→ 6節 ロックルール参照） |

---

### 3-3. Reps 入力欄

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| Reps欄 | タップ | テンキーが開く | なし | Target の値は表示するが Reps 欄へ自動入力しない |
| Reps欄 | 入力確定 | 入力値をその行に表示する | なし | プログラム設定によって AMRAP（最大回数挑戦）など入力自由な種目がある |
| Reps欄（ロック中） | タップ | 反応しない（入力不可） | なし | ロック解除後のみ入力可能 |

**補足詳細:**
- Reps は Target を参照して判断するが、Target の値を Reps 欄へ自動確定しない
- `3+ reps`（AMRAP）のような Target を持つセットは、ユーザーが実際に行った回数を直接入力する
- Reps 入力の挙動はプログラム設定に従う前提を置く（固定ロジックを持たない）

---

### 3-4. 完了チェック

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| 完了チェック | タップ（未チェック → チェック） | その行の Kg / Reps 欄をロックする。行全体を視覚的に完了状態にする | Set番号・Kg・Reps・完了状態 | Kg または Reps が空欄のままでもチェック可能とする（暫定）。警告表示の要否は未確定 |
| 完了チェック（チェック済み） | タップ（チェック解除操作） | ロック解除の確認またはトリガーとして機能する | なし | 誤タップ防止のため、直接チェック解除ではなくロック解除UIを経由する（→ 6節参照） |

---

### 3-5. Add Set

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| Add Set | タップ | その種目のセット一覧末尾に新しいセット行を追加する | なし（セッション内の一時的な追加） | 追加行の Target は空欄または直前セットを参照（暫定）。Previous は空欄または `-` |

**未確定事項:** Add Set を MVP 本実装に含めるか、構造準備に留めるかは未決定（→ 9節参照）

### 3-6. Swap

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| Swap | タップ | 種目の置き換えに関する操作UIを表示する（詳細未確定） | 置き換えた種目情報（未確定） | 操作対象が種目単位かセット単位かは未確定 |

**未確定事項:** Swap の操作対象・UIフロー・保存挙動は未定義（→ 9節参照）

### 3-7. Add Exercise

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| Add Exercise | タップ | 種目を追加するための種目選択UIを表示する（詳細未確定） | 追加した種目（未確定） | 画面下部に独立配置。セッション全体への追加操作として機能する |

**未確定事項:** Add Exercise を MVP 本実装に含めるか、構造準備に留めるかは未決定（→ 9節参照）

### 3-8. Rest

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| Rest | タップ | 休憩タイマー（インターバルタイマー）を開始する | なし | タイマーは上部固定エリアに表示される。自動開始（完了チェック直後の自動起動）は未確定 |
| Rest（タイマー動作中） | タップ | タイマーをリセットまたは停止する（暫定） | なし | タイマー詳細挙動は未確定 |

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| Calc | タップ | 重量計算補助UIを表示する（詳細未確定） | なし | 1RM（最大挙上重量）算出やバーベルプレート計算などの補助が候補 |

**未確定事項:** Calc の計算内容・UIフローは未定義（→ 9節参照）

---

### 3-10. Finish

| 要素 | ユーザー操作 | 画面の反応 | 保存される内容 | 補足 |
|---|---|---|---|---|
| Finish | タップ（全セット完了済み） | 記録完了画面へ遷移する | セッション全体の記録（全セット・Kg・Reps・完了状態） | 全種目・全セットが完了チェック済みの場合の正常フロー |
| Finish | タップ（未完了セットあり） | 確認ダイアログ（未完了セットが残っている旨を表示）を出す（暫定） | ユーザーが確認後に続行した場合、完了セットの記録のみ保存する（暫定） | 未完了セットをスキップして完了できる前提を置く。詳細は暫定（→ 9節参照） |

## 4. 入力ルール

### Kg 入力ルール

| ルール | 内容 |
|---|---|
| 初期値 | 空欄（Previous の値を自動入力しない） |
| 入力形式 | 数値のみ。単位は kg 固定 |
| 小数点 | 許容する（例: 82.5）|
| マイナス値 | 許容しない |
| 空欄のままチェック | 可能とする（暫定） |

### Reps 入力ルール

| ルール | 内容 |
|---|---|
| 初期値 | 空欄（Target の値を自動入力しない） |
| 入力形式 | 整数のみ |
| マイナス値 | 許容しない |
| AMRAP セット | ユーザーが実際に行った回数を直接入力する |
| 空欄のままチェック | 可能とする（暫定） |

## 5. 自動反映ルール

### 1セット目 Kg 入力による後続セットへの自動反映

| ルール | 内容 |
|---|---|
| トリガー | 1セット目の Kg 入力が確定したとき |
| 対象 | 同一種目の2セット目以降の Kg 欄が空欄のもの |
| 反映内容 | 1セット目の Kg 値と同じ値 |
| 上書き除外 | すでに手入力が入っている後続セットは上書きしない（暫定） |
| 自動反映後の編集 | 反映された値はユーザーが個別に上書き可能 |

**判断根拠:**
- ユーザーが意図的に入力した後続セットの値を無断で上書きすることを避ける
- ただし「手入力済み」の判断基準（ユーザーが明示的に変えた vs 自動反映後に触れていない）の実装詳細は未確定（→ 9節参照）

```
例:
  Set 1 Kg: [100] を入力確定
  → Set 2 Kg: [100]（空欄だったので自動反映）
  → Set 3 Kg: [100]（空欄だったので自動反映）
  → Set 2 Kg をユーザーが [95] に手入力
    → Set 2 は手入力済みになる
    → この後で Set 1 Kg を変更した場合の再反映は未確定
```

## 6. ロック / ロック解除ルール

### ロック状態になる条件

| 条件 | 内容 |
|---|---|
| 完了チェックを入れたとき | その行の Kg / Reps 欄がロックされる |

### ロック中の挙動

| 項目 | 内容 |
|---|---|
| Kg 欄 | タップ不可（入力不可） |
| Reps 欄 | タップ不可（入力不可） |
| 完了チェック | 直接解除不可。解除操作を経由する |
| 視覚表示 | 行全体をロック状態として区別して表示する（グレーアウトなど。視覚的詳細は wireframe に委ねる） |

### ロック解除の条件

| 条件 | 内容 |
|---|---|
| ユーザーが明示的にロック解除操作を行ったとき | 解除操作の見せ方は未確定（→ 9節参照） |
| 解除後の状態 | 入力済み状態に戻る。Kg / Reps は再編集可能 |
| 再チェック | 再編集後に完了チェックを入れると再度ロック |

### is_locked の persist（保存して残す）について

- セッション中のロック状態はUI状態として保持する
- セッションを離れて戻ったときに is_locked を復元するかは未確定（→ 9節参照）

## 7. 履歴遷移ルール

### 種目名タップによる遷移

| 操作 | 遷移先 | 遷移先の内容 |
|---|---|---|
| 種目名をタップ | 種目単体履歴画面 | その種目の過去セット記録一覧。プログラム全体履歴ではない |

- 遷移先は screens.md の「履歴」画面とは別の「種目単体履歴」として扱う
- 種目単体履歴のレイアウト詳細は未確定（→ 9節参照）

### 遷移後の画面への戻り

- 戻り操作で「今日のワークアウト」画面へ戻ることを前提とする
- 遷移中に入力したセッションデータは失われない

### Previous の参照元

- Previous はこの種目の「前回セッションでの記録」を参照する（暫定）
- 参照単位の詳細（セット番号一致優先 / 種目内最新完了優先）は未確定（→ 9節参照）

---

## 8. エラー時・例外時の扱い

| 状況 | 暫定ルール |
|---|---|
| Kg / Reps が空欄のまま完了チェック | チェックを許容する。警告表示の要否は未確定 |
| Finish 押下時に未完了セットがある | 確認ダイアログを表示し、ユーザーが続行を選んだ場合は完了セットの記録のみ保存する（暫定） |
| Previous が存在しない（初回種目） | `-` を表示する。エラーではない |
| セッション中に画面を閉じた場合 | 入力中データの保全方針は未確定 |
| Add Set / Swap / Add Exercise が未実装の場合 | ボタンを非活性（グレーアウト）またはプレースホルダーとして表示するか、完全に非表示にするかは未確定 |

## 9. ユーザー操作フロー例（ケース別）

### ケース1: 通常のセット入力フロー

```
1. Set 1 の Kg 欄をタップ
2. 重量（例: 100）を入力して確定する
   → Set 2, Set 3 の Kg 空欄へ 100 が自動反映される
3. Set 1 の Reps 欄をタップ
4. 回数（例: 5）を入力して確定する
5. Set 1 の完了チェックをタップ
   → Set 1 の Kg / Reps がロックされる
   → 行全体が完了状態の表示になる
6. Set 2 以降も同様に繰り返す
```

### ケース2: 種目名タップによる履歴確認

```
1. 種目名「Squat」をタップする
   → Squat 単体の過去セット記録一覧が表示される（種目単体履歴画面へ遷移）
2. 過去記録を確認する
3. 戻り操作で「今日のワークアウト」画面へ戻る
   → 入力中のデータはそのまま残っている
```

### ケース3: チェック済みセットの再編集

```
1. ロック済みのセット行を操作しようとする
   → Kg / Reps 欄はタップ不可
2. ロック解除操作を行う（UIは未確定）
   → その行のロックが解除される
   → Kg / Reps が再び編集可能になる
3. Kg または Reps を修正する
4. 完了チェックを再度入れる
   → 行が再ロックされる
```

### ケース4: Previous がない初回種目の入力

```
1. 「今日のワークアウト」画面を開く
2. 初めて記録する種目では、Previous に `-` が表示される
3. Target（例: 5 reps）を参照しながら Kg を入力する
4. Reps を入力する
5. 完了チェックを入れる
   → 記録が保存される
   → 次回からこのセットが Previous として表示される
```

## 10. 未確定事項

以下は現時点で確定していない。実装前に決定が必要なものと、MVP 以降でよいものを分類する。

### 実装前に決定が必要（MVP対象）

| 項目 | 内容 |
|---|---|
| ロック解除 UI | ロックされたセット行をどう解除するか（ボタン配置・確認の有無など） |
| Finish 押下時の未完了セット扱い | 確認ダイアログの文言・スキップ後の保存挙動の詳細 |
| 手入力済み判定 | 自動反映後にユーザーが手入力したかを区別する方法 |
| 空欄チェック時の警告 | Kg / Reps 空欄のまま完了チェックを入れたときに警告を出すか |

### MVP 以降で決定してよい

| 項目 | 内容 |
|---|---|
| Add Set / Swap / Add Exercise の本実装範囲 | MVP に含めるか、構造準備（ボタン非活性表示）に留めるか |
| Previous の参照単位 | セット番号一致優先か、種目内最新完了優先か |
| is_locked の persist | セッションを離れて戻ったときにロック状態を復元するか |
| 履歴画面の詳細レイアウト | 種目単体履歴画面の構成 |
| Rest / Calc の詳細挙動 | タイマー自動開始・Calc の計算内容 |
| 1セット目変更時の再反映 | 自動反映後に1セット目の Kg を変更した場合、後続セットへ再反映するか |
