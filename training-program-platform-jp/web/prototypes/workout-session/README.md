# workout-session — 静的UIプロトタイプ

## このモックの目的

「今日のワークアウト」画面の **静的UIプロトタイプ（見た目確認用の試作品）**。
画面構造・情報量・操作感をブラウザで目で確認するためのモック。
**本番実装ではない。**

---

## 起動方法

### ブラウザで直接開く（最速）

`index.html` をブラウザで直接開く。サーバー不要。

### スマホ確認

```
npx serve .
```

または Chrome DevTools → モバイルビュー（iPhone SE / 375px 幅推奨）

## 実装済みの操作モック

| 操作 | 挙動 |
|---|---|
| セッションタイマー | 00:00 から自動カウントアップ |
| 1セット目 Kg 入力確定 | 後続セットの空欄 Kg へ同値を自動反映（入力済みはスキップ） |
| 完了チェック | 行をロック（Kg / Reps 編集不可）。ロック解除ボタンが出現 |
| ロック解除ボタン | タップでロック解除・再編集可能 |
| Add Set | セット一覧末尾に行を追加（Previous は `-`）|
| 種目名タップ | トースト通知で「履歴へ遷移（未実装）」を表示 |
| Add Exercise | ダミー種目ブロックを末尾に追加 |
| Finish | 未完了セット数を confirm ダイアログで表示 |
| Rest | タイマーモーダルを表示（1分/1:30/2分/3分のプリセット付き）|
| Calc | トースト通知で「実装準備中」を表示 |
| Swap / ・・・ | トースト通知で「実装準備中」を表示 |

## 未確定のまま残している点

- ロック解除 UI の最終的な見せ方（現在はテキストボタン）
- Swap / Add Exercise の種目選択画面の構成
- Calc の計算内容（1RM / プレート計算など）
- Previous の参照ロジック（ダミーデータを手動記述）
- is_locked（ロック状態）の永続化（このモックでは保持しない）
- Rest タイマーの自動起動（完了チェック後の自動起動は未実装）
- Finish 後の記録完了画面遷移
