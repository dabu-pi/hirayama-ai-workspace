### menu_id 命名規則

```
形式: {大区分}_{補足} — 英小文字スネークケース、半角英数のみ

✅ OK例:
  insured_standard        # 保険施術・標準
  self_morning_30         # 自費・30分
  chronic_program_basic   # 慢性疼痛プログラム・基本
  gym_standard            # ジム利用・標準
  gym_monthly             # ジム利用・月額
  option_tape             # オプション・テーピング

❌ NG例（理由）:
  30min_self              # 先頭が数字 → NG
  Gym_Standard            # 大文字含む → NG
  gym standard            # スペース含む → NG
  gym_standard_           # 末尾アンダースコア → NG
  gym__standard           # 連続アンダースコア → NG
```

**ブロックB: 保険単価シナリオ（窓口単価 B列 / 総保険売上単価 C列）**

| 行 | シナリオ | 窓口単価（B列）| 総保険売上単価（C列）|
|---|---|---|---|
| 16 | 弱気 | 500円 | 2,000円 |
| 17 | 標準 | 800円 | **2,500円** |
| 18 | 強気 | 1,500円 | 3,500円 |
| B20 | ★採用する窓口単価（患者負担） | **=B17 → 800円** | （水色セル） |
| B21 | ★採用する総保険売上単価（院収入） | **=C17 → 2,500円** | （水色セル） |

> **用語整理：**
> - 窓口単価 = 患者が毎回窓口で支払う一部負担金（患者視点の価格）
> - 総保険売上単価 = 窓口負担 + 保険者への算定請求額の合計（院の1来院あたり総収入）
>
> 経営試算は総保険売上単価で判断し、患者向け価格感覚は窓口単価で判断する。

**ブロックC: 月次売上試算・整合確認**

| セル | 項目 | 現在値 | 数式 |
|---|---|---|---|
| B25 | 月次窓口売上（試算） | 100,000円 | =B20\*B8 |
| B26 | 月次総保険売上（試算） | 312,500円 | =B21\*B8 |
| B27 | 月次総保険売上（数値前提シート入力値） | 150,000円 | ='数値前提'!B21 |
| B28 | 差異（入力値 − 試算値） | **-162,500円** | =B27-B26 |

> 差異 -162,500円 の意味：総保険売上単価の標準シナリオ（2,500円）が数値前提入力（150,000円）より高い試算になっている。
> → **C17（総保険売上単価・標準シナリオ）を実績値に合わせて修正すること**で差異が縮まる。
